০১

ভেক্টর ডেটাবেস কী?

সাধারণ ডেটাবেস সংখ্যা ও টেক্সট সংরক্ষণ করে। কিন্তু AI, ছবি, শব্দ, ভিডিও — এই জটিল ডেটা কীভাবে সংরক্ষণ ও খোঁজা হবে? উত্তর হলো ভেক্টর ডেটাবেস।

💡
সহজ সংজ্ঞা ভেক্টর ডেটাবেস এমন একটি বিশেষ ডেটাবেস যা ডেটাকে সংখ্যার তালিকা (ভেক্টর/এম্বেডিং) হিসেবে সংরক্ষণ করে এবং অর্থগত মিল (semantic similarity) খুঁজে বের করতে পারে।

🗄️ সাধারণ ডেটাবেস

হুবহু মিলের উপর ভিত্তি করে খোঁজে। "বিড়াল" লিখলে শুধু "বিড়াল"ই পাবে, "ক্যাট" বা "পোষা প্রাণী" পাবে না।

🧠 ভেক্টর ডেটাবেস

অর্থগত মিলের উপর ভিত্তি করে খোঁজে। "বিড়াল" লিখলে "ক্যাট", "পোষা প্রাণী", "মিউ" — সব অর্থগতভাবে কাছের ডেটাই পাবে।

🎯 বাস্তব জীবনের উদাহরণ

ধরুন আপনি একটি বইয়ের দোকানে গেছেন। সাধারণ ডেটাবেস হলো সেই দোকান যেখানে শুধু নাম দিয়ে বই খোঁজা যায়। কিন্তু ভেক্টর ডেটাবেস হলো একজন বিশেষজ্ঞ দোকানদার যে বুঝতে পারে — "আপনি যদি 'হ্যারি পটার' পছন্দ করেন, তাহলে 'নার্নিয়ার গল্প'ও আপনার ভালো লাগবে!"

০২

ভেক্টর ও এম্বেডিং কী?

AI যখন কোনো ডেটা (টেক্সট, ছবি, শব্দ) প্রক্রিয়া করে, সে তাকে একটি সংখ্যার তালিকায় রূপান্তরিত করে। এই তালিকাকেই বলা হয় ভেক্টর বা এম্বেডিং।

"বিড়াল" → ভেক্টর রূপান্তর

একটি শব্দ কীভাবে ৩৮৪-মাত্রার ভেক্টরে পরিণত হয়

প্রতিটি ঘর একটি মাত্রা (dimension) — রং যত গাঢ়, মান তত বেশি

Python
# "বিড়াল" শব্দটিকে ভেক্টরে রূপান্তর করা
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')

# একটি বাক্যকে এম্বেড করো
sentence = "বিড়াল একটি সুন্দর প্রাণী"
embedding = model.encode(sentence)

print(f"ভেক্টরের আকার: {embedding.shape}")
# আউটপুট: ভেক্টরের আকার: (384,)
# মানে ৩৮৪টি সংখ্যার একটি তালিকা!

print(embedding[:5])
# আউটপুট: [-0.0234, 0.0891, -0.1234, 0.0567, 0.0923]

কেন ভেক্টর কাজ করে?

ভেক্টর স্পেসে কাছাকাছি শব্দগুলো একই ধরনের অর্থ বহন করে:

✅
মূল ধারণা একই ধরনের অর্থের শব্দগুলো ভেক্টর স্পেসে কাছাকাছি থাকে। তাই "বিড়াল" খুঁজলে "কুকুর"ও পাওয়া যায়, কারণ দুটোই প্রাণী — একই "এলাকায়" থাকে!

১

ডেটা প্রস্তুতি (Ingestion)

প্রতিটি ডকুমেন্ট/ছবি/শব্দকে একটি AI মডেলের মাধ্যমে ভেক্টরে রূপান্তরিত করা হয়। যেমন: OpenAI-এর text-embedding-ada-002 বা BERT মডেল।

২

ইন্ডেক্স তৈরি (Indexing)

লক্ষ লক্ষ ভেক্টর দ্রুত খোঁজার জন্য বিশেষ ইন্ডেক্স তৈরি করা হয় (HNSW, IVF, LSH ইত্যাদি অ্যালগরিদম ব্যবহার করে)।

৩

কোয়েরি প্রক্রিয়াকরণ (Query)

ব্যবহারকারীর প্রশ্নটিকেও একই মডেল দিয়ে ভেক্টরে রূপান্তরিত করা হয়।

৪

নিকটতম প্রতিবেশী খোঁজা (ANN Search)

কোয়েরি ভেক্টরের সবচেয়ে কাছের K টি ভেক্টর খুঁজে বের করা হয় (Approximate Nearest Neighbor)।

৫

ফলাফল প্রদান (Results)

সবচেয়ে মিলসম্পন্ন ডকুমেন্টগুলো স্কোর সহ ফেরত দেওয়া হয়।

০৪

দূরত্ব পরিমাপ (Distance Metrics)

দুটো ভেক্টর কতটা কাছাকাছি তা মাপার কয়েকটি পদ্ধতি আছে:

📐

Cosine Similarity

cos(θ) = A·B / ‖A‖‖B‖

দুটো ভেক্টরের মধ্যবর্তী কোণ মাপে। টেক্সট সিমিলারিটির জন্য সেরা। মান: -১ থেকে ১ (১ = সম্পূর্ণ একই)

টেক্সটের জন্য সেরা
📏

Euclidean Distance

d = √Σ(aᵢ-bᵢ)²

সরল রেখার দূরত্ব মাপে। ছবি ও অডিওর জন্য ভালো। মান: ০ (একই) থেকে বড়।

ছবির জন্য সেরা
🔢

Dot Product

A·B = Σ(aᵢ × bᵢ)

দ্রুততম গণনা। OpenAI এম্বেডিংয়ের জন্য আদর্শ। নর্মালাইজড ভেক্টরে Cosine-এর সমান।

দ্রুততম
Python - দূরত্ব গণনা
import numpy as np

# দুটো ভেক্টর
birat = np.array([0.8, 0.2, 0.1])  # "বিড়াল"
kukhur = np.array([0.7, 0.3, 0.15]) # "কুকুর"
vat = np.array([0.1, 0.9, 0.8])    # "ভাত"

# Cosine Similarity
def cosine_sim(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

print(f"বিড়াল-কুকুর মিল: {cosine_sim(birat, kukhur):.3f}")
# আউটপুট: বিড়াল-কুকুর মিল: 0.994 (খুব কাছে!)

print(f"বিড়াল-ভাত মিল: {cosine_sim(birat, vat):.3f}")
# আউটপুট: বিড়াল-ভাত মিল: 0.312 (দূরে!)
০৫

ইন্ডেক্সিং অ্যালগরিদম

কোটি কোটি ভেক্টরের মধ্যে দ্রুত খোঁজার জন্য বিশেষ অ্যালগরিদম ব্যবহার করা হয়:

🕸️ HNSW

Hierarchical Navigable Small World

গ্রাফ-ভিত্তিক অ্যালগরিদম। অনেক স্তরের নেটওয়ার্ক তৈরি করে দ্রুত নেভিগেট করে। সবচেয়ে জনপ্রিয় — Pinecone, Weaviate, Qdrant ব্যবহার করে।

সবচেয়ে দ্রুত উচ্চ নির্ভুলতা

📦 IVF

Inverted File Index

ভেক্টরগুলোকে ক্লাস্টারে ভাগ করে। প্রথমে ক্লাস্টার খোঁজো, তারপর সেই ক্লাস্টারে বিস্তারিত খোঁজো। কম মেমোরিতে কাজ করে।

কম মেমোরি বড় ডেটার জন্য

#️⃣ LSH

Locality Sensitive Hashing

কাছাকাছি ভেক্টরগুলোকে একই হ্যাশ বাকেটে রাখে। খুবই দ্রুত কিন্তু নির্ভুলতা কম। প্রাথমিক ফিল্টারিংয়ে ব্যবহার হয়।

কম নির্ভুলতা অতি দ্রুত

🔢 PQ

Product Quantization

ভেক্টরকে ছোট ছোট অংশে কম্প্রেস করে। স্টোরেজ ৯৭% কমিয়ে আনে। IVF-এর সাথে একসাথে ব্যবহার হয় (IVF+PQ)।

৯৭% কম স্টোরেজ FAISS-এ জনপ্রিয়

HNSW কীভাবে কাজ করে:

উপরের স্তরে দ্রুত "লাফিয়ে" যাও, নিচের স্তরে সূক্ষ্মভাবে খোঁজো — এভাবেই HNSW মিলিয়ন নোড থেকে মিলিসেকেন্ডে উত্তর দেয়!

০৬

ঐতিহ্যবাহী vs ভেক্টর ডেটাবেস

বৈশিষ্ট্য SQL/NoSQL ভেক্টর ডেটাবেস
ডেটা ফরম্যাট সারি ও কলাম / JSON উচ্চ-মাত্রিক ভেক্টর
খোঁজার পদ্ধতি হুবহু মিল (Exact match) অর্থগত মিল (Semantic)
কোয়েরি ভাষা SQL / API ভেক্টর কোয়েরি
স্কেলিং ভার্টিকাল/হরিজন্টাল বিশেষভাবে ভেক্টরের জন্য
ব্যবহারক্ষেত্র ট্রানজ্যাকশন, রিপোর্ট AI, সিমিলারিটি সার্চ
উদাহরণ MySQL, MongoDB, PostgreSQL Pinecone, Weaviate, Qdrant
⚠️
সতর্কতা ভেক্টর ডেটাবেস ঐতিহ্যবাহী ডেটাবেসকে প্রতিস্থাপন করে না, বরং সম্পূরক হিসেবে কাজ করে। বেশিরভাগ সিস্টেমে দুটোই একসাথে ব্যবহার হয়।
০৭

জনপ্রিয় ভেক্টর ডেটাবেস

🌲

Pinecone

ক্লাউড-ম্যানেজড

সবচেয়ে জনপ্রিয় ম্যানেজড ভেক্টর ডেটাবেস। সেটআপ শূন্য, স্কেল করা সহজ। স্টার্টআপ ও এন্টারপ্রাইজ উভয়ের জন্য।

  • HNSW ইন্ডেক্স
  • ফ্রি টায়ার উপলব্ধ
  • LangChain ইন্টিগ্রেশন
🔷

Weaviate

ওপেন-সোর্স

গ্রাফকিউএল সাপোর্ট সহ ওপেন-সোর্স। নিজস্ব সার্ভারে চালানো যায়। মডুলার আর্কিটেকচার।

  • GraphQL API
  • বিল্ট-ইন মডেল সাপোর্ট
  • মাল্টি-মডেল সার্চ
🎯

Qdrant

Rust-ভিত্তিক

Rust দিয়ে তৈরি, তাই অত্যন্ত দ্রুত ও মেমোরি-দক্ষ। ফিল্টারিং ও পেলোড সাপোর্ট চমৎকার।

  • সবচেয়ে দ্রুত পারফরম্যান্স
  • উন্নত ফিল্টারিং
  • Docker সাপোর্ট
🦙

Chroma

ডেভেলপার-ফার্স্ট

ডেভেলপারদের জন্য সবচেয়ে সহজ। লোকালি চালানো যায়, কোনো সেটআপ ছাড়াই। র‍্যাপিড প্রোটোটাইপিংয়ের জন্য আদর্শ।

  • পাইথনে ৩ লাইনে শুরু
  • LangChain/LlamaIndex সাপোর্ট
  • ইন-মেমোরি মোড

⚡ FAISS (Facebook AI Similarity Search)

Meta-র তৈরি লাইব্রেরি। ডেটাবেস নয়, লাইব্রেরি — তবে সবচেয়ে শক্তিশালী। GPU সাপোর্ট আছে। বিলিয়ন ভেক্টর হ্যান্ডেল করতে পারে। রিসার্চ ও প্রোডাকশন উভয়ে জনপ্রিয়।

০৮

ব্যবহারের ক্ষেত্র

Retrieval-Augmented Generation — ChatGPT-এর মতো AI-কে নিজের ডেটাবেস থেকে তথ্য দিয়ে আরও স্মার্ট করা।

🔍 সিমান্টিক সার্চ

"সস্তায় ভালো হোটেল" লিখলেই সংশ্লিষ্ট হোটেল পাওয়া — শুধু কীওয়ার্ড মিল না, অর্থ বুঝে খোঁজা।

🟣 Chroma দিয়ে শুরু (সবচেয়ে সহজ)

Python - Chroma
import chromadb
from chromadb.utils import embedding_functions

# ১. Chroma ক্লায়েন্ট তৈরি করো
client = chromadb.Client()

# ২. একটি কালেকশন তৈরি করো
collection = client.create_collection(
    name="বাংলা_ডকুমেন্ট",
    embedding_function=embedding_functions.DefaultEmbeddingFunction()
)

# ৩. কিছু ডকুমেন্ট যোগ করো
collection.add(
    documents=[
        "বিড়াল একটি পোষা প্রাণী যে মাছ খেতে ভালোবাসে",
        "কুকুর মানুষের সবচেয়ে বিশ্বস্ত বন্ধু",
        "ভাত বাংলাদেশের প্রধান খাবার",
        "বাংলাদেশের জাতীয় ফুল শাপলা",
        "পাখি আকাশে উড়তে পারে কারণ তাদের পালক আছে",
    ],
    ids=["doc1", "doc2", "doc3", "doc4", "doc5"]
)

# ৪. সিমান্টিক সার্চ করো
results = collection.query(
    query_texts=["কোন প্রাণী মানুষের কাছের?"],
    n_results=2
)

print(results['documents'])
# আউটপুট: [['কুকুর মানুষের সবচেয়ে বিশ্বস্ত বন্ধু',
#            'বিড়াল একটি পোষা প্রাণী যে মাছ খেতে ভালোবাসে']]

🤖 সহজ RAG সিস্টেম (LangChain + Chroma)

Python - RAG System
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain.chains import RetrievalQA
from langchain.schema import Document

# ১. আমাদের জ্ঞানভাণ্ডার তৈরি করো
docs = [
    Document(page_content="বাংলাদেশ ১৯৭১ সালে স্বাধীনতা লাভ করে"),
    Document(page_content="ঢাকা বাংলাদেশের রাজধানী এবং বৃহত্তম শহর"),
    Document(page_content="পদ্মা সেতু বাংলাদেশের সবচেয়ে দীর্ঘ সেতু"),
    Document(page_content="বাংলাদেশের জাতীয় পাখি দোয়েল"),
]

# ২. ভেক্টর স্টোর তৈরি করো
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(
    documents=docs,
    embedding=embeddings
)

# ৩. RAG চেইন তৈরি করো
llm = ChatOpenAI(model="gpt-3.5-turbo")
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=vectorstore.as_retriever(search_kwargs={"k": 2})
)

# ৪. প্রশ্ন করো!
answer = qa_chain.invoke("বাংলাদেশের রাজধানী কোথায়?")
print(answer['result'])
# আউটপুট: বাংলাদেশের রাজধানী ঢাকা।
🎉
RAG কেন গুরুত্বপূর্ণ? ChatGPT ২০২৩ সালের পর কিছু জানে না। কিন্তু RAG দিয়ে তুমি AI-কে তোমার নিজের ডেটাবেস থেকে উত্তর দিতে শেখাতে পারো — যেকোনো ডকুমেন্ট, ওয়েবসাইট, বা ডেটা থেকে!

🌲 Pinecone উদাহরণ

Python - Pinecone
from pinecone import Pinecone, ServerlessSpec
import numpy as np

# ১. Pinecone সংযোগ
pc = Pinecone(api_key="তোমার-API-কী")

# ২. ইন্ডেক্স তৈরি করো
pc.create_index(
    name="bangla-search",
    dimension=1536,  # OpenAI embedding সাইজ
    metric="cosine",
    spec=ServerlessSpec(cloud="aws", region="us-east-1")
)

index = pc.Index("bangla-search")

# ৩. ডেটা আপলোড করো (ভেক্টর + মেটাডেটা)
vectors = [
    {
        "id": "প্রোডাক্ট-০০১",
        "values": np.random.rand(1536).tolist(),  # আসলে এম্বেডিং
        "metadata": {
            "নাম": "নীল শাড়ি",
            "দাম": 1500,
            "ক্যাটাগরি": "পোশাক"
        }
    }
]
index.upsert(vectors=vectors)

# ৪. সার্চ করো + ফিল্টার করো
results = index.query(
    vector=np.random.rand(1536).tolist(),
    top_k=5,
    filter={"দাম": {"$lt": 2000}},  # ২০০০ টাকার কম
    include_metadata=True
)

for match in results.matches:
    print(f"স্কোর: {match.score:.3f} | {match.metadata['নাম']}")

→
প্রশ্নটি ভেক্টরে রূপান্তরিত হয়

LLM
