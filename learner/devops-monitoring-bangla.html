Prometheus

Metrics collection, alerting rules, PromQL queries, exporters — সব কিছু বাংলায়।

🎯 Prometheus কী?

Prometheus হলো open-source monitoring ও alerting system যা SoundCloud তৈরি করেছিল এবং এখন CNCF এর অংশ। এটি time-series ডেটা সংগ্রহ করে এবং PromQL দিয়ে query করা যায়।

সহজ ভাষায়: আপনার সার্ভার, অ্যাপ, ডেটাবেজ ঠিকমতো চলছে কিনা — CPU কত%, মেমরি কতটুকু ব্যবহার হচ্ছে, কতগুলো HTTP request আসছে — এসব metrics Prometheus সেকেন্ডে সেকেন্ডে ট্র্যাক করে।

📐 PromQL — Prometheus Query Language

PromQL দিয়ে metrics থেকে যেকোনো তথ্য বের করা যায়। এটি শেখা Grafana dashboard তৈরির জন্য অপরিহার্য।

PromQL — বেসিক থেকে অ্যাডভান্সড 
## ===== INSTANT VECTOR (এখনকার মান) =====

# সব HTTP request এর সংখ্যা
http_requests_total

# শুধু status=200 এর request
http_requests_total{status="200"}

# production environment এর error request
http_requests_total{env="prod", status=~"5.."}

## ===== RANGE VECTOR (নির্দিষ্ট সময়ের ডেটা) =====

# গত ৫ মিনিটের HTTP request ডেটা
http_requests_total[5m]

## ===== FUNCTIONS (গণনা ও বিশ্লেষণ) =====

# প্রতি সেকেন্ডে গড় request rate (গত ৫ মিনিট)
rate(http_requests_total[5m])

# CPU ব্যবহারের শতাংশ (node exporter)
100 - (avg by (instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100)

# মেমরি ব্যবহারের শতাংশ
(node_memory_MemTotal_bytes - node_memory_MemAvailable_bytes) / node_memory_MemTotal_bytes * 100

# ডিস্ক ব্যবহারের শতাংশ
(node_filesystem_size_bytes - node_filesystem_free_bytes) / node_filesystem_size_bytes * 100

# গত ১ ঘন্টায় 99th percentile response time
histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[1h]))

# কোন instance সবচেয়ে বেশি error করছে
topk(5, rate(http_requests_total{status=~"5.."}[5m]))

# সব job এর মোট request
sum by (job) (rate(http_requests_total[5m]))
🚨 Alerting Rules

কখন alert পাঠাবে তা নির্ধারণ করে alerting rules। YAML ফরম্যাটে লেখা হয়।

alert_rules.yml 
groups:
  - name: server-alerts
    rules:

      # CPU ৮৫% এর বেশি হলে warning
      - alert: HighCPUUsage
        expr: cpu_usage_percent > 85
        for: 5m         # ৫ মিনিট ধরে থাকলে তখনই alert
        labels:
          severity: warning
        annotations:
          summary: "CPU ব্যবহার বেশি হয়েছে"
          description: "{{ $labels.instance }} তে CPU {{ $value }}% ব্যবহার হচ্ছে"

      # সার্ভার ডাউন হলে critical alert
      - alert: InstanceDown
        expr: up == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "সার্ভার বন্ধ হয়ে গেছে!"
          description: "{{ $labels.job }} / {{ $labels.instance }} down"

      # মেমরি ৯০% এর বেশি
      - alert: HighMemoryUsage
        expr: node_memory_usage_percent > 90
        for: 10m
        labels:
          severity: critical
        annotations:
          summary: "মেমরি প্রায় শেষ!"

      # HTTP error rate ৫% এর বেশি
      - alert: HighErrorRate
        expr: |
          rate(http_requests_total{status=~"5.."}[5m]) /
          rate(http_requests_total[5m]) > 0.05
        for: 2m
        labels:
          severity: warning
        annotations:
          description: "Error rate {{ $value | humanizePercentage }}"

Python — Custom Metrics Exporter 
from prometheus_client import (
    Counter, Gauge, Histogram, Summary,
    start_http_server
)
import time, random

# Counter — শুধু বাড়ে (request count, error count)
REQUEST_COUNT = Counter(
    'app_requests_total',
    'মোট HTTP request সংখ্যা',
    ['method', 'endpoint', 'status']
)

# Gauge — বাড়তে-কমতে পারে (active users, queue size)
ACTIVE_USERS = Gauge('app_active_users', 'এখন active ব্যবহারকারীর সংখ্যা')

# Histogram — response time distribution
REQUEST_DURATION = Histogram(
    'app_request_duration_seconds',
    'Request সম্পন্ন হতে সময়',
    buckets=[0.01, 0.05, 0.1, 0.5, 1.0, 5.0]
)

# Decorator দিয়ে function track করুন
def track_request(method, endpoint):
    def decorator(func):
        def wrapper(*args, **kwargs):
            with REQUEST_DURATION.time():
                result = func(*args, **kwargs)
            REQUEST_COUNT.labels(method, endpoint, '200').inc()
            return result
        return wrapper
    return decorator

# :8000/metrics এ expose করুন
start_http_server(8000)
print("✅ Metrics server চালু: http://localhost:8000/metrics")

Grafana

Beautiful dashboards, data source integration, alerting, panels — সম্পূর্ণ গাইড।

🎯 Grafana কী?

Grafana হলো সবচেয়ে জনপ্রিয় open-source visualization platform। Prometheus, Elasticsearch, MySQL, PostgreSQL, InfluxDB সহ ৫০+ data source থেকে ডেটা এনে সুন্দর dashboard তৈরি করা যায়।

সহজ কথায়: Prometheus ডেটা সংগ্রহ করে, আর Grafana সেই ডেটাকে সুন্দর গ্রাফ ও chart এ রূপান্তর করে।

  • Bar chart — বার গ্রাফ

ELK Stack

Elasticsearch, Logstash, Kibana — সম্পূর্ণ log management সিস্টেম বাংলায়।

🎯 ELK Stack কী?

ELK হলো তিনটি open-source টুলের সমন্বয়:

  • E Elasticsearch — ডেটা store এবং search engine। লগ JSON format এ সংরক্ষণ করে এবং দ্রুত search করে।
  • L Logstash — ডেটা pipeline। বিভিন্ন source থেকে লগ সংগ্রহ করে, process করে, Elasticsearch এ পাঠায়।
  • K Kibana — ওয়েব UI। Elasticsearch এর ডেটা দেখার ও বিশ্লেষণের interface।

আধুনিক stack এ Beats (lightweight data shipper) এবং Elastic Agent ও ব্যবহার হয়।

⚙️ Logstash Pipeline কনফিগ

Logstash pipeline তিনটি ধাপে কাজ করে: Input → Filter → Output

logstash.conf — Nginx লগ Processing 
# ========== INPUT: কোথা থেকে লগ আনবে ==========
input {
  # Filebeat থেকে
  beats {
    port => 5044
  }

  # সরাসরি ফাইল থেকে
  file {
    path => "/var/log/nginx/access.log"
    type => "nginx-access"
    start_position => "beginning"
  }

  # Syslog থেকে
  syslog {
    port => 5140
    type => "syslog"
  }
}

# ========== FILTER: লগ parse ও transform ==========
filter {
  if [type] == "nginx-access" {

    # Grok দিয়ে Nginx লগ parse করুন
    grok {
      match => {
        "message" => "%{COMBINEDAPACHELOG}"
      }
    }

    # IP থেকে দেশ বের করুন
    geoip {
      source => "clientip"
      target => "geoip"
    }

    # তারিখ parse করুন
    date {
      match => ["timestamp", "dd/MMM/yyyy:HH:mm:ss Z"]
    }

    # Response time এর ধরন নির্ধারণ করুন
    ruby {
      code => "
        rt = event.get('request_time').to_f
        event.set('response_category',
          rt < 0.1 ? 'fast' :
          rt < 1.0 ? 'normal' : 'slow'
        )
      "
    }

    # অপ্রয়োজনীয় field মুছুন
    mutate {
      remove_field => ["message", "@version"]
      add_field => { "environment" => "production" }
    }
  }
}

# ========== OUTPUT: কোথায় পাঠাবে ==========
output {
  elasticsearch {
    hosts => ["http://elasticsearch:9200"]
    index => "nginx-logs-%{+YYYY.MM.dd}"
    user => "elastic"
    password => "পাসওয়ার্ড"
  }

  # Debug করার জন্য console এও দেখুন
  stdout {
    codec => rubydebug
  }
}

Python — Elasticsearch Client 
from elasticsearch import Elasticsearch
from datetime import datetime

# সংযোগ করুন
es = Elasticsearch(
    ["http://localhost:9200"],
    basic_auth=("elastic", "পাসওয়ার্ড")
)

# লগ পাঠান
def log_event(level, service, message, **kwargs):
    doc = {
        "timestamp": datetime.utcnow(),
        "level": level,
        "service": service,
        "message": message,
        **kwargs
    }
    es.index(
        index=f"app-logs-{datetime.now().strftime('%Y.%m.%d')}",
        document=doc
    )

# Full-text search করুন
def search_logs(query_text, level=None):
    query = {
        "bool": {
            "must": [{"match": {"message": query_text}}]
        }
    }
    if level:
        query["bool"]["filter"] = [{"term": {"level": level}}]

    result = es.search(index="app-logs-*", query=query)
    for hit in result["hits"]["hits"]:
        print(f"[{hit['_source']['level']}] {hit['_source']['message']}")

Python — OpenTelemetry + Jaeger Integration 
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.jaeger.thrift import JaegerExporter
from opentelemetry.sdk.resources import SERVICE_NAME, Resource
import time

# Tracer setup
jaeger_exporter = JaegerExporter(
    agent_host_name="localhost",
    agent_port=6831,
)

provider = TracerProvider(
    resource=Resource.create({SERVICE_NAME: "payment-service"})
)
provider.add_span_processor(BatchSpanProcessor(jaeger_exporter))
trace.set_tracer_provider(provider)

tracer = trace.get_tracer("payment-service")

# Tracing ব্যবহার করুন
def process_payment(order_id, amount):
    # Parent span তৈরি করুন
    with tracer.start_as_current_span("process-payment") as span:
        span.set_attribute("order.id", order_id)
        span.set_attribute("payment.amount", amount)

        # Child span — ডেটাবেজ অপারেশন
        with tracer.start_as_current_span("db-query") as db_span:
            db_span.set_attribute("db.type", "postgresql")
            time.sleep(0.05)  # DB query সিমুলেশন

        # Child span — বাইরের API কল
        with tracer.start_as_current_span("payment-gateway-api") as api_span:
            api_span.set_attribute("http.url", "https://gateway.example.com/charge")
            time.sleep(0.12)  # API call সিমুলেশন

        print(f"✅ Payment processed: {order_id}")

# চালান
process_payment("order-456", 1500.00)

Python — Vault Integration (hvac library) 
import hvac
import os
from functools import lru_cache

class VaultClient:
    def __init__(self):
        self.client = hvac.Client(
            url=os.getenv("VAULT_ADDR", "http://localhost:8200")
        )
        self._authenticate()

    def _authenticate(self):
        # AppRole authentication (production এ)
        role_id = os.getenv("VAULT_ROLE_ID")
        secret_id = os.getenv("VAULT_SECRET_ID")

        if role_id and secret_id:
            self.client.auth.approle.login(
                role_id=role_id,
                secret_id=secret_id
            )
        else:
            # Development: token দিয়ে
            self.client.token = os.getenv("VAULT_TOKEN")

    @lru_cache(maxsize=128)
    def get_secret(self, path):
        # Cache করুন — বারবার Vault call করবেন না
        response = self.client.secrets.kv.read_secret_version(
            path=path
        )
        return response["data"]["data"]

    def get_db_config(self):
        secret = self.get_secret("database/prod")
        return {
            "host": secret["host"],
            "port": secret["port"],
            "user": secret["username"],
            "password": secret["password"]
        }

# ব্যবহার
vault = VaultClient()
db_config = vault.get_db_config()
print(f"✅ DB Config loaded securely from Vault!")
