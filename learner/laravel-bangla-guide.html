লোকালে Laravel প্রজেক্ট রান করা

🖥️

Laravel লোকালে চালানোর আগে কিছু জিনিস থাকতে হবে — PHP (8.1+), Composer, Node.js/NPM। এই তিনটা ইনস্টল থাকলেই লোকালে যেকোনো Laravel প্রজেক্ট চালানো যায়।

প্রথমে প্রজেক্ট ক্লোন বা ডাউনলোড করুন

bash Terminal 
# Git থেকে clone করলে
git clone https://github.com/your-repo/project.git
cd project

# অথবা ZIP ডাউনলোড করে extract করুন, তারপর folder-এ যান
cd C:/xampp/htdocs/myproject

Composer দিয়ে Dependencies ইনস্টল

bash Terminal 
# vendor folder তৈরি হবে — সব PHP package ইনস্টল হবে
composer install

# Production এ হলে (dev packages বাদ দিয়ে)
composer install --no-dev --optimize-autoloader

.env ফাইল সেটআপ

bash Terminal 
# .env.example থেকে .env কপি করুন
cp .env.example .env

# Application Key তৈরি করুন (খুব গুরুত্বপূর্ণ!)
php artisan key:generate
⚠️
মনে রাখুন!

APP_KEY ছাড়া Laravel কাজ করবে না। প্রতিটা প্রজেক্টে এটা generate করতেই হবে। এই key session ও encryption এর জন্য ব্যবহার হয়।

Database সেটআপ

env .env ফাইল 
# লোকাল Database Config
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=my_database_name
DB_USERNAME=root
DB_PASSWORD=         # XAMPP এ সাধারণত খালি
bash Terminal 
# Migration চালিয়ে Database table তৈরি করুন
php artisan migrate

# Seeder দিয়ে ডেমো ডেটা ভরুন (দরকার হলে)
php artisan db:seed

# Migrate + Seed একসাথে
php artisan migrate --seed

Storage Link তৈরি

bash Terminal 
# storage/app/public কে public/storage এ link করে
php artisan storage:link

# এটা না করলে upload করা image/file দেখা যাবে না

Artisan Server দিয়ে রান করা

⚡

Artisan Server হলো Laravel এর বিল্ট-ইন PHP development server। XAMPP, WAMP ছাড়াই শুধু PHP দিয়ে প্রজেক্ট চালানো যায়। Development এর জন্য অনেক সুবিধাজনক।

✅ সুবিধা
  • আলাদা Apache/Nginx লাগে না
  • যেকোনো Port এ চালানো যায়
  • Setup খুব সহজ
  • Real-time error দেখা যায়
  • Multiple project আলাদা port এ
❌ সীমাবদ্ধতা
  • শুধু development এর জন্য
  • Production এ ব্যবহার করা উচিত না
  • Performance কম
  • একটা request একসময় handle করে
  • SSL সাপোর্ট নেই

Basic Commands

bash Terminal 
# Default: localhost:8000 এ চালু হবে
php artisan serve

# Custom port নির্ধারণ করতে
php artisan serve --port=8080

# Custom host নির্ধারণ করতে (অন্য device থেকে access করতে)
php artisan serve --host=0.0.0.0 --port=8000

# এখন ব্রাউজারে যান: http://localhost:8000
💡
Multiple Project একসাথে চালানো

একাধিক Laravel project চালাতে আলাদা আলাদা terminal খুলুন এবং ভিন্ন ভিন্ন port ব্যবহার করুন। যেমন: --port=8001, --port=8002

Artisan Serve কীভাবে কাজ করে?

১

PHP Built-in Server চালু করে

আসলে php -S localhost:8000 -t public/ কমান্ডটাই চালায়। Laravel এর router.php ফাইল দিয়ে request handle করে।

২

public/ ফোল্ডার Document Root

সব request public/index.php এর মধ্য দিয়ে যায়। এই ফাইলটাই Laravel এর entry point।

৩

Request Laravel এ আসে

index.php → bootstrap → Kernel → Router → Controller → Response। এভাবে request process হয়।

৩

Serve ছাড়া URL/Folder দিয়ে রান (XAMPP/WAMP)

📂

XAMPP বা WAMP ব্যবহার করলে Artisan serve না চালিয়েও প্রজেক্ট চালানো যায়। কিন্তু সঠিক ফোল্ডার structure ও Virtual Host সেটআপ করতে হয়।

পদ্ধতি ১: সরাসরি htdocs এ রাখা (সহজ কিন্তু সমস্যা আছে)

📁 ফোল্ডার Structure
structure XAMPP htdocs 
C:/xampp/htdocs/
└── myproject/
    ├── app/
    ├── public/          ← এটাই আসল web root হওয়া উচিত
    ├── resources/
    ├── .env
    └── ...

# URL হবে: http://localhost/myproject/public/
# সমস্যা: URL এ /public/ আসে যা ugly দেখায়
🚫
বড় সমস্যা!

সরাসরি htdocs এ রাখলে যে কেউ localhost/myproject/.env ব্রাউজ করে .env ফাইল দেখতে পারবে। এটা বিশাল security risk। সঠিক পদ্ধতি হলো Virtual Host বা শুধু public ফোল্ডার expose করা।

পদ্ধতি ২: Virtual Host সেটআপ (সঠিক পদ্ধতি)

apache config C:/xampp/apache/conf/extra/httpd-vhosts.conf 
<VirtualHost *:80>
    ServerName    myproject.local
    DocumentRoot  "C:/xampp/htdocs/myproject/public"

    <Directory "C:/xampp/htdocs/myproject/public">
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog  "logs/myproject-error.log"
</VirtualHost>
hosts file C:/Windows/System32/drivers/etc/hosts (Admin হিসেবে খুলুন) 
127.0.0.1    myproject.local

# এখন browser এ: http://myproject.local → সরাসরি কাজ করবে!

পদ্ধতি ৩: .htaccess দিয়ে /public সরিয়ে দেওয়া

apache htdocs/myproject/.htaccess (root এ তৈরি করুন) 
Options -Indexes
RewriteEngine On
RewriteRule ^$ public/ [L]
RewriteRule (.*) public/$1 [L]

# এখন: http://localhost/myproject/ কাজ করবে
# /public/ URL এ আসবে না কিন্তু folder টা accessible
পদ্ধতি URL Security Recommended
htdocs + /public/ localhost/project/public/ বিপজ্জনক না
Root .htaccess localhost/project/ মাঝামাঝি লোকালে ঠিক আছে
Virtual Host myproject.local/ নিরাপদ ✅ সেরা
Artisan Serve localhost:8000/ নিরাপদ ✅ Development

Asset Build করা (CSS, JS, Images)

🎨

Laravel এ CSS ও JavaScript resources/ ফোল্ডারে থাকে। এগুলো সরাসরি browser এ কাজ করে না। Vite বা Mix দিয়ে compile করে public/ ফোল্ডারে নিতে হয়।

Laravel Asset কীভাবে কাজ করে?

Source Files → resources/

এখানে আপনি code লেখেন।

Build Tool Process করে

Output → public/build/ বা public/

Compiled ফাইলগুলো public ফোল্ডারে যায়। Browser এখান থেকে load করে।

Blade Template এ @vite দিয়ে Load

@vite(['resources/css/app.css', 'resources/js/app.js']) — এটা সঠিক file path দেয়।

vite.config.js বোঝা

javascript vite.config.js 
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel({
            input: [
                'resources/css/app.css',   // CSS entry point
                'resources/js/app.js',    // JS entry point
            ],
            refresh: true,  // Blade change হলে auto reload
        }),
    ],
});

NPM Build — Development vs Production

📦

প্রথমে NPM Dependencies ইনস্টল করুন

bash Terminal 
# node_modules ফোল্ডার তৈরি হবে
npm install

# অথবা yarn ব্যবহার করলে
yarn install

Development Mode (Watch)

bash Terminal 
# ===== Development Build (Vite) =====
npm run dev

# এটা করলে:
# ✅ Hot Module Replacement (HMR) চালু হয়
# ✅ CSS/JS change হলে browser auto refresh হয়
# ✅ Vite dev server port 5173 এ চলে
# ❌ Files minify হয় না (development তাই)
# ❌ public/build/ এ কিছু লেখে না সরাসরি
⚠️
গুরুত্বপূর্ণ!

npm run dev চালানো অবস্থায় একটা আলাদা Vite server চলে (port 5173)। php artisan serve ও npm run dev দুটোই একসাথে চালাতে হবে development এ।

Production Build

bash Terminal 
# ===== Production Build =====
npm run build

# এটা করলে:
# ✅ public/build/ ফোল্ডারে compiled ফাইল তৈরি হয়
# ✅ CSS ও JS minify হয় (size অনেক কমে যায়)
# ✅ Filename এ hash যোগ হয় (cache busting)
# ✅ public/build/manifest.json তৈরি হয়
# ✅ Vite server আর দরকার নেই

# Build শেষে এই ফাইলগুলো তৈরি হবে:
# public/build/assets/app-{hash}.css
# public/build/assets/app-{hash}.js
# public/build/manifest.json
বিষয় npm run dev npm run build
ব্যবহার Development Production/Server
Minification না হ্যাঁ
HMR / Hot Reload হ্যাঁ না
public/build/ তৈরি না (memory তে) হ্যাঁ
File Size বড় (debug info আছে) ছোট (optimized)
Server Deploy কখনো না অবশ্যই

Config, Route, View Cache — সব কিছু

🔧

Laravel Cache ব্যবহার করে অনেক কাজ দ্রুত করে। Config, Route, View cache করলে প্রতিটা request এ ফাইল read না করে cache থেকে নেয়, ফলে performance অনেক বাড়ে।

Config Cache

bash Terminal 
# সব config ফাইল একটা cache ফাইলে merge করে
php artisan config:cache
# ফাইল তৈরি হয়: bootstrap/cache/config.php

# Config cache clear করতে
php artisan config:clear

# ⚠️ Cache করার পর .env পড়া যায় না সরাসরি!
# .env পরিবর্তন করলে config:clear তারপর config:cache দিন
🚫
Config Cache করার পর .env কাজ করে না!

config:cache দেওয়ার পর code এ env('APP_KEY') সরাসরি call করলে null পাবেন। সবসময় config('app.key') ব্যবহার করুন।

Route Cache

bash Terminal 
# সব route একটা cache ফাইলে compile করে (অনেক দ্রুত হয়)
php artisan route:cache
# ফাইল তৈরি হয়: bootstrap/cache/routes-v7.php

# Route cache clear
php artisan route:clear

# Route list দেখতে
php artisan route:list

# ⚠️ Route file এ Closure (anonymous function) থাকলে cache হবে না!
# Controller method ব্যবহার করুন সবসময়

View Cache

bash Terminal 
# সব Blade template pre-compile করে রাখে
php artisan view:cache
# Storage/framework/views/ এ compiled PHP ফাইল তৈরি হয়

# View cache clear
php artisan view:clear

সব কিছু একসাথে — Optimize Command

bash Terminal — Server Deploy এর পর এটাই চালান 
# ===== সব Cache একসাথে ===== 
php artisan optimize

# এটা করে: config:cache + route:cache + view:cache একসাথে
# Production এ এটাই সবচেয়ে সহজ ও কার্যকর পদ্ধতি

# ===== সব Cache Clear একসাথে =====
php artisan optimize:clear

# এটা করে: config:clear + route:clear + view:clear + cache:clear

# ===== Application Cache (Database/Redis/File) =====
php artisan cache:clear    # App cache clear

# ===== Autoloader Optimize =====
composer dump-autoload --optimize
Command কী করে কখন লাগে
config:cache সব config মিলিয়ে একটা ফাইল Deploy পরে
route:cache Route compile করে Deploy পরে
view:cache Blade template compile Deploy পরে
cache:clear App data cache মুছে Bug হলে
optimize সব cache একসাথে Deploy পরে সেরা
optimize:clear সব cache clear Problem হলে

Route ও View কীভাবে কাজ করে

🛣️

Route কীভাবে কাজ করে?

php routes/web.php 
// ১. Simple Route — সরাসরি View দেখায়
Route::get('/', function () {
    return view('welcome');  // resources/views/welcome.blade.php
});

// ২. Controller Route — Best Practice
Route::get('/products', [ProductController::class, 'index']);
Route::post('/products', [ProductController::class, 'store']);

// ৩. Named Route
Route::get('/about', function () {
    return view('about');
})->name('about');
// Blade এ: {{ route('about') }} → /about URL দেবে

// ৪. Route Parameter
Route::get('/product/{id}', [ProductController::class, 'show']);

// ৫. Resource Route (CRUD সব একসাথে)
Route::resource('products', ProductController::class);

// ৬. Middleware সহ Route
Route::middleware('auth')->group(function () {
    Route::get('/dashboard', [DashboardController::class, 'index']);
});

View (Blade Template) কীভাবে কাজ করে?

php resources/views/products/index.blade.php 
{{-- Layout extend করা --}}
@extends('layouts.app')

@section('title', 'Products')

@section('content')
    {{-- Variable প্রিন্ট (XSS safe) --}}
    <h1>{{ $title }}</h1>

    {{-- Loop --}}
    @foreach($products as $product)
        <p>{{ $product->name }}</p>
    @endforeach

    {{-- Condition --}}
    @if($products->isEmpty())
        <p>কোনো পণ্য নেই</p>
    @endif

    {{-- Asset URL --}}
    <img src="{{ asset('images/logo.png') }}">
    
    {{-- Route URL --}}
    <a href="{{ route('products.index') }}">সব পণ্য</a>
@endsection

Controller থেকে View এ data পাঠানো

php app/Http/Controllers/ProductController.php 
public function index()
{
    $products = Product::paginate(10);

    // পদ্ধতি ১: compact()
    return view('products.index', compact('products'));

    // পদ্ধতি ২: with()
    return view('products.index')->with('products', $products);

    // পদ্ধতি ৩: array
    return view('products.index', ['products' => $products]);
}

Server এ .env — সম্পূর্ণ Security Guide

🔒
🚨
সবচেয়ে বড় ভুল!

.env ফাইল কখনো Git এ commit করবেন না বা public_html এর ভেতরে রাখবেন না। এতে database password, API key সব expose হয়ে যায়।

Production .env এর সঠিক Config

env .env — Production Server 
# ===== APP CONFIG =====
APP_NAME="My Laravel App"
APP_ENV=production              # ← অবশ্যই production!
APP_KEY=base64:xxxxxxxxxx...     # ← key:generate দিয়ে বানান
APP_DEBUG=false                 # ← অবশ্যই false! Error দেখাবে না
APP_URL=https://yourdomain.com   # ← আপনার domain

# ===== DATABASE =====
DB_CONNECTION=mysql
DB_HOST=127.0.0.1              # বা hosting এর DB host
DB_PORT=3306
DB_DATABASE=cpanelusername_dbname  # cPanel: username_dbname
DB_USERNAME=cpanelusername_dbuser
DB_PASSWORD=StrongPassword123!   # শক্তিশালী password

# ===== CACHE / SESSION =====
CACHE_DRIVER=file              # Redis না থাকলে file
SESSION_DRIVER=file
SESSION_LIFETIME=120
QUEUE_CONNECTION=database

# ===== MAIL =====
MAIL_MAILER=smtp
MAIL_HOST=mail.yourdomain.com
MAIL_PORT=465
MAIL_USERNAME=noreply@yourdomain.com
MAIL_PASSWORD=mailpassword
MAIL_ENCRYPTION=ssl
MAIL_FROM_ADDRESS=noreply@yourdomain.com
MAIL_FROM_NAME="My App"

# ===== LOG =====
LOG_CHANNEL=stack
LOG_LEVEL=error               # শুধু error log রাখুন

Production এর জন্য Critical Settings

🔴 APP_DEBUG=false কেন?

Debug true থাকলে error এ database password, file path, code সব দেখায়। Hacker এর কাজ সহজ হয়ে যায়। Production এ সবসময় false।

🟢 APP_ENV=production কেন?

Production mode এ Laravel অনেক optimization করে, error detail লুকায়, cache aggressively ব্যবহার করে। Performance বাড়ে।

🔑 APP_KEY গুরুত্ব

Session, cookie, encrypted data সব এই key দিয়ে encrypt হয়। Key চুরি হলে সব decrypt করা সম্ভব। কখনো public করবেন না।

📁 .env Location

Laravel root এ রাখুন, public_html এর বাইরে। কখনো web দিয়ে access করা যাবে না এমন জায়গায় রাখুন।

.gitignore এ .env যোগ আছে কিনা দেখুন

gitignore .gitignore 
# এগুলো .gitignore এ থাকতেই হবে:
.env
.env.backup
.env.production
/node_modules
/public/hot
/public/storage
/storage/*.key
/vendor

.htaccess দিয়ে .env Block করুন (Extra Security)

apache public/.htaccess (অতিরিক্ত সুরক্ষা) 
# .env সরাসরি web দিয়ে access হলে block করুন
<Files .env>
    Order allow,deny
    Deny from all
</Files>

# Laravel এর default .htaccess (public/ ফোল্ডারে)
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteRule ^ index.php [L]
</IfModule>
