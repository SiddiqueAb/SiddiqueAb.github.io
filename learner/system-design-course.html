⚖️ Trade-offs — কোনো Perfect System নেই
Consistency vs Availability
Strong consistency দিলে availability কমে। Facebook likes-এ eventual consistency ঠিক আছে, কিন্তু bank balance-এ strong consistency দরকার।
Latency vs Throughput
Batch processing-এ throughput বাড়ে কিন্তু latency বাড়ে। Real-time chat-এ latency কম চাই।
Read vs Write Performance
Read-heavy হলে caching ও replication। Write-heavy হলে write-optimized DB বা sharding।
Cost vs Performance
বেশি server মানে বেশি খরচ। Smart caching দিয়ে কম server-এ বেশি কাজ।

Horizontal vs Vertical Scaling

স্কেলিং কৌশল — বেশি ট্রাফিক কীভাবে সামলাবেন
💡 সহজ উপমা
Vertical Scaling (Scale Up): আপনার রান্নাঘরের চুলা বড় করলেন — ৪ বার্নার থেকে ৮ বার্নার। একটি শক্তিশালী মেশিন।

Horizontal Scaling (Scale Out): একটি বড় চুলার বদলে ৪টি আলাদা চুলা রাখলেন। অনেকগুলো সাধারণ মেশিন।

Load Balancer

Traffic বণ্টন — একটি server-এ চাপ না পড়ে সব server-এ সমানভাবে
💡 উপমা
টোল প্লাজা: ঢাকা-চট্টগ্রাম হাইওয়েতে একটিমাত্র টোল বুথ থাকলে জ্যাম হবে। তাই ১০টি বুথ রাখা হয় — প্রতিটিতে সমান গাড়ি যায়। Load Balancer এই কাজটিই করে।
🔄 Algorithm সমূহ
Algorithmকীভাবে কাজ করেকখন ব্যবহার
Round Robin1→2→3→1→2→3 পালাক্রমেসব server সমান শক্তিশালী
Weighted Round Robinশক্তিশালী server বেশি পায়আলাদা spec-এর server
Least Connectionsযার active connection কমLong-running requests
IP HashUser IP দেখে একই server-এSession sticky দরকার
RandomRandom server সিলেক্টSimple, fast

Caching Strategy

Database-এর উপর চাপ কমান, response দ্রুত করুন
💡 উপমা
রেস্তোরাঁর Menu: প্রতিবার chef-কে জিজ্ঞেস না করে menu board দেখলেই হয়। Cache হলো সেই menu board — frequently used data দ্রুত দেওয়ার জায়গা।
🗂️ Cache Strategy সমূহ
Cache-Aside (Lazy Loading)
App নিজে cache check করে। Miss হলে DB থেকে এনে cache-এ রাখে। Laravel-এ সবচেয়ে common।
Write-Through
Write হলে একসাথে DB ও Cache উভয়ে লেখা হয়। Cache সবসময় fresh। Write slow হয়।
Write-Behind (Write-Back)
Cache-এ লেখো, পরে asynchronously DB-তে। দ্রুত write, কিন্তু data loss risk আছে।
Read-Through
Cache নিজেই DB থেকে ডেটা আনে। App শুধু cache-এর সাথে কথা বলে।
⚠️ Cache Invalidation সমস্যা
"There are only two hard things in Computer Science: cache invalidation and naming things." — Phil Karlton
Cache কখন পুরনো হবে? কতক্ষণ রাখবেন? কীভাবে বাতিল করবেন? — এই প্রশ্নগুলোর সঠিক উত্তর জানা জরুরি। 
// ProductService.php — Cache-Aside Pattern
class ProductService
{
    public function getProduct(int $id): ?Product
    {
        $key = "product:{$id}";

        // ① Cache check করুন
        return Cache::remember($key, 3600, function() use ($id) {
            // ② Miss হলে DB থেকে আনুন
            return Product::with(['category', 'images'])->find($id);
        });
    }

    public function getPopularProducts(): Collection
    {
        return Cache::remember('products:popular', 1800, function() {
            return Product::withCount('orders')
                ->orderByDesc('orders_count')
                ->take(20)->get();
        });
    }

    public function updateProduct(int $id, array $data): Product
    {
        $product = Product::findOrFail($id);
        $product->update($data);

        // ③ Update হলে cache বাতিল করুন
        Cache::forget("product:{$id}");
        Cache::forget('products:popular');

        return $product;
    }
}

// Multi-Layer Caching: L1 (In-Memory) → L2 (Redis) → L3 (DB)
class MultiLayerCache
{
    private static array $memory = []; // L1: Request-এর মধ্যে

    public function get(string $key, callable $callback): mixed
    {
        // L1: PHP Memory
        if (isset(self::$memory[$key]))
            return self::$memory[$key];

        // L2: Redis
        $value = Cache::get($key);
        if ($value !== null) {
            self::$memory[$key] = $value;
            return $value;
        }

        // L3: Database
        $value = $callback();
        Cache::put($key, $value, 3600);
        self::$memory[$key] = $value;

        return $value;
    }
}
// Cache Stampede সমস্যা ও সমাধান
// সমস্যা: Cache expire হলে হাজারো request একসাথে DB-তে যায়!

// সমাধান ১: Atomic Lock
public function getExpensiveData(): array
{
    $key = 'expensive:report';

    if ($cached = Cache::get($key))
        return $cached;

    // Lock দিয়ে শুধু একটি request DB-তে যাবে
    return Cache::lock('lock:' . $key, 10)->block(5, function() use ($key) {
        // Lock পাওয়ার পর আবার check করুন
        if ($cached = Cache::get($key)) return $cached;

        $data = $this->fetchFromDatabase();
        Cache::put($key, $data, 3600);
        return $data;
    });
}

// সমাধান ২: Probabilistic Early Expiration
// TTL expire হওয়ার আগেই probabilistically refresh করুন
public function getWithEarlyRefresh(string $key, int $ttl, callable $cb): mixed
{
    [$value, $expiry] = Cache::get($key, [null, 0]);

    if ($value && ($expiry - time()) > -$ttl * log(rand()/PHP_INT_MAX))
        return $value;

    $value = $cb();
    Cache::put($key, [$value, time() + $ttl], $ttl);
    return $value;
}

Database Design

Normalization, Indexing, Query Optimization
📋 Normalization — Data Redundancy কমানো
1NF: প্রতিটি column-এ atomic value (array নয়)
2NF: Partial dependency নেই (composite PK সব column-এ depend করে)
3NF: Transitive dependency নেই
BCNF: প্রতিটি determinant candidate key

কিন্তু! Read-heavy system-এ denormalization করা হয় — JOIN কমিয়ে query দ্রুত করতে।
⚡ Indexing — Query দ্রুত করার মূল চাবিকাঠি
// Laravel Migration — Index strategy
Schema::create('orders', function (Blueprint $table) {
    $table->id();
    $table->foreignId('user_id')->constrained();
    $table->string('status', 20)->index(); // Single index
    $table->decimal('total', 10, 2);
    $table->timestamp('created_at')->index();

    // Composite Index — WHERE user_id = ? AND status = ?
    $table->index(['user_id', 'status'], 'idx_user_status');

    // Covering Index — SELECT তে যা লাগে সব index-এ
    $table->index(['user_id', 'created_at', 'total'], 'idx_user_orders');
});

Schema::table('products', function (Blueprint $table) {

Rate Limiting

API abuse বন্ধ করুন, সিস্টেম রক্ষা করুন
💡 উপমা
ATM মেশিন: দিনে সর্বোচ্চ ৫ বার টাকা তোলা যায়। এর বেশি চেষ্টা করলে blocked। Rate Limiting-ও তাই — নির্দিষ্ট সময়ে নির্দিষ্ট request-এর বেশি হলে block।
🔢 Algorithms
Token Bucket:
Bucket-এ token থাকে। Request আসলে token খরচ হয়। Time-এর সাথে token যোগ হয়। Burst allow করে।
Fixed Window:
প্রতি মিনিটে ১০০ request। মিনিট শেষে counter reset। সহজ কিন্তু window boundary-তে burst সম্ভব।
Sliding Window:
Rolling time window। যেকোনো ৬০ second-এ ১০০ request। Fixed window-এর সমস্যা নেই।
Leaky Bucket:
Queue-এ রাখো, নির্দিষ্ট rate-এ process করো। Consistent throughput নিশ্চিত করে।

// Custom Rate Limiter Middleware
class ApiRateLimiter
{
    public function handle(Request $request, Closure $next): Response
    {
        $key    = 'rate:' . $request->ip();
        $limit  = 100; // per minute
        $window = 60;  // seconds

        $current = Redis::incr($key);

        if ($current === 1)
            Redis::expire($key, $window); // প্রথমবার TTL set

        $remaining = max(0, $limit - $current);
        $retryAfter = Redis::ttl($key);

        if ($current > $limit) {
            return response()->json([
                'message'     => 'Too Many Requests',
                'retry_after' => $retryAfter,
            ], 429)->withHeaders([
                'X-RateLimit-Limit'     => $limit,
                'X-RateLimit-Remaining' => 0,
                'Retry-After'           => $retryAfter,
            ]);
        }

        return $next($request)->withHeaders([
            'X-RateLimit-Limit'     => $limit,
            'X-RateLimit-Remaining' => $remaining,
        ]);
    }
}
// Redis Sliding Window Rate Limiter
class SlidingWindowRateLimiter
{
    public function isAllowed(string $key, int $limit, int $window): bool
    {
        $now    = microtime(true) * 1000; // milliseconds
        $cutoff = $now - ($window * 1000);

        return Redis::eval(
            '
            local key = KEYS[1]
            local now = tonumber(ARGV[1])
            local cutoff = tonumber(ARGV[2])
            local limit = tonumber(ARGV[3])

            redis.call("ZREMRANGEBYSCORE", key, 0, cutoff)
            local count = redis.call("ZCARD", key)

            if count < limit then
                redis.call("ZADD", key, now, now)
                redis.call("EXPIRE", key, tonumber(ARGV[4]))
                return 1
            end
            return 0
            ',
            1, $key, $now, $cutoff, $limit, $window
        ) === 1;
    }
}

// Short Code Generation Strategies:

// Option 1: Hash-based (MD5/SHA)
// MD5(longUrl) → take first 7 chars
// ❌ Collision possible (though rare)
$code = substr(md5($longUrl), 0, 7);

// Option 2: Counter + Base62 (✅ Recommended)
// DB auto-increment ID → Base62 encode
// ID: 1000 → "QI8" (7 chars = 62^7 ≈ 3.5 trillion URLs)

class Base62Encoder
{
    private const CHARS = '0123456789abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ';

    public static function encode(int $n): string
    {
        $result = '';
        while ($n > 0) {
            $result = self::CHARS[$n % 62] . $result;
            $n = (int)($n / 62);
        }
        return str_pad($result, 7, '0', STR_PAD_LEFT);
    }

    public static function decode(string $s): int
    {
        $n = 0;
        foreach (str_split($s) as $c)
            $n = $n * 62 + strpos(self::CHARS, $c);
        return $n;
    }
}

Circuit Breaker

💡 উপমা
বৈদ্যুতিক Circuit Breaker: বাড়িতে শর্ট সার্কিট হলে main switch পড়ে যায় — পুরো বাড়ি নষ্ট হওয়া থেকে বাঁচে। Software-এও dependent service fail করলে Circuit Breaker "open" হয়ে সেই service-এ call বন্ধ করে দেয়।
States: Closed (সব ঠিক আছে) → Open (fail-এর হার বেশি, call বন্ধ) → Half-Open (পরীক্ষামূলক কিছু call করে দেখা)
💻 Laravel Circuit Breaker
// composer require ganesha/circuit-breaker বা custom
class CircuitBreaker
{
    public function call(string $service, callable $fn, callable $fallback): mixed
    {
        $key       = "circuit:{$service}";
        $failures  = (int)Redis::get("{$key}:failures");
        $state     = Redis::get("{$key}:state") ?: 'closed';

        if ($state === 'open') {
            Log::warning("Circuit open for {$service}, using fallback");
            return $fallback();
        }

        try {
            $result = $fn();
            Redis::del("{$key}:failures");
            Redis::set("{$key}:state", 'closed');
            return $result;
        } catch (\Exception $e) {
            $failures = Redis::incr("{$key}:failures");

            if ($failures >= 5) { // Threshold
                Redis::set("{$key}:state", 'open');
                Redis::expire("{$key}:state", 60); // 1 min reset
            }
            return $fallback();
        }
    }
}

{
    public function __construct(
        public float  $amount,
