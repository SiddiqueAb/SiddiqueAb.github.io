নেটওয়ার্কিং কী এবং কেন?

কম্পিউটার নেটওয়ার্কিং হলো দুই বা ততোধিক কম্পিউটার বা ডিভাইসকে একসাথে সংযুক্ত করার প্রক্রিয়া যাতে তারা ডেটা, রিসোর্স এবং সেবা শেয়ার করতে পারে। সহজ ভাষায়, যখন তুমি WhatsApp-এ বার্তা পাঠাও, YouTube-এ ভিডিও দেখো, বা ব্যাংক থেকে টাকা তোলো — এই সবই নেটওয়ার্কিং-এর কারণে সম্ভব।

💡
Engineer দৃষ্টিভঙ্গি নেটওয়ার্কিং মানে শুধু ক্যাবল সংযুক্ত করা নয়। এটি একটি সম্পূর্ণ ইকোসিস্টেম যেখানে হার্ডওয়্যার, সফটওয়্যার, প্রোটোকল এবং সিকিউরিটি একসাথে কাজ করে।

নেটওয়ার্কিং কেন শিখবে?

উচ্চ বেতন

Network Engineer-দের গড় বেতন অনেক বেশি। CCNA, CCNP সার্টিফিকেশন থাকলে আন্তর্জাতিক চাকরির সুযোগ।

🏗️

ইন্টারনেটের ভিত্তি

সব ধরনের IT কাজেই নেটওয়ার্কিং জ্ঞান লাগে — ডেভলপার থেকে শুরু করে সিকিউরিটি এক্সপার্ট পর্যন্ত।

সাইবার সিকিউরিটি

নেটওয়ার্কিং না জানলে সাইবার সিকিউরিটি বোঝা অসম্ভব। এটি সিকিউরিটির ভিত্তি।

☁️

ক্লাউড কম্পিউটিং

AWS, Azure, Google Cloud — সব ক্লাউড প্ল্যাটফর্মে নেটওয়ার্কিং জ্ঞান অপরিহার্য।

নেটওয়ার্কের ধরন (সংক্ষিপ্ত)

🏠

PAN (Personal Area Network)

ব্যক্তিগত ডিভাইসের নেটওয়ার্ক। উদাহরণ: Bluetooth হেডফোন, স্মার্টওয়াচ। পরিসর: ~10 মিটার।

🏢

LAN (Local Area Network)

একই ভবন বা ক্যাম্পাসের নেটওয়ার্ক। উদাহরণ: অফিস নেটওয়ার্ক, স্কুল ল্যাব। পরিসর: ~1 কিমি।

🏙️

MAN (Metropolitan Area Network)

শহর বা মেট্রোপলিটন এলাকার নেটওয়ার্ক। উদাহরণ: কেবল TV নেটওয়ার্ক। পরিসর: ~50 কিমি।

🌍

WAN (Wide Area Network)

দেশ বা মহাদেশ জুড়ে বিস্তৃত নেটওয়ার্ক। উদাহরণ: ইন্টারনেট নিজেই। পরিসর: সীমাহীন।

🔢
সবচেয়ে গুরুত্বপূর্ণ কনসেপ্ট

OSI মডেল — ৭ স্তর

OSI (Open Systems Interconnection) মডেল হলো নেটওয়ার্কিং-এর সবচেয়ে গুরুত্বপূর্ণ কনসেপ্ট। এটি একটি ধারণাগত ফ্রেমওয়ার্ক যা বর্ণনা করে কীভাবে বিভিন্ন নেটওয়ার্ক সিস্টেম একে অপরের সাথে যোগাযোগ করে। ISO ১৯৮৪ সালে এটি তৈরি করে।

⚡
মনে রাখার ট্রিক! উপর থেকে নিচে: "All People Seem To Need Data Processing" → Application, Presentation, Session, Transport, Network, Data Link, Physical
📊 OSI মডেল — ৭টি লেয়ার
7
Application Layer
HTTP, FTP, SMTP, DNS — ব্যবহারকারীর সাথে সরাসরি ইন্টারঅ্যাকশন
6
Presentation Layer
SSL/TLS, এনক্রিপশন, ডেটা ফরম্যাটিং (JPEG, MP4, ASCII)
5
Session Layer
সেশন তৈরি, রক্ষণাবেক্ষণ ও সমাপ্তি (NetBIOS, RPC)
4
Transport Layer
TCP/UDP — নির্ভরযোগ্য ডেটা ট্রান্সফার, পোর্ট নম্বর
3
Network Layer
IP অ্যাড্রেসিং, রাউটিং — রাউটার এখানে কাজ করে
2
Data Link Layer
MAC অ্যাড্রেস, ফ্রেমিং — সুইচ এখানে কাজ করে
1
Physical Layer
বিট (0/1), ক্যাবল, হাব — ভৌত সংযোগ

প্রতিটি লেয়ারের বিস্তারিত

Layer 7 — Application Layer (অ্যাপ্লিকেশন স্তর)

এটি সরাসরি ব্যবহারকারী বা অ্যাপ্লিকেশনের সাথে কাজ করে। যখন তুমি ব্রাউজারে google.com টাইপ করো, এই লেয়ার HTTP রিকোয়েস্ট তৈরি করে। প্রোটোকল: HTTP, HTTPS, FTP, SMTP, POP3, IMAP, DNS, Telnet, SSH।

Layer 4 — Transport Layer (ট্রান্সপোর্ট স্তর) — সবচেয়ে গুরুত্বপূর্ণ!

TCP (Transmission Control Protocol)

  • নির্ভরযোগ্য ডেলিভারি নিশ্চিত করে
  • 3-Way Handshake ব্যবহার করে
  • ডেটা হারিয়ে গেলে পুনরায় পাঠায়
  • ধীর কিন্তু নিরাপদ
  • ব্যবহার: HTTP, Email, File Transfer

UDP (User Datagram Protocol)

  • কোনো গ্যারান্টি ছাড়া ডেটা পাঠায়
  • কোনো Handshake নেই
  • ডেটা হারালে পাত্তা দেয় না
  • দ্রুত কিন্তু অনির্ভরযোগ্য
  • ব্যবহার: Video Call, Gaming, DNS

Layer 3 — Network Layer (নেটওয়ার্ক স্তর)

এই লেয়ার IP অ্যাড্রেস ব্যবহার করে প্যাকেটকে উৎস থেকে গন্তব্যে পৌঁছে দেয়। রাউটার এই লেয়ারে কাজ করে। প্রোটোকল: IP, ICMP, OSPF, BGP, RIP।

Layer 2 — Data Link Layer (ডেটা লিংক স্তর)

MAC অ্যাড্রেস ব্যবহার করে একই নেটওয়ার্কে ডেটা ট্রান্সফার করে। সুইচ এবং ব্রিজ এই লেয়ারে কাজ করে। ডেটাকে "ফ্রেম"-এ ভাগ করে।

Layer 1 — Physical Layer (ফিজিক্যাল স্তর)

বাস্তব ভৌত সংযোগ — ক্যাবল, ফাইবার অপটিক, রেডিও ওয়েভ। ডেটা বিট (0 এবং 1) আকারে পাঠানো হয় বৈদ্যুতিক সংকেত, আলো বা রেডিও তরঙ্গ হিসেবে।

🔗
বাস্তব ব্যবহারিক মডেল

TCP/IP মডেল — ইন্টারনেটের ভিত্তি

TCP/IP মডেল হলো বাস্তবে ব্যবহৃত মডেল যার উপর ইন্টারনেট নির্মিত। OSI মডেল যেখানে ৭ লেয়ার, TCP/IP মডেলে মাত্র ৪ লেয়ার।

TCP/IP লেয়ার OSI সমতুল্য প্রোটোকল কাজ
Application Layer 5, 6, 7 HTTP DNS SMTP ব্যবহারকারী অ্যাপ্লিকেশন
Transport Layer 4 TCP UDP end-to-end যোগাযোগ
Internet Layer 3 IP ICMP ARP প্যাকেট রাউটিং
Network Access Layer 1, 2 Ethernet Wi-Fi ভৌত মাধ্যমে ডেটা

ডেটা কীভাবে পাঠানো হয়? (Encapsulation)

যখন তুমি কাউকে ইমেইল পাঠাও, ডেটা বিভিন্ন লেয়ারে প্যাক হয়:

7

Application → "ডেটা" তৈরি

তোমার ইমেইলের কন্টেন্ট। এটি সরাসরি অ্যাপ্লিকেশন ডেটা।

4

Transport → "সেগমেন্ট" তৈরি

TCP হেডার যোগ করা হয় — পোর্ট নম্বর, সিকোয়েন্স নম্বর।

3

Network → "প্যাকেট" তৈরি

IP হেডার যোগ — উৎস ও গন্তব্য IP অ্যাড্রেস।

2

Data Link → "ফ্রেম" তৈরি

MAC অ্যাড্রেস হেডার ও ট্রেইলার যোগ।

1

Physical → "বিট" পাঠানো

0 এবং 1 হিসেবে বৈদ্যুতিক সংকেত বা আলো হিসেবে পাঠানো হয়।

🎯
গন্তব্যে পৌঁছানোর পর? গন্তব্য কম্পিউটার উল্টো প্রক্রিয়ায় (De-encapsulation) প্রতিটি লেয়ারের হেডার সরিয়ে মূল ডেটা বের করে।
📍
নেটওয়ার্ক পরিচয়

IP অ্যাড্রেস — নেটওয়ার্কের ঠিকানা

IP অ্যাড্রেস হলো নেটওয়ার্কে প্রতিটি ডিভাইসের অনন্য পরিচয়। ঠিক যেমন তোমার বাড়ির ডাক ঠিকানা, IP অ্যাড্রেস নেটওয়ার্কে ডিভাইস খুঁজে পেতে ব্যবহৃত হয়।

IPv4 — ৩২-বিট অ্যাড্রেস

সবচেয়ে বেশি ব্যবহৃত ফরম্যাট। ৪টি অকটেটে বিভক্ত, প্রতিটি ০-২৫৫ মানের।

IPv4 উদাহরণ
192.168.1.100
│   │   │  └── Host অংশ (কোন ডিভাইস)
│   │   └───── সাবনেট
│   └───────── নেটওয়ার্ক অংশ
└───────────── নেটওয়ার্ক অংশ

বাইনারিতে: 11000000.10101000.00000001.01100100
মোট বিট:   32 বিট → মোট ~4.3 বিলিয়ন অ্যাড্রেস

IP ক্লাস সমূহ

Class A
1.0.0.0 — 126.255.255.255
বড় সংস্থা, সরকার। ১৬ মিলিয়ন হোস্ট সম্ভব।
Class B
128.0.0.0 — 191.255.255.255
মাঝারি সংস্থা। ৬৫,০০০ হোস্ট সম্ভব।
Class C
192.0.0.0 — 223.255.255.255
ছোট নেটওয়ার্ক। ২৫৪টি হোস্ট সম্ভব।
Class D
224.0.0.0 — 239.255.255.255
Multicast গ্রুপের জন্য সংরক্ষিত।
Class E
240.0.0.0 — 255.255.255.255
গবেষণার জন্য সংরক্ষিত।

Private IP vs Public IP

🏠 Private IP (ব্যক্তিগত)

  • 10.0.0.0 – 10.255.255.255
  • 172.16.0.0 – 172.31.255.255
  • 192.168.0.0 – 192.168.255.255
  • ইন্টারনেটে সরাসরি যোগাযোগ করতে পারে না
  • বিনামূল্যে, যেকেউ ব্যবহার করতে পারে

🌍 Public IP (সার্বজনীন)

  • ISP দ্বারা বরাদ্দ করা হয়
  • বিশ্বে একক ও অনন্য
  • ইন্টারনেটে সরাসরি যোগাযোগযোগ্য
  • Static বা Dynamic হতে পারে
  • NAT ব্যবহার করে Private → Public

IPv6 — ভবিষ্যতের অ্যাড্রেস

IPv4-এর ৪.৩ বিলিয়ন অ্যাড্রেস ফুরিয়ে আসছে। তাই IPv6 তৈরি হয়েছে যার ১২৮ বিট থেকে ৩৪০ ট্রিলিয়ন ট্রিলিয়ন ট্রিলিয়ন অ্যাড্রেস সম্ভব।

# IPv4
192.168.1.1

# IPv6
2001:0db8:85a3:0000:0000:8a2e:0370:7334
সংক্ষিপ্ত: 2001:db8:85a3::8a2e:370:7334

নেটওয়ার্ক বিভাজন

সাবনেটিং — নেটওয়ার্ক ভাগ করা

সাবনেটিং হলো একটি বড় নেটওয়ার্ককে ছোট ছোট অংশে (সাবনেট) ভাগ করার প্রক্রিয়া। এতে নেটওয়ার্ক পরিচালনা সহজ হয়, নিরাপত্তা বাড়ে এবং ট্রাফিক কমে।

📐
Subnet Mask কী? Subnet mask নির্ধারণ করে কোন অংশ Network ID এবং কোন অংশ Host ID। উদাহরণ: 255.255.255.0 মানে প্রথম ৩ অকটেট নেটওয়ার্ক, শেষ অকটেট হোস্ট।

CIDR (Classless Inter-Domain Routing) নোটেশন

192.168.1.0/24
                └── /24 মানে প্রথম 24 বিট Network অংশ

# সাধারণ CIDR এবং তাদের হোস্ট সংখ্যা:
/30 → Subnet Mask: 255.255.255.252 → 2টি হোস্ট
/29 → Subnet Mask: 255.255.255.248 → 6টি হোস্ট
/28 → Subnet Mask: 255.255.255.240 → 14টি হোস্ট
/27 → Subnet Mask: 255.255.255.224 → 30টি হোস্ট
/26 → Subnet Mask: 255.255.255.192 → 62টি হোস্ট
/25 → Subnet Mask: 255.255.255.128 → 126টি হোস্ট
/24 → Subnet Mask: 255.255.255.0   → 254টি হোস্ট
/16 → Subnet Mask: 255.255.0.0     → 65,534টি হোস্ট
/8  → Subnet Mask: 255.0.0.0       → 16,777,214টি হোস্ট

ধরো তোমার নেটওয়ার্ক 192.168.10.0/24 এবং তোমার ৪টি ডিপার্টমেন্টের জন্য আলাদা সাবনেট চাই:

# /26 দিয়ে ভাগ করলে ৪টি সাবনেট পাবো, প্রতিটিতে ৬২ হোস্ট

বিক্রয় বিভাগ:   192.168.10.0/26   (192.168.10.1 - 192.168.10.62)

IT বিভাগ:        192.168.10.128/26 (192.168.10.129 - 192.168.10.190)
Management:      192.168.10.192/26 (192.168.10.193 - 192.168.10.254)

# প্রতিটি সাবনেটের শেষ IP = Broadcast Address (ব্যবহার করা যায় না)

নেটওয়ার্ক হার্ডওয়্যার

নেটওয়ার্ক ডিভাইস সমূহ

📡

Hub (হাব)

OSI Layer 1 — সবচেয়ে সরল ডিভাইস। একটি পোর্টে ডেটা পেলে সব পোর্টে পাঠায়। অদক্ষ ও এখন প্রায় অব্যবহৃত। Half-duplex।

🔀

Switch (সুইচ)

OSI Layer 2 — MAC অ্যাড্রেস ব্যবহার করে শুধু সঠিক পোর্টে ডেটা পাঠায়। দক্ষ, Full-duplex। অফিস নেটওয়ার্কের মূল ডিভাইস।

🛣️

Router (রাউটার)

OSI Layer 3 — IP অ্যাড্রেস ব্যবহার করে বিভিন্ন নেটওয়ার্কের মধ্যে ডেটা পাঠায়। ইন্টারনেট সংযোগের জন্য অপরিহার্য।

📶

Access Point (AP)

ওয়্যারলেস সংযোগ — তারযুক্ত নেটওয়ার্কে ওয়্যারলেস অ্যাক্সেস যোগ করে। Wi-Fi নেটওয়ার্কের কেন্দ্র।

🔥

Firewall (ফায়ারওয়াল)

নিরাপত্তা ডিভাইস — নেটওয়ার্ক ট্র্যাফিক পর্যবেক্ষণ ও নিয়ন্ত্রণ করে। অননুমোদিত অ্যাক্সেস ব্লক করে।

⚖️

Load Balancer

ট্রাফিক বিতরণ — একাধিক সার্ভারে ট্রাফিক সমানভাবে বিতরণ করে। উচ্চ প্রাপ্যতা ও পারফরম্যান্স নিশ্চিত করে।

🔌

Modem (মডেম)

সংকেত রূপান্তরকারী — ডিজিটাল সংকেতকে অ্যানালগে (এবং উল্টো) রূপান্তর করে ISP সংযোগের জন্য।

🔄

Proxy Server

মধ্যস্থতাকারী — ক্লায়েন্ট ও সার্ভারের মাঝে থেকে ট্রাফিক ফরওয়ার্ড করে। ক্যাশিং ও ফিল্টারিং করতে পারে।

ক্যাবল গতি দূরত্ব ব্যবহার
Cat5e 1 Gbps 100m সাধারণ অফিস নেটওয়ার্ক
Cat6 10 Gbps 55m ডেটা সেন্টার, আধুনিক অফিস

Coaxial 10 Mbps 500m পুরনো নেটওয়ার্ক, কেবল TV

ইন্টারনেটের ফোনবুক

DNS — Domain Name System

DNS হলো ইন্টারনেটের "ফোনবুক"। তুমি যখন google.com টাইপ করো, DNS সেটাকে IP অ্যাড্রেসে (যেমন 142.250.185.46) পরিণত করে।

DNS রেজোলিউশন কীভাবে কাজ করে?

1

ব্রাউজার ক্যাশ চেক

প্রথমে ব্রাউজার দেখে এই ডোমেনের IP আগে সেভ আছে কিনা।

2

OS ক্যাশ ও Hosts ফাইল

অপারেটিং সিস্টেমের local DNS ক্যাশ ও /etc/hosts ফাইল চেক।

3

Recursive DNS Resolver (ISP)

তোমার ISP-এর DNS সার্ভারকে জিজ্ঞেস করা হয়।

4

Root Name Server

Root server জানায় কোন TLD server (.com, .org, .bd) এর সাথে কথা বলতে হবে।

5

Authoritative DNS Server

ডোমেনের নিজস্ব DNS সার্ভার চূড়ান্ত IP অ্যাড্রেস দেয়।

গুরুত্বপূর্ণ DNS রেকর্ড

রেকর্ড টাইপ কাজ উদাহরণ
A ডোমেন → IPv4 অ্যাড্রেস google.com → 142.250.185.46
AAAA ডোমেন → IPv6 অ্যাড্রেস google.com → 2607:f8b0::...
CNAME একটি ডোমেন → অন্য ডোমেন www.example.com → example.com
MX ইমেইল সার্ভার gmail.com-এর mail server
TXT যেকোনো টেক্সট তথ্য SPF, DKIM রেকর্ড
NS Name Server রেকর্ড কোন DNS সার্ভার authority
PTR Reverse DNS (IP → ডোমেন) 46.185.250.142 → google.com
DNS টুলস
# DNS রেজোলিউশন চেক করো
nslookup google.com
dig google.com
dig google.com MX          # MX রেকর্ড দেখো

# Reverse DNS lookup
dig -x 8.8.8.8

# নির্দিষ্ট DNS সার্ভার ব্যবহার করে lookup
dig @8.8.8.8 google.com     # Google DNS দিয়ে
dig @1.1.1.1 google.com     # Cloudflare DNS দিয়ে

DHCP — স্বয়ংক্রিয় IP বরাদ্দ

DHCP (Dynamic Host Configuration Protocol) স্বয়ংক্রিয়ভাবে নেটওয়ার্ক ডিভাইসগুলোকে IP অ্যাড্রেস, Subnet Mask, Default Gateway এবং DNS Server তথ্য প্রদান করে।

D

Discover — ক্লায়েন্ট ব্রডকাস্ট করে

নতুন ডিভাইস নেটওয়ার্কে যুক্ত হয়ে ব্রডকাস্ট পাঠায়: "কেউ আছো যে আমাকে IP দিতে পারো?"

O

Offer — DHCP সার্ভার সাড়া দেয়

সার্ভার একটি IP অ্যাড্রেস অফার করে: "আমি তোমাকে 192.168.1.50 দিতে পারি।"

R

ক্লায়েন্ট বলে: "আমি 192.168.1.50 চাই।" (broadcast, যাতে অন্য DHCP সার্ভাররা জানে)

A

Acknowledge — সার্ভার নিশ্চিত করে

সার্ভার নিশ্চিত করে: "192.168.1.50 তোমার — Lease Time: 24 ঘন্টা।"

পথ নির্ধারণ

রাউটিং — প্যাকেটের পথ খোঁজা

রাউটিং হলো নেটওয়ার্কে প্যাকেট পাঠানোর সঠিক পথ নির্ধারণ করার প্রক্রিয়া। রাউটার Routing Table ব্যবহার করে সেরা পথ বেছে নেয়।

রাউটিং-এর ধরন

📋

Static Routing (স্ট্যাটিক)

Administrator ম্যানুয়ালি রুট কনফিগার করেন। ছোট নেটওয়ার্কে ভালো, পরিবর্তন করা কঠিন।

🤖

Dynamic Routing (ডায়নামিক)

রাউটার স্বয়ংক্রিয়ভাবে রুট শেখে ও আপডেট করে। বড় নেটওয়ার্কে অপরিহার্য।

🏠

Default Route

অন্য কোনো রুট না জানলে এই পথ ব্যবহার করে। 0.0.0.0/0 দিয়ে প্রকাশ।

রাউটিং প্রোটোকল

প্রোটোকল ধরন অ্যালগোরিদম ব্যবহার
RIP Distance Vector Hop Count ছোট নেটওয়ার্ক (সর্বোচ্চ 15 hop)
OSPF Link State Dijkstra বড় এন্টারপ্রাইজ নেটওয়ার্ক
EIGRP Hybrid DUAL Cisco নেটওয়ার্ক
BGP Path Vector Best Path ইন্টারনেট (ISP-এর মধ্যে)
Cisco Router কনফিগারেশন
# Static Route যোগ করা
Router(config)# ip route 192.168.2.0 255.255.255.0 192.168.1.1

# Default Route
Router(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1

# OSPF কনফিগারেশন
Router(config)# router ospf 1
Router(config-router)# network 192.168.1.0 0.0.0.255 area 0

# Routing Table দেখো
Router# show ip route

নিরাপত্তা ব্যবস্থা

নেটওয়ার্ক সিকিউরিটি

নেটওয়ার্ক সিকিউরিটি হলো নেটওয়ার্ক ও ডেটাকে অননুমোদিত অ্যাক্সেস, ক্ষতি বা চুরি থেকে রক্ষা করার প্রক্রিয়া। CIA Triad হলো সিকিউরিটির তিন মূল স্তম্ভ।

🔒

Confidentiality (গোপনীয়তা)

শুধু অনুমোদিত ব্যক্তিই ডেটা দেখতে পাবে। এনক্রিপশন, অ্যাক্সেস কন্ট্রোল ব্যবহার করে নিশ্চিত করা হয়।

✅

Integrity (অখণ্ডতা)

ডেটা পাঠানোর সময় পরিবর্তিত হয়নি। Hash (SHA-256, MD5) দিয়ে যাচাই করা হয়।

⚡

Availability (প্রাপ্যতা)

অনুমোদিত ব্যবহারকারীরা সবসময় সেবা পাবে। DDoS প্রতিরোধ, backup, redundancy।

নেটওয়ার্ক এনক্রিপশন

Symmetric Encryption

  • একই চাবি এনক্রিপ্ট ও ডিক্রিপ্ট করে
  • দ্রুত কিন্তু চাবি শেয়ার করা ঝুঁকিপূর্ণ
  • AES-256 (সবচেয়ে নিরাপদ)
  • 3DES, Blowfish, ChaCha20

Asymmetric Encryption

  • Public key + Private key জোড়া
  • Public key দিয়ে এনক্রিপ্ট, Private দিয়ে ডিক্রিপ্ট
  • RSA-2048/4096, ECC
  • SSL/TLS, SSH এ ব্যবহৃত

SSL/TLS কীভাবে কাজ করে?

HTTPS-এ SSL/TLS ব্যবহার করে নিরাপদ সংযোগ স্থাপিত হয়:

1

Client Hello

ক্লায়েন্ট সমর্থিত TLS ভার্সন ও Cipher Suite-এর তালিকা পাঠায়।

2

Server Hello + Certificate

সার্ভার তার SSL Certificate (Public Key সহ) পাঠায়।

3

Certificate Verification

ক্লায়েন্ট CA (Certificate Authority) দিয়ে Certificate যাচাই করে।

4

Session Key Exchange

Asymmetric encryption ব্যবহার করে একটি Symmetric Session Key তৈরি হয়।

5

Secure Communication

এখন থেকে সব যোগাযোগ Session Key দিয়ে AES-256 এনক্রিপ্টেড।

নেটওয়ার্ক সিকিউরিটি Best Practices

  • সব সংযোগে HTTPS/TLS ব্যবহার করো
  • নেটওয়ার্ককে VLAN দিয়ে সেগমেন্ট করো
  • সব ডিভাইসের ডিফল্ট পাসওয়ার্ড পরিবর্তন করো
  • অপ্রয়োজনীয় পোর্ট বন্ধ রাখো (port hardening)
  • নিয়মিত নেটওয়ার্ক লগ পর্যালোচনা করো
  • IDS/IPS ব্যবহার করো (Intrusion Detection/Prevention)
  • নেটওয়ার্ক Access Control List (ACL) ব্যবহার করো
  • Two-Factor Authentication (2FA) চালু রাখো
  • নিয়মিত সফটওয়্যার আপডেট ও প্যাচ প্রয়োগ করো
  • নেটওয়ার্ক স্ক্যান ও Vulnerability Assessment করো

নেটওয়ার্ক রক্ষাকবচ

ফায়ারওয়াল — প্রথম প্রতিরক্ষা

ফায়ারওয়াল নেটওয়ার্ক ট্র্যাফিক পর্যবেক্ষণ করে এবং নির্দিষ্ট নিয়ম (Rules) অনুযায়ী ট্র্যাফিক Allow বা Block করে।

ফায়ারওয়ালের ধরন

📦

Packet Filtering

IP অ্যাড্রেস ও পোর্ট দেখে সিদ্ধান্ত নেয়। সহজ ও দ্রুত কিন্তু সীমিত নিরাপত্তা।

🔍

Stateful Inspection

সংযোগের অবস্থা (state) ট্র্যাক করে। অনেক বেশি নিরাপদ, আধুনিক ফায়ারওয়ালে ব্যবহৃত।

🔬

Application Layer (WAF)

HTTP, DNS, FTP লেভেলে ট্র্যাফিক পরীক্ষা করে। SQL Injection, XSS ব্লক করে।

🤖

Next-Gen Firewall (NGFW)

DPI, Application Awareness, IPS, User Identity — সব কিছু একসাথে।

ফায়ারওয়াল Rules উদাহরণ (iptables)

Linux iptables
# সব incoming ব্লক করো (ডিফল্ট deny)
iptables -P INPUT DROP

# HTTP ও HTTPS অনুমোদন করো
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# SSH অনুমোদন করো (নির্দিষ্ট IP থেকে)
iptables -A INPUT -p tcp --dport 22 -s 203.0.113.5 -j ACCEPT

# Established সংযোগ অনুমোদন করো
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# loopback অনুমোদন করো
iptables -A INPUT -i lo -j ACCEPT

# বর্তমান rules দেখো
iptables -L -v -n

VPN — Virtual Private Network

VPN হলো ইন্টারনেটের মাধ্যমে একটি নিরাপদ, এনক্রিপ্টেড "টানেল" তৈরির প্রযুক্তি। এটি তোমার ডেটা সুরক্ষিত রাখে এবং ভৌগোলিক বিধিনিষেধ এড়াতে সাহায্য করে।

VPN-এর ধরন

🏢 Site-to-Site VPN

  • দুটি অফিস নেটওয়ার্ক সংযুক্ত করে
  • Permanent সংযোগ
  • IPsec বেশি ব্যবহৃত হয়
  • কর্পোরেট WAN-এ সাধারণ

👤 Remote Access VPN

  • ব্যক্তিগত ব্যবহারকারী অফিসে সংযুক্ত হয়
  • On-demand সংযোগ
  • SSL/TLS বা IPsec
  • Work-from-home-এ সাধারণ

জনপ্রিয় VPN প্রোটোকল

সাধারণ নেটওয়ার্ক আক্রমণ

💥

DDoS (Distributed Denial of Service)

হাজার হাজার ডিভাইস দিয়ে একসাথে সার্ভারে আক্রমণ করে সার্ভিস বন্ধ করা। প্রতিরোধ: CDN, Rate Limiting, DDoS Protection।

🎭

Man-in-the-Middle (MITM)

দুই পক্ষের যোগাযোগের মাঝে ঢুকে ডেটা পড়া বা পরিবর্তন করা। প্রতিরোধ: TLS, Certificate Pinning।

🎣

DNS Poisoning/Spoofing

DNS ক্যাশে ভুল তথ্য ঢোকানো যাতে ব্যবহারকারী ভুল সাইটে যায়। প্রতিরোধ: DNSSEC।

🔓

ARP Spoofing

নেটওয়ার্কে ভুল MAC অ্যাড্রেস জানিয়ে ট্র্যাফিক ডাইভার্ট করা। প্রতিরোধ: Dynamic ARP Inspection।

🔎

Port Scanning

নেটওয়ার্কের খোলা পোর্ট খোঁজা আক্রমণের আগে। প্রতিরোধ: ফায়ারওয়াল, অপ্রয়োজনীয় পোর্ট বন্ধ।

📡

Packet Sniffing

নেটওয়ার্কে অনুমতি ছাড়া প্যাকেট ক্যাপচার করা। প্রতিরোধ: এনক্রিপশন, VPN।

Wi-Fi সিকিউরিটি প্রোটোকল

🔴

WEP (1997) — ব্যবহার করো না!

মিনিটের মধ্যে ক্র্যাক করা সম্ভব। সম্পূর্ণ অনিরাপদ, কখনোই ব্যবহার করা উচিত নয়।

🟡

WPA (2003) — দুর্বল

WEP-এর উন্নতি কিন্তু এখনো দুর্বলতা আছে। বর্তমানে ব্যবহার এড়িয়ে চলো।

🟢

WPA2 (2004) — গ্রহণযোগ্য

AES-CCMP এনক্রিপশন। বাড়ি ও ছোট অফিসের জন্য এখনো যথেষ্ট নিরাপদ।

🔵

WPA3 (2018) — সর্বোত্তম

SAE Handshake, ব্রুটফোর্স প্রতিরোধী। যেখানে সম্ভব ব্যবহার করো।

2.4 GHz vs 5 GHz

📡 2.4 GHz

  • দূরপাল্লার সংযোগ (বেশি দূরত্ব)
  • দেয়াল ভেদ করতে পারে ভালো
  • ধীর গতি (বেশি ডিভাইস ব্যবহার করে)
  • বেশি interference (মাইক্রোওয়েভ, Bluetooth)
  • পুরনো ডিভাইসে সাপোর্ট

📡 5 GHz

  • বেশি গতি (কম ভিড়)
  • কম interference
  • কম দূরত্ব, দেয়াল কম ভেদ করে
  • বেশি চ্যানেল উপলব্ধ
  • কাছের ডিভাইসের জন্য আদর্শ

আধুনিক নেটওয়ার্কিং

ক্লাউড নেটওয়ার্কিং — ভবিষ্যতের প্রযুক্তি

ক্লাউড নেটওয়ার্কিং হলো ইন্টারনেটের মাধ্যমে রিমোট সার্ভার ও নেটওয়ার্ক রিসোর্স ব্যবহার করা। আর নিজের সার্ভার রুম দরকার নেই।

ক্লাউড সার্ভিস মডেল

🏗️

IaaS (Infrastructure as a Service)

ভার্চুয়াল সার্ভার, স্টোরেজ, নেটওয়ার্ক। তুমি নিজেই OS ও সফটওয়্যার ইনস্টল করো। উদাহরণ: AWS EC2, Azure VM।

🛠️

PaaS (Platform as a Service)

ডেভেলপমেন্ট প্ল্যাটফর্ম রেডি। শুধু অ্যাপ্লিকেশন কোড লেখো। উদাহরণ: Heroku, Google App Engine।

☁️

SaaS (Software as a Service)

সম্পূর্ণ সফটওয়্যার ব্রাউজারে। ইনস্টল করতে হয় না। উদাহরণ: Gmail, Slack, Salesforce।

গুরুত্বপূর্ণ ক্লাউড নেটওয়ার্কিং কনসেপ্ট

🕸️

VPC (Virtual Private Cloud)

ক্লাউডে তোমার নিজস্ব আলাদা নেটওয়ার্ক। AWS VPC, Azure VNet এই সুবিধা দেয়।

⚖️

Load Balancing

ট্র্যাফিক একাধিক সার্ভারে ভাগ করা। High Availability ও Scalability নিশ্চিত করে।

🌐

CDN (Content Delivery Network)

বিশ্বের বিভিন্ন জায়গায় কন্টেন্ট ক্যাশ করে দ্রুত ডেলিভারি। Cloudflare, Akamai।

📏

Auto Scaling

ট্র্যাফিক বাড়লে স্বয়ংক্রিয়ভাবে সার্ভার বাড়ে, কমলে কমে। খরচ সাশ্রয়ী।

SDN (Software-Defined Networking)

SDN হলো নেটওয়ার্ক নিয়ন্ত্রণকে হার্ডওয়্যার থেকে আলাদা করে সফটওয়্যারে নিয়ে আসা। Control Plane এবং Data Plane আলাদা থাকে — ফলে নেটওয়ার্ক প্রোগ্রামেবল হয়।

🔧
নেটওয়ার্ক ডায়াগনস্টিক

নেটওয়ার্ক টুলস

নেটওয়ার্ক ইঞ্জিনিয়ার হিসেবে এই টুলগুলো তোমার দৈনন্দিন সাথী হবে।

Command Line টুলস

অপরিহার্য নেটওয়ার্ক কমান্ড
━━━━━━ সংযোগ পরীক্ষা ━━━━━━
ping google.com                    # সংযোগ আছে কিনা দেখো
ping -c 5 8.8.8.8                  # ৫টি প্যাকেট পাঠাও

━━━━━━ পথ নির্ধারণ ━━━━━━
traceroute google.com              # Linux/Mac
tracert google.com                 # Windows

━━━━━━ DNS লুকআপ ━━━━━━
nslookup google.com
dig google.com
dig +short google.com

━━━━━━ নেটওয়ার্ক ইন্টারফেস ━━━━━━
ifconfig                           # Linux/Mac (পুরনো)
ip addr show                       # Linux (আধুনিক)
ipconfig /all                      # Windows

━━━━━━ পোর্ট ও সংযোগ ━━━━━━
netstat -an                        # সব সংযোগ দেখো
ss -tulpn                          # Linux: listening ports
netstat -ano | findstr :80         # Windows: পোর্ট 80 চেক

━━━━━━ ARP টেবিল ━━━━━━
arp -a                             # ARP ক্যাশ দেখো

━━━━━━ Routing Table ━━━━━━
route -n                           # Linux
route print                        # Windows
ip route show                      # Linux (আধুনিক)

━━━━━━ পোর্ট স্ক্যান ━━━━━━
nmap -sV 192.168.1.1               # নিজের নেটওয়ার্ক স্ক্যান
nmap -p 80,443 192.168.1.0/24     # নির্দিষ্ট পোর্ট

━━━━━━ ব্যান্ডউইথ পরীক্ষা ━━━━━━
iperf3 -s                          # সার্ভার মোড
iperf3 -c 192.168.1.100            # ক্লায়েন্ট মোড

━━━━━━ প্যাকেট ক্যাপচার ━━━━━━
tcpdump -i eth0 port 80            # HTTP ট্র্যাফিক দেখো
tcpdump -i eth0 host 8.8.8.8      # নির্দিষ্ট হোস্ট

সবচেয়ে জনপ্রিয় প্যাকেট অ্যানালাইজার। নেটওয়ার্ক ট্র্যাফিক ক্যাপচার ও বিশ্লেষণ। সম্পূর্ণ বিনামূল্যে।

🗺️

Nmap / Zenmap

নেটওয়ার্ক স্ক্যানার। পোর্ট, সার্ভিস, OS সনাক্ত করে। Zenmap হলো Nmap-এর GUI।

📊

PRTG / Zabbix

নেটওয়ার্ক মনিটরিং টুল। রিয়েল-টাইম পারফরম্যান্স, আপটাইম ট্র্যাক করে।

🔒

Nessus / OpenVAS

Vulnerability Scanner। নেটওয়ার্কের দুর্বলতা খুঁজে বের করে।

নেটওয়ার্ক ইঞ্জিনিয়ার হওয়ার রোডম্যাপ

1

মূল ভিত্তি (Beginner)

OSI মডেল, TCP/IP, IP Addressing, Subnetting, DHCP, DNS — এগুলো ভালোভাবে শিখো।

2

সার্টিফিকেশন: CompTIA Network+

নেটওয়ার্কিং-এর সার্বজনীন সার্টিফিকেশন। বেসিক থেকে ইন্টারমিডিয়েট স্তর।

3

সার্টিফিকেশন: Cisco CCNA

সবচেয়ে মূল্যবান এন্ট্রি-লেভেল সার্টিফিকেশন। Cisco ডিভাইস কনফিগারেশন শেখো।

4

Practical Lab Practice

Cisco Packet Tracer বা GNS3 দিয়ে নেটওয়ার্ক সিমুলেশন করো। হাতে-কলমে অভিজ্ঞতা অর্জন।

5

উন্নত স্তর: CCNP / Security+

নেটওয়ার্ক সিকিউরিটি, ক্লাউড নেটওয়ার্কিং এবং উন্নত রাউটিং প্রোটোকল শিখো।

🎯
নেটওয়ার্ক ইঞ্জিনিয়ারের দৈনন্দিন কাজ নেটওয়ার্ক ডিজাইন ও পরিকল্পনা, ডিভাইস কনফিগারেশন, ট্রাবলশুটিং, পারফরম্যান্স মনিটরিং, সিকিউরিটি প্যাচ প্রয়োগ, ডকুমেন্টেশন তৈরি এবং ক্যাপাসিটি পরিকল্পনা।
