১) Problem Solving Mindset (DSA শুরু করার আগে)

What (কি করবেন?)

সমস্যা দেখেই কোড লিখবেন না। আগে ৩টি জিনিস লিখুন:

  • Input কী?
  • Output কী?
  • Constraint কত? (n = 10³ না 10⁶?)

When (কখন optimize?)

যখন full scan বা nested loop বড় n এ ধীর হবে। উদাহরণ:

n = 100000 হলে O(n²) প্রায় অসম্ভব; O(n log n) বা O(n) দরকার।

How (কিভাবে?)

Step:

  1. Brute force লিখুন
  2. Time complexity হিসাব করুন
  3. Pattern apply করুন (HashMap/Two pointers/Window)
  4. Edge case দিয়ে test করুন
Decision formula: যদি বারবার lookup লাগে ⇒ HashMap; যদি sorted data ⇒ Binary Search/Two pointers; যদি range sum query ⇒ Prefix Sum; যদি sequence state repeated ⇒ Dynamic Programming.

২) Big-O Complexity (Backend engineer এর ভাষায়)

Big-O দিয়ে বোঝানো হয় ইনপুট বড় হলে আপনার কোডের সময়/মেমোরি কীভাবে বাড়ে।

Complexity নাম বাস্তব অর্থ
O(1)Constantইনপুট যতই হোক, সময় প্রায় একই
O(log n)Logarithmicখুব স্কেলেবল (Binary Search)
O(n)Linearএকবার পুরো ডাটা দেখা লাগে
O(n log n)Efficient Sortingভালো মানের sort (Merge/Quick গড়পড়তা)
O(n²)QuadraticNested loop, বড় ডাটায় ধীর

API উদাহরণ

১০০ ইউজার list এ id খোঁজা:

Loop search → O(n), কিন্তু id=>user map হলে → O(1) average

DB সম্পর্ক

DB index না থাকলে full table scan ≈ O(n)

Index থাকলে seek অপারেশন log-টাইপ আচরণ করে

Memory complexity

HashMap speed বাড়ায়, কিন্তু extra memory লাগে O(n)।

Trade-off বুঝে ব্যবহার করুন।

৩) Data Structure Selection Guide (কখন কোনটা?)

Array

What: ধারাবাহিক লিস্ট

কখন ব্যবহার করবেন: Indexed data, ordered list, simple collection

Use case: Product list, user ids, paginated response

How: index access O(1), search O(n)

নোট: মাঝখানে insert/delete খরচ বেশি

Hash Map (Associative Array)

What: key→value store

কখন: key দিয়ে দ্রুত খোঁজা দরকার

Use case: email → user lookup, cache map

How: hash table; average insert/search/delete O(1)

নোট: order guaranteed না

Stack (LIFO)

What: Last In First Out

কখন: শেষ ঢুকেছে, আগে বের হবে

Use case: Undo feature, function call stack

নোট: push/pop O(1)

Queue (FIFO)

কখন: আগে ঢুকেছে, আগে বের হবে

Use case: Job queue, message processing

নোট: Laravel queue-এর মূল ধারণা এটি

Tree

What: hierarchical relation

কখন: parent-child relation

Use case: Category tree, menu hierarchy

How: DFS/BFS traverse করে query

নোট: BST balanced হলে search দ্রুত

Graph

What: node + edge network

কখন: node-network relation

Use case: Social connections, route planning

How: shortest path/connected components এ BFS/DFS

নোট: BFS/DFS খুব গুরুত্বপূর্ণ

৫) Detailed PHP Code + Logic (What/When/How)

Example A: Hash Map দিয়ে দ্রুত lookup

<?php
$users = [
  ['id' => 1, 'email' => 'a@mail.com'],
  ['id' => 2, 'email' => 'b@mail.com'],
];

$byEmail = [];
foreach ($users as $u) {
  $byEmail[$u['email']] = $u;
}

$target = 'b@mail.com';
$result = $byEmail[$target] ?? null;
print_r($result);

What: একবার map বানিয়ে বারবার দ্রুত search

When: একই list থেকে একাধিক lookup

How logic: key তৈরি করে direct access; loop search এ প্রতিবার O(n), map এ O(1) average

Complexity: Build O(n), Search O(1) average, Memory O(n)

Example B: Queue (FIFO) — job processing idea

<?php
$queue = new SplQueue();
$queue->enqueue('SendWelcomeEmail');
$queue->enqueue('GenerateInvoicePdf');
$queue->enqueue('SyncCRM');

while (!$queue->isEmpty()) {
  $job = $queue->dequeue();
  echo "Processing: $job\n";
}

What: আগে আসা job আগে process

When: async ব্যাকগ্রাউন্ড কাজ

How logic: enqueue => শেষ লাইনে যোগ, dequeue => প্রথম থেকে বের

Laravel mapping: `dispatch()` করা job worker FIFO ধাঁচে টানে

Example C: Binary Search (sorted data)

<?php
function binarySearch(array $arr, int $target): int {
  $l = 0; $r = count($arr) - 1;
  while ($l <= $r) {
    $m = intdiv($l + $r, 2);
    if ($arr[$m] === $target) return $m;
    if ($arr[$m] < $target) $l = $m + 1;
    else $r = $m - 1;
  }
  return -1;
}

echo binarySearch([2,4,7,9,11,15], 11); // 4

What: search space অর্ধেক করে কমানো

When: data sorted + random access possible

How logic: mid compare, ছোট/বড় পাশে move

Complexity: O(log n)

Example D: Two Pointers — pair sum

<?php
function hasPairSum(array $arr, int $target): bool {
  sort($arr);
  $i = 0; $j = count($arr) - 1;
  while ($i < $j) {
    $sum = $arr[$i] + $arr[$j];
    if ($sum === $target) return true;
    if ($sum < $target) $i++;
    else $j--;
  }
  return false;
}

What: দুই প্রান্ত থেকে pointer move

When: sorted array তে pair/triplet/range problem

How logic: sum ছোট হলে left++, বড় হলে right--

Use case: price pair matching, budget split

Example E: Sliding Window — longest unique substring

<?php
function longestUniqueLen(string $s): int {
  $lastPos = [];
  $left = 0;
  $best = 0;

  for ($right = 0; $right < strlen($s); $right++) {
    $ch = $s[$right];
    if (isset($lastPos[$ch]) && $lastPos[$ch] >= $left) {
      $left = $lastPos[$ch] + 1;
    }
    $lastPos[$ch] = $right;
    $best = max($best, $right - $left + 1);
  }

  return $best;
}

echo longestUniqueLen('abcaefbb'); // 5 (bcaef)

When: continuous range/substring নিয়ে condition check

How logic: duplicate এলে window shrink, না হলে expand

Complexity: O(n)

Example F: Prefix Sum — range sum query

<?php
function buildPrefix(array $arr): array {
  $prefix = [0];
  $sum = 0;
  foreach ($arr as $v) {
    $sum += $v;
    $prefix[] = $sum;
  }
  return $prefix;
}

function rangeSum(array $prefix, int $l, int $r): int {
  return $prefix[$r + 1] - $prefix[$l];
}

$a = [3, 5, 2, 8, 1];
$p = buildPrefix($a);
echo rangeSum($p, 1, 3); // 5+2+8 = 15

When: একই array তে বারবার range sum query

How logic: cumulative sum store করে O(1) query

Complexity: Build O(n), each query O(1)

Example G: BFS on Tree-like categories

<?php
$graph = [
  1 => [2,3],
  2 => [4],
  3 => [5,6],
  4 => [],
  5 => [],
  6 => []
];

function bfs(array $graph, int $start): array {
  $q = new SplQueue();
  $q->enqueue($start);
  $visited = [$start => true];
  $order = [];

  while (!$q->isEmpty()) {
    $node = $q->dequeue();
    $order[] = $node;
    foreach ($graph[$node] as $nei) {
      if (!isset($visited[$nei])) {
        $visited[$nei] = true;
        $q->enqueue($nei);
      }
    }
  }
  return $order;
}

When: level wise traverse দরকার হলে

Use case: depth-wise category expand, shortest step count

Example H: Dynamic Programming (Fibonacci)

<?php
function fibDP(int $n): int {
  if ($n <= 1) return $n;
  $dp = array_fill(0, $n + 1, 0);
  $dp[1] = 1;

  for ($i = 2; $i <= $n; $i++) {
    $dp[$i] = $dp[$i - 1] + $dp[$i - 2];
  }
  return $dp[$n];
}

When: same subproblem repeated হলে

How logic: previous result cache করে পুনরায় calculation কমানো

Complexity: O(n), naive recursion O(2^n)

৬) Sorting কবে কোনটা?

Algorithm গড় সময় কখন ব্যবহার
Built-in sort() সাধারণত O(n log n) Production-এ প্রথম পছন্দ (নিজে sort লিখার দরকার কম)
Merge Sort O(n log n) Stable sort দরকার হলে
Quick Sort Avg O(n log n) প্র্যাকটিক্যালি দ্রুত, কিন্তু worst O(n²)
Bubble/Selection O(n²) শুধু শেখার জন্য
Rule: Production-এ built-in `sort/usort` আগে ব্যবহার করুন। custom sort সাধারণত তখনই দরকার যখন interview বা special comparator logic লাগে।

// $codes = ['E500','E401','E500','E500','E404'];
$freq = [];

  $freq[$c] = ($freq[$c] ?? 0) + 1;
