৩ Core Web Vitals

Core Web Vitals — সবচেয়ে গুরুত্বপূর্ণ মেট্রিক্স

Google ৩টি মেট্রিককে সবচেয়ে বেশি গুরুত্ব দেয়। এগুলোকে বলা হয় Core Web Vitals।

🎯 LCP — Largest Contentful Paint

কী? পেজের সবচেয়ে বড় এলিমেন্ট (হিরো ইমেজ বা বড় টেক্সট) কতক্ষণে লোড হয়।

কেন? ব্যবহারকারী কতক্ষণে সাইটের মূল কন্টেন্ট দেখতে পায় তা মাপে।

🟢 ভালো: ২.৫ সেকেন্ডের কম

🟡 উন্নতি দরকার: ২.৫–৪ সেকেন্ড

🔴 খারাপ: ৪ সেকেন্ডের বেশি

LCP ঠিক করার উপায়:

  • হিরো ইমেজ WebP ফরম্যাটে রাখুন
  • <img loading="eager" fetchpriority="high"> ব্যবহার করুন
  • সার্ভার রেসপন্স টাইম (TTFB) কমান
  • CSS-এ critical styles inline করুন

⚡ INP — Interaction to Next Paint (নতুন!)

কী? ব্যবহারকারী ক্লিক/ট্যাপ করার পর পেজ কতক্ষণে সাড়া দেয়।

কেন? সাইট ইন্টারেক্টিভ কিনা তা মাপে। এটি FID-এর জায়গা নিয়েছে।

🟢 ভালো: ২০০ms-এর কম

🟡 উন্নতি দরকার: ২০০–৫০০ms

🔴 খারাপ: ৫০০ms-এর বেশি

INP ঠিক করার উপায়:

  • Heavy JavaScript Main Thread থেকে সরিয়ে Web Worker-এ দিন
  • Event Handler-এ দীর্ঘ কাজ এড়ান
  • React-এ unnecessary re-render বন্ধ করুন
  • Debounce ও Throttle ব্যবহার করুন

📐 CLS — Cumulative Layout Shift

কী? পেজ লোডের সময় এলিমেন্টগুলো কতটা নড়াচড়া করে।

কেন? আপনি কিছু ক্লিক করতে গেলে অন্য কিছু ক্লিক হয়ে যায় — এটা CLS-এর কারণে।

🟢 ভালো: ০.১-এর কম

🟡 উন্নতি দরকার: ০.১–০.২৫

🔴 খারাপ: ০.২৫-এর বেশি

CLS ঠিক করার উপায়:

  • সব ইমেজে width ও height অ্যাট্রিবিউট দিন
  • Ads এবং Embed-এর জন্য নির্দিষ্ট স্থান রাখুন
  • Font লোডিং-এ font-display: swap ব্যবহার করুন
  • Dynamic content উপর থেকে না যোগ করে নিচে যোগ করুন
৪ ইমেজ অপটিমাইজেশন

ইমেজ অপটিমাইজেশন — সবচেয়ে সহজ উপায়ে বড় উন্নতি

বেশিরভাগ ওয়েবসাইটের ৬০-৮০% সাইজ শুধু ইমেজের কারণে। সঠিকভাবে অপটিমাইজ করলে পেজ সাইজ ৫০-৭০% কমানো সম্ভব।

ইমেজ ফরম্যাট কোনটা ব্যবহার করবেন?

ফরম্যাট কখন ব্যবহার করবেন সাইজ সাপোর্ট
WebP ফটো, স্ক্রিনশট, সাধারণ ছবি সবচেয়ে ছোট ৯৫%+ ব্রাউজার
AVIF হাই-কোয়ালিটি ফটো আরও ছোট ৮০%+ ব্রাউজার
SVG লোগো, আইকন, ইলাস্ট্রেশন খুব ছোট সব ব্রাউজার

JavaScript অপটিমাইজেশন

<!-- ✅ async: অন্য স্ক্রিপ্টের উপর নির্ভর না করলে -->
<script src="analytics.js" async></script>

<!-- ✅ তৃতীয় পক্ষের স্ক্রিপ্ট দেরিতে লোড করুন -->
<script>
  // পেজ ইন্টারেক্টিভ হওয়ার পর Google Analytics লোড করুন
  window.addEventListener('load', () => {
    setTimeout(() => {
      const script = document.createElement('script');
      script.src = 'https://www.googletagmanager.com/gtag/js?id=GA_ID';
      script.async = true;
      document.head.appendChild(script);
    }, 3000); // ৩ সেকেন্ড পরে লোড হবে
  });
</script>

৬ সার্ভার ও VPS অপটিমাইজেশন

সার্ভার কনফিগারেশন — এটা না করলে সব বৃথা!

দুর্দান্ত কোড লিখলেও সার্ভার ঠিকমতো কনফিগার না করলে ওয়েবসাইট দ্রুত হবে না। এই সেকশনে Nginx দিয়ে সবচেয়ে ভালো কনফিগ দেখানো হয়েছে।

Nginx — সেরা Performance কনফিগারেশন

NGINX — nginx.conf
# ===== GZIP কম্প্রেশন =====
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css application/json
           application/javascript text/xml application/xml
           application/xml+rss text/javascript image/svg+xml;
gzip_comp_level 6;

# ===== Brotli কম্প্রেশন (Gzip থেকেও ভালো) =====
brotli on;
brotli_comp_level 6;
brotli_types text/plain text/css application/json
             application/javascript image/svg+xml;

# ===== Browser Cache Headers =====
server {
  # Static assets — ১ বছর Cache
  location ~* \.(css|js|png|jpg|jpeg|webp|avif|woff2|ico)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
  }

  # HTML — Cache করবেন না (সবসময় নতুন দরকার)
  location ~* \.html$ {
    expires -1;
    add_header Cache-Control "no-store, no-cache";
  }

  # ===== Keep-Alive কানেকশন =====
  keepalive_timeout 65;
  keepalive_requests 1000;

  # ===== HTTP/2 চালু করুন =====
  listen 443 ssl http2;
  ssl_certificate /path/to/cert.pem;
  ssl_certificate_key /path/to/key.pem;

  # ===== Security + Speed Headers =====
  add_header X-Content-Type-Options "nosniff";
  add_header Referrer-Policy "no-referrer-when-downgrade";
}

৭ CDN ও Caching

CDN ও Cache — বিশ্বের যেকোনো জায়গা থেকে দ্রুত

CDN কী এবং কেন দরকার?

CDN (Content Delivery Network) হলো বিশ্বজুড়ে ছড়িয়ে থাকা সার্ভারের নেটওয়ার্ক। আপনার সাইটের ফাইল এই সার্ভারগুলোতে কপি হয়ে থাকে। ব্যবহারকারী তার কাছের সার্ভার থেকে ফাইল পায়, তাই দ্রুত লোড হয়।

CDNফ্রি প্ল্যানবৈশিষ্ট্যকার জন্য
Cloudflare আছে DDoS সুরক্ষা, ফ্রি SSL, Cache সবার জন্য সেরা
BunnyCDN ট্রায়াল সস্তা, দ্রুত, বাংলাদেশ POP বাজেট-সচেতন
Cloudfront সীমিত AWS ইকোসিস্টেম AWS ব্যবহারকারী
Fastly নেই এন্টারপ্রাইজ গ্রেড বড় কোম্পানি

Redis — Application Level Caching

PHP — Redis Cache উদাহরণ
<?php
// Redis সংযোগ স্থাপন
$redis = new Redis();
$redis->connect('127.0.0.1', 6379);

// Cache থেকে ডেটা নেওয়ার চেষ্টা
$cacheKey = 'homepage_data';
$cachedData = $redis->get($cacheKey);

if ($cachedData) {
    // Cache-এ আছে — সরাসরি দাও (খুব দ্রুত!)
    $data = json_decode($cachedData, true);
} else {
    // Cache-এ নেই — Database থেকে আনো
    $data = getDataFromDatabase();

    // Cache-এ সংরক্ষণ করো (১ ঘণ্টার জন্য)
    $redis->setex($cacheKey, 3600, json_encode($data));
}

// WordPress-এ Redis Object Cache plugin ইনস্টল করুন
// তাহলে WP_Cache সয়ংক্রিয়ভাবে Redis ব্যবহার করবে
💡
WordPress ব্যবহারকারীদের জন্য: W3 Total Cache বা WP Rocket ব্যবহার করুন। Cloudflare ফ্রিতে ব্যবহার করুন। Redis Object Cache plugin যোগ করুন। এই তিনটি করলেই WordPress অনেক দ্রুত হবে।

৯ ট্রাফিক ক্যালকুলেশন

কতটুকু ট্রাফিকের জন্য কত VPS দরকার?
