⚡ পারফরম্যান্স টেস্টিং

সাইট কত দ্রুত কাজ করছে তা পরীক্ষা করা। লোড টাইম, রেসপন্স টাইম এবং স্কেলেবিলিটি পরীক্ষা।

🔒 সিকিউরিটি টেস্টিং

হ্যাকিং এবং আক্রমণ থেকে সুরক্ষা পরীক্ষা। XSS, SQL Injection, CSRF প্রতিরোধ যাচাই।

✅ ফাংশনাল টেস্টিং

প্রতিটি ফিচার সঠিকভাবে কাজ করছে কিনা পরীক্ষা। User flows এবং ব্যবহারকারীর অভিজ্ঞতা যাচাই।

🎯 টেস্টিং এর প্রধান ধরন

  • ইউনিট টেস্টিং - কোডের প্রতিটি ছোট অংশ আলাদাভাবে পরীক্ষা
  • ইন্টিগ্রেশন টেস্টিং - বিভিন্ন মডিউল একসাথে কাজ করছে কিনা
  • সিস্টেম টেস্টিং - সম্পূর্ণ সিস্টেম একসাথে পরীক্ষা
  • E2E টেস্টিং - ব্যবহারকারীর সম্পূর্ণ যাত্রা পরীক্ষা

১. ইউনিট টেস্টিং (Unit Testing)

কোডের প্রতিটি ছোট অংশ (ফাংশন, মেথড) আলাদাভাবে পরীক্ষা করা। দ্রুত এক্সিকিউশন এবং দ্রুত ফিডব্যাক।

২. ইন্টিগ্রেশন টেস্টিং

বিভিন্ন মডিউল একসাথে কাজ করছে কিনা তা পরীক্ষা। ডাটাবেস, API এবং বাহ্যিক সার্ভিসের সাথে ইন্টিগ্রেশন।

৩. E2E টেস্টিং

ব্যবহারকারীর দৃষ্টিকোণ থেকে সম্পূর্ণ অ্যাপ্লিকেশন পরীক্ষা। ব্রাউজার অটোমেশন ব্যবহার করে।

৪. পারফরম্যান্স টেস্টিং

লোড, স্ট্রেস, স্পাইক এবং স্থিতিশীলতা পরীক্ষা। বিভিন্ন পরিস্থিতিতে সিস্টেমের আচরণ মূল্যায়ন।

⚠️ সতর্কতা

নিরাপত্তা পরীক্ষা সবসময় অনুমোদিত পরিবেশে করুন। অননুমোদিত সিস্টেমে পরীক্ষা করা আইনত দণ্ডনীয় অপরাধ।

OWASP Top 10

Open Web Application Security Project এর সবচেয়ে গুরুত্বপূর্ণ নিরাপত্তা ঝুঁকি:

  • A01: Broken Access Control - অননুমোদিত অ্যাক্সেস
  • A02: Cryptographic Failures - এনক্রিপশন সমস্যা
  • A03: Injection - SQL, XSS, Command
  • A04: Insecure Design - দুর্বল আর্কিটেকচার
  • A05: Security Misconfiguration - ভুল সেটআপ

🔐 XSS (Cross-Site Scripting)

ম্যালিসিয়াস স্ক্রিপ্ট ইনজেকশন থেকে সুরক্ষা পরীক্ষা করুন।

// XSS পেলোড টেস্ট কেস const xssPayloads = [ "<script>alert('XSS')</script>", "<img src=x onerror=alert('XSS')>", "<svg onload=alert('XSS')>", "javascript:alert('XSS')", "<body onload=alert('XSS')>", "<iframe src=javascript:alert('XSS')>" ]; // Sanitization function function sanitizeInput(input) { return input .replace(/</g, '&lt;') .replace(/>/g, '&gt;') .replace(/"/g, '&quot;'); }

প্রতিকার

  • Input sanitization এবং validation
  • Output encoding (HTML entities)
  • Content Security Policy (CSP) হেডার

🛡️ সিকিউরিটি হেডার পরীক্ষা

প্রতিটি HTTP রেসপন্সে সঠিক সিকিউরিটি হেডার আছে কিনা পরীক্ষা করুন।

// সিকিউরিটি হেডার সেটিং const securityHeaders = { // Content Security Policy 'Content-Security-Policy': "default-src 'self'; script-src 'self' 'nonce-{random}'; style-src 'self' 'unsafe-inline'", // XSS Protection 'X-XSS-Protection': '1; mode=block', // Clickjacking Protection 'X-Frame-Options': 'DENY', // MIME Type Sniffing 'X-Content-Type-Options': 'nosniff', // HSTS (HTTPS Only) 'Strict-Transport-Security': 'max-age=31536000; includeSubDomains', // Referrer Policy 'Referrer-Policy': 'strict-origin-when-cross-origin' };
হেডার কাজ গুরুত্ব
HSTS HTTPS বাধ্যতামূলক করে ★★★
CSP XSS এবং Injection প্রতিরোধ ★★★
X-Frame-Options Clickjacking প্রতিরোধ ★★
X-Content-Type-Options MIME sniffing বন্ধ ★★

ধাপ ১: রিকোয়ারমেন্ট বিশ্লেষণ

প্রতিটি ফিচারের সঠিক আচরণ নথিভুক্ত করুন। User Story এবং Acceptance Criteria তৈরি করুন।

ধাপ ২: টেস্ট কেস ডিজাইন

Positive (সঠিক ইনপুট) এবং Negative (ভুল ইনপুট) টেস্ট কেস তৈরি করুন।

ধাপ ৩: টেস্ট এনভায়রনমেন্ট সেটআপ

সঠিক টেস্ট ডেটা এবং পরিবেশ তৈরি করুন।

ধাপ ৪: টেস্ট এক্সিকিউশন

টেস্ট কেস রান করুন এবং ফলাফল রেকর্ড করুন।

ধাপ ৫: বাগ রিপোর্টিং

সমস্যাগুলো বিস্তারিতভাবে ডকুমেন্ট করুন এবং ট্র্যাক করুন।
