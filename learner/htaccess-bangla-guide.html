সার্ভার কনফিগারেশন ফাইল কী?
Introduction to server configuration files

যখন একটি ওয়েবসাইট কোনো সার্ভারে রান করে, তখন সার্ভারকে বলতে হয় — কোন ফাইল দেখাবে, কোন URL-কে কোথায় পাঠাবে, কোন ফাইল লুকানো থাকবে, কে অ্যাক্সেস পাবে ইত্যাদি। এই নির্দেশনাগুলো দেওয়া হয় কনফিগারেশন ফাইলের মাধ্যমে।

এই ফাইলগুলো সাধারণত সাধারণ টেক্সট ফাইল, কিন্তু এগুলোর নির্দিষ্ট নাম এবং নির্দিষ্ট সিনট্যাক্স থাকে। প্রতিটি সার্ভার সফটওয়্যার (Apache, Nginx, PHP ইত্যাদি) তার নিজের কনফিগারেশন ফাইল পড়ে।

.htaccess — সম্পূর্ণ গাইড
Apache Hypertext Access Configuration
🤔 .htaccess কী?

.htaccess (Hypertext Access) হলো Apache ওয়েব সার্ভারের একটি বিশেষ কনফিগারেশন ফাইল। এটি ডট (.) দিয়ে শুরু হয় কারণ Linux/Unix সিস্টেমে ডট দিয়ে শুরু হওয়া ফাইলগুলো লুকানো থাকে।

এই ফাইলটি একটি ডিরেক্টরিতে রাখলে শুধু সেই ডিরেক্টরি এবং তার সাব-ডিরেক্টরির উপর কাজ করে। মানে আপনি প্রতিটি ফোল্ডারে আলাদা আলাদা নিয়ম বানাতে পারবেন।

✅
কখন .htaccess ব্যবহার করবেন:
শেয়ার্ড হোস্টিংয়ে যেখানে মেইন Apache কনফিগ ফাইল edit করার অনুমতি নেই। cPanel ব্যবহারকারীদের জন্য এটিই প্রধান কনফিগ টুল।
⚠️
সতর্কতা:
.htaccess ফাইল প্রতিটি request-এ পড়া হয়, তাই অনেক বেশি নিয়ম থাকলে সাইট ধীর হতে পারে। সম্ভব হলে মেইন Apache config ব্যবহার করুন।

Redirect — URL পুনর্নির্দেশনা
301, 302, 307 redirect সব ধরনের
Redirect কোড বোঝা
কোডনামকখন ব্যবহার করবেনSEO প্রভাব
301Permanentপুরানো URL চিরতরে বন্ধ হলেLink juice ট্রান্সফার হয় ✅
302Temporaryসাময়িকভাবে অন্য জায়গায় পাঠাতেLink juice ট্রান্সফার হয় না
307Temp (strict)Method পরিবর্তন না করে redirect302 এর মতো
410Goneপেজ মুছে ফেলা হয়েছে বলাতেSearch engine cache মুছে
.htaccess — সব ধরনের Redirect উদাহরণ
Apache Config 
# ==============================
# ১. সিম্পল 301 Redirect (পুরানো পেজ → নতুন পেজ)
# ==============================
Redirect 301 /purano-page.html https://example.com/notun-page

# ২. সম্পূর্ণ ডোমেইন redirect (www → non-www)
RewriteEngine On
RewriteCond %{HTTP_HOST} ^www\.example\.com$ [NC]
RewriteRule ^(.*)$ https://example.com/$1 [R=301,L]

# ৩. non-www → www তে redirect
RewriteCond %{HTTP_HOST} !^www\. [NC]
RewriteRule ^(.*)$ https://www.%{HTTP_HOST}/$1 [R=301,L]

# ৪. HTTP → HTTPS redirect (SSL force করা)
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# ৫. একটি সম্পূর্ণ ফোল্ডার redirect
Redirect 301 /purano-folder/ https://example.com/notun-folder/

# ৬. Maintenance mode — সব traffic একটি পেজে
RewriteEngine On
RewriteCond %{REMOTE_ADDR} !^123\.456\.789\.000$  # আপনার IP বাদে
RewriteCond %{REQUEST_URI} !/maintenance\.html$ [NC]
RewriteRule ^(.*)$ /maintenance.html [R=302,L]

# ৭. পুরানো .html extension বাদ দেওয়া
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^([^\.]+)$ $1.html [NC,L]
💡
প্রো টিপ: RewriteCond হলো শর্ত, RewriteRule হলো কাজ। শর্ত সত্য হলেই কাজটি চলে। [R=301,L] মানে: 301 redirect করো এবং এটাই শেষ নিয়ম (L = Last)।

URL Rewriting — সুন্দর URL তৈরি
SEO-friendly clean URLs with mod_rewrite
URL Rewrite কেন দরকার?

ডিফল্টভাবে PHP ওয়েবসাইটে URL হয় এরকম: example.com/page.php?id=5&cat=news

URL Rewrite দিয়ে এটাকে সুন্দর করা যায়: example.com/news/page-title

এটি SEO-র জন্য ভালো, ব্যবহারকারীর জন্য বোধগম্য, এবং প্রফেশনাল দেখায়।

.htaccess — URL Rewrite উদাহরণ
Apache Config 
RewriteEngine On

# ===============================================
# ১. WordPress / Laravel এর জন্য মূল Rewrite
# সব request index.php তে পাঠায়
# ===============================================
RewriteBase /
RewriteRule ^index\.php$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.php [L]

# ===============================================
# ২. ?id=5 → /product/5 তে convert করা
# পুরানো: example.com/product.php?id=5
# নতুন:   example.com/product/5
# ===============================================
RewriteRule ^product/([0-9]+)/?$ product.php?id=$1 [L,QSA]

# ৩. news/2024/tech-news → news.php?year=2024&slug=tech-news
RewriteRule ^news/([0-9]{4})/([a-z0-9-]+)/?$ news.php?year=$1&slug=$2 [L,QSA]

# ৪. URL এর শেষের slash সবসময় যোগ করা
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule [^/]$ %{REQUEST_URI}/ [R=301,L]

# ৫. .php extension লুকানো
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_FILENAME}.php -f
RewriteRule ^(.+)$ $1.php [L,QSA]
🔣 Regex (Regular Expression) সংক্ষিপ্ত রেফারেন্স
প্যাটার্নমানেউদাহরণ
.যেকোনো একটি charactera.c → abc, aXc
*আগের character শূন্য বা অনেকবারgo*d → gd, god, good
+আগের character এক বা বেশিবারgo+d → god, good
?আগের character থাকতে পারে বা নাও পারেcolou?r
^শুরু^admin → admin দিয়ে শুরু
$শেষ\.html$ → .html দিয়ে শেষ
([0-9]+)এক বা বেশি সংখ্যা (capture)$1 দিয়ে ব্যবহার করুন
[a-z]ছোট হাতের যেকোনো letter—
[NC]Case insensitive—
[QSA]Query string append?a=1 সহ রাখে

RewriteEngine On

RewriteCond %{HTTP_REFERER} !^http(s)?://(www\.)?example\.com [NC]
RewriteRule \.(jpg|jpeg|png|gif|webp)$ - [NC,F,L]

# ৯. SQL Injection ও bad bots ব্লক
RewriteCond %{QUERY_STRING} (union.*select|insert.*into|delete.*from|drop.*table) [NC,OR]
RewriteCond %{HTTP_USER_AGENT} (libwww-perl|wget|curl|nikto|sqlmap) [NC]
RewriteRule .* - [F,L]

Password Protection — পাসওয়ার্ড সুরক্ষা
.htpasswd দিয়ে ফোল্ডার লক করুন

কোনো ফোল্ডারে ঢোকার আগে username/password চাইতে চাইলে .htaccess এবং .htpasswd একসাথে ব্যবহার করুন। Admin panel, staging site বা development সাইট লুকাতে এটি দারুণ কাজে আসে।

.htaccess
# ফোল্ডার password protect করা
AuthType Basic
AuthName "Protected Area"
AuthUserFile /home/user/.htpasswd
Require valid-user
.htpasswd
# format: username:encrypted_password
admin:$apr1$xyz$hashedpassword123
user2:$apr1$abc$anotherhashedpassword
💡
Password তৈরি করুন: Terminal-এ: htpasswd -c /path/.htpasswd username অথবা অনলাইনে: htpasswd generator ব্যবহার করুন। Password কখনো plain text রাখবেন না।

Performance 
# ==============================
# Browser Caching — ফাইল cache করা
# ব্রাউজার একবার ডাউনলোড করে রাখবে
# ==============================
<IfModule mod_expires.c>
    ExpiresActive On
    # ছবি — ১ বছর
    ExpiresByType image/jpeg "access plus 1 year"
    ExpiresByType image/png "access plus 1 year"
    ExpiresByType image/webp "access plus 1 year"
    ExpiresByType image/gif "access plus 1 year"
    # CSS ও JS — ১ মাস
    ExpiresByType text/css "access plus 1 month"
    ExpiresByType application/javascript "access plus 1 month"
    # Fonts — ১ বছর
    ExpiresByType font/woff2 "access plus 1 year"
    ExpiresByType application/font-woff "access plus 1 year"
    # HTML — ১ দিন
    ExpiresByType text/html "access plus 1 day"
</IfModule>

# ==============================
# GZIP Compression — ফাইল ছোট করে পাঠানো
# ==============================
<IfModule mod_deflate.c>
    AddOutputFilterByType DEFLATE text/html
    AddOutputFilterByType DEFLATE text/css
    AddOutputFilterByType DEFLATE text/javascript
    AddOutputFilterByType DEFLATE application/javascript
    AddOutputFilterByType DEFLATE application/json
    AddOutputFilterByType DEFLATE image/svg+xml
</IfModule>

# ETag বন্ধ করা (optional)
Header unset ETag
FileETag None

# /etc/nginx/sites-available/example.com

server {
    listen 80;
    server_name example.com www.example.com;

    # HTTP → HTTPS redirect
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name example.com www.example.com;

    # SSL সার্টিফিকেট
    ssl_certificate     /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    # Web root
    root /var/www/example.com/public;
    index index.php index.html;

    # Laravel / WordPress এর জন্য
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    # PHP-FPM দিয়ে PHP চালানো
    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }

    # Static files cache
    location ~* \.(jpg|jpeg|png|gif|css|js|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # .htaccess ব্লক
    location ~ /\.ht {
        deny all;
    }

    # Gzip
    gzip on;
    gzip_types text/plain text/css application/javascript application/json;
}

php.ini — PHP কনফিগারেশন ফাইল
PHP এর মূল সেটিংস ফাইল

php.ini হলো PHP-র মূল কনফিগারেশন ফাইল। এটি সার্ভারে PHP ইন্সটল হওয়ার সময় তৈরি হয়। Linux-এ সাধারণত /etc/php/8.x/apache2/php.ini বা /etc/php/8.x/cli/php.ini তে থাকে।

শেয়ার্ড হোস্টিংয়ে সরাসরি এডিট করা যায় না, তাই .htaccess দিয়ে অনেক সেটিং পরিবর্তন করতে হয়। VPS বা Dedicated সার্ভারে সরাসরি এডিট করা যায়।

php.ini — গুরুত্বপূর্ণ সেটিংস
PHP Config 
; ===============================
; php.ini — গুরুত্বপূর্ণ সেটিংস
; ===============================

; Error handling
display_errors = Off      ; Production-এ Off রাখুন
log_errors = On
error_log = /var/log/php_errors.log
error_reporting = E_ALL   ; Development-এ E_ALL

; File upload সীমা
upload_max_filesize = 64M
post_max_size = 64M
max_file_uploads = 20

; Execution time
max_execution_time = 300   ; সেকেন্ডে
max_input_time = 300

; Memory
memory_limit = 256M

; Timezone (বাংলাদেশের জন্য)
date.timezone = Asia/Dhaka

; Session সেটিংস
session.gc_maxlifetime = 3600
session.cookie_secure = 1    ; HTTPS এ
session.cookie_httponly = 1  ; JS থেকে লুকানো

; OPcache (PHP speed up)
opcache.enable = 1
opcache.memory_consumption = 256
opcache.max_accelerated_files = 10000

.env — Environment Variables
গোপন তথ্য ও পরিবেশ-নির্ভর সেটিংস

.env ফাইলে database password, API keys, secret keys ইত্যাদি গোপন তথ্য রাখা হয়। এই ফাইলটি কখনো Git-এ push করবেন না। Laravel, Node.js, React, Django সব ফ্রেমওয়ার্কেই এটি ব্যবহার হয়।

robots.txt — Search Bot নিয়ন্ত্রণ
Google, Bing কে কী দেখাবেন সেটি বলুন

robots.txt ওয়েবসাইটের root-এ রাখা হয়। Google, Bing ইত্যাদি search bot প্রথমেই এই ফাইল পড়ে জানে কোন পেজ crawl করবে আর কোনটা করবে না।

URL: https://example.com/robots.txt

robots.txt
# সব bot-কে নির্দেশনা
User-agent: *
Disallow: /admin/        # admin পেজ লুকাও
Disallow: /private/      # private ফোল্ডার
Disallow: /wp-admin/     # WordPress admin
Disallow: /cart/         # Shopping cart
Disallow: /checkout/
Disallow: /*.pdf$        # PDF index করো না
Allow: /wp-admin/admin-ajax.php  # WordPress AJAX allow

# শুধু Google bot-এর জন্য
User-agent: Googlebot
Allow: /

# কোনো একটি bot সম্পূর্ণ ব্লক
User-agent: AhrefsBot
Disallow: /

# Sitemap এর লোকেশন
Sitemap: https://example.com/sitemap.xml
⚠️
মনে রাখুন: robots.txt একটি নির্দেশনা, বাধ্যবাধকতা নয়। খারাপ bots এটি মানে না। সংবেদনশীল তথ্য লুকাতে .htaccess ব্যবহার করুন, robots.txt নয়।
🗺️
sitemap.xml — SEO Site Map
Search engine-কে সব পেজের তালিকা দিন
sitemap.xml
XML 
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">

    <!-- হোম পেজ -->
    <url>
        <loc>https://example.com/</loc>
        <lastmod>2024-12-01</lastmod>
        <changefreq>daily</changefreq>  <!-- কতটুকু পরিবর্তন হয় -->
        <priority>1.0</priority>    <!-- গুরুত্ব (0.0 - 1.0) -->
    </url>

    <!-- About পেজ -->
    <url>
        <loc>https://example.com/about</loc>
        <lastmod>2024-11-15</lastmod>
        <changefreq>monthly</changefreq>
        <priority>0.8</priority>
    </url>

    <!-- Blog পেজ -->
    <url>
        <loc>https://example.com/blog/post-title</loc>
        <lastmod>2024-12-10</lastmod>
        <changefreq>weekly</changefreq>
        <priority>0.6</priority>
    </url>

</urlset>
💡
changefreq মান: always, hourly, daily, weekly, monthly, yearly, never। WordPress সাইটে Yoast SEO বা Rank Math plugin দিয়ে auto sitemap তৈরি হয়।

🪟 web.config — Windows IIS সার্ভার

Windows Server-এ IIS ব্যবহার করলে Apache-র .htaccess এর বদলে web.config ব্যবহার করতে হয়।

web.config
XML / IIS 
<?xml version="1.0" encoding="UTF-8"?>
<configuration>
    <system.webServer>
        <!-- URL Rewrite -->
        <rewrite>
            <rules>
                <rule name="Laravel" stopProcessing="true">
                    <match url="^" />
                    <action type="Rewrite" url="index.php/{R:0}" />
                </rule>
            </rules>
        </rewrite>

        <!-- Security Headers -->
        <httpProtocol>
            <customHeaders>
                <add name="X-Frame-Options" value="SAMEORIGIN" />
                <add name="X-Content-Type-Options" value="nosniff" />
            </customHeaders>
        </httpProtocol>
    </system.webServer>
</configuration>
🐳 Dockerfile — Container তৈরি

Docker ব্যবহার করলে Dockerfile দিয়ে আপনার অ্যাপের পরিবেশ তৈরি করা হয়।

Dockerfile (PHP/Laravel)

FROM php:8.2-fpm

# Extensions ইন্সটল
RUN docker-php-ext-install pdo pdo_mysql bcmath

# Composer ইন্সটল
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

# Working directory
WORKDIR /var/www/html

# ফাইল copy
COPY . .

# Dependencies ইন্সটল
RUN composer install --no-dev --optimize-autoloader

# Port expose করুন

প্রো টিপস ও সাধারণ ভুল
Expert advice for beginners and pros
✅ সঠিক অভ্যাস
  • সবসময় ব্যাকআপ নিন — .htaccess edit করার আগে বর্তমান ফাইলের কপি রাখুন। একটি ভুল নিয়ম পুরো সাইট 500 Error দিতে পারে।
  • .env Git-এ push করবেন না — কখনো .env ফাইল GitHub বা GitLab-এ আপলোড করবেন না। এতে database password, API key সব প্রকাশ হয়ে যায়।
  • RewriteEngine On সবসময় আগে — RewriteRule লেখার আগে অবশ্যই RewriteEngine On লিখুন নাহলে কিছুই কাজ করবে না।
  • Production-এ Display Errors Off রাখুন — php.ini বা .htaccess-এ display_errors = Off রাখুন। Error দেখালে হ্যাকাররা সার্ভারের তথ্য পেয়ে যায়।
  • robots.txt পাবলিক — robots.txt সবসময় publicly accessible থাকে। এতে গোপন URL লিখবেন না কারণ হ্যাকাররা এটি পড়তে পারে।
  • HTTPS force করুন — সবসময় HTTP থেকে HTTPS-এ redirect রাখুন। এটি SEO ও নিরাপত্তা উভয়ের জন্যই জরুরি।
  • নিয়মিত অডিট করুন — মাঝে মাঝে .htaccess চেক করুন পুরানো, অব্যবহৃত নিয়ম মুছে ফেলুন কারণ এটি সাইটের গতি ধীর করে।

🎯 সম্পূর্ণ .htaccess Template (WordPress/PHP)
Complete Template 
# ================================================
# সম্পূর্ণ .htaccess Template
# WordPress / PHP সাইটের জন্য
# ================================================

# ১. Directory listing বন্ধ
Options -Indexes

# ২. HTTP → HTTPS redirect
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

# ৩. www → non-www
RewriteCond %{HTTP_HOST} ^www\.(.+)$ [NC]
RewriteRule ^ https://%1%{REQUEST_URI} [R=301,L]

# ৪. WordPress Core Rewrite
RewriteBase /
RewriteRule ^index\.php$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.php [L]

# ৫. গুরুত্বপূর্ণ ফাইল protect
<FilesMatch "(\.env|wp-config\.php|\.htpasswd)">
    Require all denied
</FilesMatch>

# ৬. Security Headers
Header always set X-Frame-Options "SAMEORIGIN"
Header always set X-XSS-Protection "1; mode=block"
Header always set X-Content-Type-Options "nosniff"

# ৭. PHP সেটিংস
php_value upload_max_filesize 64M
php_value post_max_size 64M
php_value memory_limit 256M
php_flag display_errors Off

# ৮. Browser Caching
<IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType image/jpeg "access plus 1 year"
    ExpiresByType text/css "access plus 1 month"
    ExpiresByType application/javascript "access plus 1 month"
</IfModule>

# ৯. GZIP Compression
<IfModule mod_deflate.c>
    AddOutputFilterByType DEFLATE text/html text/css application/javascript
</IfModule>

# ১০. Custom Error Pages
ErrorDocument 404 /404.html
ErrorDocument 500 /500.html
