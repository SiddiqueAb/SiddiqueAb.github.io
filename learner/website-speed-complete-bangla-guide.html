রেন্ডারিং পদ্ধতি — কোনটা কখন?

🔵 SSG — Static Site Generation

কী হয়: Build সময়ে সব HTML তৈরি হয়। সার্ভারে রেডি ফাইল থাকে।

সুবিধা: সবচেয়ে দ্রুত, সার্ভারলেস হোস্টিং সম্ভব, CDN-এ সহজ ক্যাশ।

সীমাবদ্ধতা: ডেটা পরিবর্তন হলে আবার build করতে হয়।

উদাহরণ: ব্লগ, ডকুমেন্টেশন, পোর্টফোলিও

🟡 ISR — Incremental Static Regeneration

কী হয়: SSG কিন্তু নির্দিষ্ট সময় পর পেজ আপডেট হয়।

সুবিধা: SSG-এর স্পিড + dynamic content।

উদাহরণ: নিউজ সাইট, প্রোডাক্ট পেজ (Next.js-এ আছে)

🟢 SSR — Server-Side Rendering

কী হয়: প্রতিটি request-এ সার্ভার HTML তৈরি করে পাঠায়।

সুবিধা: সবসময় fresh ডেটা, user-specific content।

সীমাবদ্ধতা: SSG-এর চেয়ে ধীর, সার্ভার চাপ বেশি।

উদাহরণ: ই-কমার্স প্রোডাক্ট, ড্যাশবোর্ড, User-specific পেজ

🔴 CSR — Client-Side Rendering (SPA)

কী হয়: ব্রাউজারে JavaScript দিয়ে সব render হয়।

সীমাবদ্ধতা: প্রথম লোড ধীর, SEO সমস্যা।

উদাহরণ: Admin Panel, ড্যাশবোর্ড (SEO দরকার নেই)

প্রতিটি Metric বিশ্লেষণ

⏱️ TTFB — Time to First Byte

ব্রাউজার request পাঠানোর পর সার্ভার কতক্ষণে প্রথম byte পাঠায়।

🟢 ভালো: <800ms | 🟡 ঠিকঠাক: 800ms-1.8s | 🔴 খারাপ: >1.8s

কারণ ও সমাধান:
  • Database slow query → Index যোগ করুন
  • PHP অপটিমাইজ নেই → OPcache চালু করুন
  • সার্ভার দূরে → Singapore DC বা CDN ব্যবহার করুন
  • কোনো Cache নেই → Redis বা Full-page cache যোগ করুন

🎨 FCP — First Contentful Paint

পেজে প্রথম কোনো কিছু (টেক্সট/ছবি) দেখা যায় কতক্ষণে।

🟢 ভালো: <1.8s | 🟡 ঠিকঠাক: 1.8-3s | 🔴 খারাপ: >3s

কারণ ও সমাধান:
  • Render-blocking CSS → Critical CSS inline করুন
  • Render-blocking JS → defer/async ব্যবহার করুন
  • Font লোড দেরি → font-display:swap ব্যবহার করুন
  • বড় HTML → HTML Minify করুন

PageSpeed ব্যবহারের সঠিক পদ্ধতি

  • ১
    Mobile আগে ঠিক করুন: Mobile স্কোর সবসময় Desktop-এর চেয়ে কম হয়। Google Mobile-First Indexing করে — তাই Mobile স্কোরই বেশি গুরুত্বপূর্ণ।
  • ২
    Opportunities সবার আগে দেখুন: এগুলো ঠিক করলে সবচেয়ে বেশি উপকার হবে। প্রতিটিতে সম্ভাব্য টাইম সেভিং দেখানো হয়।
  • ৩
    একটা একটা করে ঠিক করুন: একসাথে সব পরিবর্তন করলে কোন পরিবর্তনে কতটুকু উন্নতি হলো বোঝা যায় না।
  • ৪
    Multiple Run করুন: একবার টেস্ট করে সিদ্ধান্ত নেবেন না — ৩-৫ বার রান করে গড় নিন। সার্ভার লোডের কারণে ভ্যারিয়েশন হয়।
  • ৫
    WebPageTest.org দিয়েও টেস্ট করুন: Singapore বা Mumbai location সিলেক্ট করে টেস্ট করলে বাংলাদেশের কাছাকাছি ফলাফল পাবেন।

⚡ INP — Interaction to Next Paint (২০২৪ নতুন মেট্রিক)

ব্যবহারকারী click/tap/keyboard করলে পেজ কতক্ষণে visual response দেয়। FID-কে replace করেছে।

📌
FID vs INP পার্থক্য: FID শুধু প্রথম interaction মাপত। INP পুরো পেজ ব্যবহারের সময় সব interaction-এর worst-case মাপে। তাই INP উন্নত করা FID-এর চেয়ে কঠিন।

INP বাড়ার কারণ ও সমাধান

❌ কারণ ১

Long Task — Main Thread ৫০ms+ ব্লক হচ্ছে JavaScript-এ

✅ সমাধান

বড় JavaScript task ভেঙে ছোট ছোট করুন। setTimeout(fn, 0) বা scheduler.postTask() ব্যবহার করুন।

❌ কারণ ২

Third-party Scripts — Google Tag Manager, FB Pixel Main Thread block করছে

✅ সমাধান

Third-party script পেজ লোডের পরে async লোড করুন। Partytown লাইব্রেরি ব্যবহার করুন।

❌ কারণ ৩

React/Vue অতিরিক্ত re-render

✅ সমাধান

React.memo, useMemo, useCallback ব্যবহার করুন। State minimally রাখুন।

❌ কারণ ৪

Event Listener-এ ভারী কাজ

✅ সমাধান

Input event-এ Debounce, Scroll event-এ Throttle ব্যবহার করুন।

JavaScript
INP উন্নত করার কোড
// ❌ ভুল — Main Thread block করে
button.addEventListener('click', () => {
  heavyCalculation(); // ৫০০ms লাগে!
  updateUI();
});

// ✅ সঠিক — Task ভেঙে দিন
button.addEventListener('click', () => {
  updateUI(); // আগে UI update করুন (instant feedback)
  
  // ভারী কাজ পরে করুন
  setTimeout(() => {
    heavyCalculation();
  }, 0);
});

// ✅ আরও ভালো — Web Worker ব্যবহার করুন
const worker = new Worker('heavy-worker.js');

button.addEventListener('click', () => {
  updateUI(); // Main thread-এ instant
  worker.postMessage({ data }); // Background thread-এ
});

// ✅ Debounce উদাহরণ — Search Input
function debounce(fn, delay) {
  let timer;
  return (...args) => {
    clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}

const handleSearch = debounce((query) => {
  fetchSearchResults(query);
}, 300); // ৩০০ms পরে চলবে

searchInput.addEventListener('input', (e) => {
  handleSearch(e.target.value);
});

📐 CLS — Cumulative Layout Shift (বিস্তারিত)

পেজ লোডের সময় এলিমেন্ট নড়াচড়া করলে CLS বাড়ে। ব্যবহারকারী কিছু ক্লিক করতে গেলে হঠাৎ এলিমেন্ট সরে গেলে বিরক্তিকর।

CLS বাড়ার সাধারণ কারণ

❌ কারণ ১ — সবচেয়ে সাধারণ

ইমেজে width/height নেই — ব্রাউজার জানে না ইমেজ কতটুকু জায়গা নেবে, লোড হওয়ার পর সব নিচে নামে

✅ সমাধান

সব ইমেজে সবসময় width ও height অ্যাট্রিবিউট দিন। CSS-এ aspect-ratio ব্যবহার করুন।

❌ কারণ ২

Ads বা Embed — Google Ads, YouTube Embed-এর জায়গা আগে রাখা নেই

✅ সমাধান

Ads-এর জন্য fixed height div রাখুন। min-height বা aspect-ratio ব্যবহার করুন।

❌ কারণ ৩

Web Font FOUT — ফলব্যাক ফন্ট থেকে Web Font লোড হওয়ার সময় টেক্সট এলিমেন্ট সাইজ পরিবর্তন হয়

✅ সমাধান

font-display: swap + font preload + size-adjust ব্যবহার করুন

Caching Technology — সম্পূর্ণ গাইড

Caching হলো ওয়েব পারফরম্যান্সের সবচেয়ে শক্তিশালী টুল। সঠিক caching strategy দিয়ে একটি সাইটকে ১০-১০০ গুণ দ্রুত করা সম্ভব।

Cache-এর স্তরসমূহ — কোনটা কোথায় কাজ করে

Browser Cache
0ms

10-50ms

0.1-1ms

50-500ms

Service Worker Cache — Offline-First Strategy

JavaScript
Service Worker Cache Strategy
// service-worker.js
const CACHE_NAME = 'mysite-v1';
const STATIC_ASSETS = [
  '/', '/styles.css', '/app.js', '/offline.html'
];

// Install: Static assets cache করুন
self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME)
      .then(cache => cache.addAll(STATIC_ASSETS))
  );
  self.skipWaiting();
});

// Fetch: Cache-First Strategy (Static)
self.addEventListener('fetch', (event) => {
  const { request } = event;
  
  // Static assets — Cache First
  if (request.destination === 'image' ||
      request.url.includes('.css') ||
      request.url.includes('.js')) {
    event.respondWith(
      caches.match(request).then(cached => {
        return cached || fetch(request).then(response => {
          const clone = response.clone();
          caches.open(CACHE_NAME).then(c => c.put(request, clone));
          return response;
        });
      })
    );
  }
  
  // HTML — Network First (সবসময় নতুন)
  else if (request.mode === 'navigate') {
    event.respondWith(
      fetch(request).catch(() => caches.match('/offline.html'))
    );
  }
});

১৩ সাধারণ সমস্যা ও সমাধান

১৫টি সাধারণ পারফরম্যান্স সমস্যা ও সমাধান

PageSpeed-এ যে সমস্যাগুলো সবচেয়ে বেশি দেখা যায় এবং তাদের সমাধান।

❌ সমস্যা ১

"Eliminate render-blocking resources" — CSS/JS পেজ render ব্লক করছে

✅ সমাধান

CSS-এ preload + onload trick, JS-এ defer যোগ করুন। Critical CSS inline করুন।

❌ সমস্যা ২

"Properly size images" — ইমেজ দরকারের চেয়ে বড়

✅ সমাধান

srcset + sizes দিয়ে responsive image ব্যবহার করুন। মোবাইলে ছোট ইমেজ পাঠান।

❌ সমস্যা ৩

"Defer offscreen images" — নিচের ইমেজ শুরুতেই লোড হচ্ছে

✅ সমাধান

সব below-the-fold ইমেজে loading="lazy" যোগ করুন।

❌ সমস্যা ৪

"Serve images in next-gen formats" — JPEG/PNG ব্যবহার হচ্ছে

✅ সমাধান

সব ইমেজ WebP-তে convert করুন। <picture> tag দিয়ে AVIF/WebP/JPEG fallback রাখুন।

❌ সমস্যা ৫

"Remove unused JavaScript" — বড় JS bundle কিন্তু বেশিরভাগ ব্যবহার হয় না

✅ সমাধান

Code splitting করুন, Tree shaking চালু রাখুন। Webpack Bundle Analyzer দিয়ে দেখুন কোন library বড়।

❌ সমস্যা ৬

"Remove unused CSS" — বড় CSS কিন্তু অধিকাংশ ব্যবহার হয় না

✅ সমাধান

PurgeCSS বা UnCSS ব্যবহার করুন। Tailwind CSS ব্যবহার করলে অটোমেটিক purge হয়।

❌ সমস্যা ৭

"Minify CSS/JS/HTML" — Whitespace ও comments আছে production-এ

✅ সমাধান

Build process-এ cssnano, Terser, html-minifier যোগ করুন। Webpack/Vite production mode-এ অটো করে।

❌ সমস্যা ৮

"Enable text compression" — Gzip/Brotli চালু নেই

✅ সমাধান

Nginx-এ gzip চালু করুন। Cloudflare ব্যবহার করলে অটোমেটিক Brotli পাবেন।

❌ সমস্যা ৯

"Reduce initial server response time (TTFB)"

✅ সমাধান

PHP OPcache, Redis Object Cache, Full Page Cache চালু করুন। Database slow query ঠিক করুন।

❌ সমস্যা ১০

"Avoid chaining critical requests" — বহু sequential request

✅ সমাধান

HTTP/2 চালু করুন (multiplexing করে)। Resource hints দিয়ে critical resources preload করুন।

❌ সমস্যা ১১

"Avoid enormous network payloads" — মোট পেজ সাইজ অনেক বেশি

✅ সমাধান

লক্ষ্য রাখুন: HTML <50KB, CSS <100KB, JS <200KB, মোট <1MB। বড় library replace করুন।

❌ সমস্যা ১২

"Reduce the impact of third-party code"

✅ সমাধান

Google Tag Manager ব্যবহার করুন এবং সব script দেরিতে fire করুন। Partytown দিয়ে Web Worker-এ রান করুন।

❌ সমস্যা ১৩

"Avoid large layout shifts" (CLS)

✅ সমাধান

সব ইমেজে width/height দিন। Ads-এ min-height রাখুন। Web Font-এ size-adjust ব্যবহার করুন।

❌ সমস্যা ১৪

"Use efficient cache policy on static assets"

✅ সমাধান

Static files-এ Cache-Control: public, max-age=31536000, immutable। File হলে নতুন filename দিন (versioning)।

❌ সমস্যা ১৫

"Preload Largest Contentful Paint image"

✅ সমাধান

<head>-এ <link rel="preload" href="hero.webp" as="image"> যোগ করুন। fetchpriority="high" দিন।

Ubuntu VPS Initial Setup Script # ======================================== # System update apt update && apt upgrade -y # Nginx ইনস্টল apt install nginx -y systemctl enable nginx # PHP 8.2 + FPM ইনস্টল apt install php8.2-fpm php8.2-mysql php8.2-redis \ php8.2-mbstring php8.2-xml php8.2-curl php8.2-zip -y # MySQL ইনস্টল apt install mysql-server -y mysql_secure_installation # Redis ইনস্টল apt install redis-server -y sed -i 's/supervised no/supervised systemd/' /etc/redis/redis.conf systemctl enable redis-server # Let's Encrypt SSL apt install certbot python3-certbot-nginx -y certbot --nginx -d example.com -d www.example.com # Firewall সেটআপ ufw allow ssh ufw allow 'Nginx Full' ufw enable # PHP OPcache চালু cat >> /etc/php/8.2/fpm/conf.d/10-opcache.ini << 'EOF' opcache.enable=1 opcache.memory_consumption=256 opcache.max_accelerated_files=20000 opcache.validate_timestamps=0 opcache.save_comments=1 opcache.fast_shutdown=1 EOF # PHP-FPM অপটিমাইজেশন (১GB RAM VPS) sed -i 's/pm = dynamic/pm = dynamic/' /etc/php/8.2/fpm/pool.d/www.conf sed -i 's/pm.max_children = 5/pm.max_children = 15/' /etc/php/8.2/fpm/pool.d/www.conf sed -i 's/pm.start_servers = 2/pm.start_servers = 4/' /etc/php/8.2/fpm/pool.d/www.conf sed -i 's/pm.min_spare_servers = 1/pm.min_spare_servers = 2/' /etc/php/8.2/fpm/pool.d/www.conf sed -i 's/pm.max_spare_servers = 3/pm.max_spare_servers = 6/' /etc/php/8.2/fpm/pool.d/www.conf # সব সার্ভিস restart systemctl restart nginx php8.2-fpm mysql redis-server echo "✅ Setup Complete!"
