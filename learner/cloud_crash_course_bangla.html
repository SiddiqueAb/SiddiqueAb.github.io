01 — Amazon Web Services

AWS — ক্লাউডের রাজা

AWS হলো বিশ্বের সবচেয়ে বড় এবং সবচেয়ে বেশি ব্যবহৃত ক্লাউড প্ল্যাটফর্ম। Amazon তৈরি করেছে এবং ২০০৬ সাল থেকে চলছে।

🖥️ EC2
Elastic Compute Cloud

ক্লাউডে ভার্চুয়াল সার্ভার। তোমার নিজের কম্পিউটারের মতো, কিন্তু AWS-এর ডেটাসেন্টারে। যেকোনো OS (Linux/Windows), যেকোনো সাইজ।

  • t2.micro = ফ্রি টায়ার, ছোট প্রজেক্টের জন্য
  • m5.xlarge = প্রোডাকশন অ্যাপের জন্য
  • Auto Scaling — ট্রাফিক বাড়লে সার্ভার বাড়াও
  • Load Balancer দিয়ে ট্রাফিক ভাগ করো
# EC2 instance চালু করো aws ec2 run-instances \ --image-id ami-0abcdef1234567890 \ --instance-type t2.micro \ --key-name MyKeyPair
🪣 S3
Simple Storage Service

Object storage — যেকোনো ফাইল স্টোর করো। ছবি, ভিডিও, ব্যাকআপ, স্ট্যাটিক ওয়েবসাইট সব। ৯৯.৯৯% durability গ্যারান্টি।

  • Bucket = ফোল্ডারের মতো, globally unique নাম
  • Object = যেকোনো ফাইল, সর্বোচ্চ ৫ TB
  • Static Website Hosting করা যায় সরাসরি
  • Versioning — ফাইলের পুরনো ভার্সন রাখো
# S3-তে ফাইল আপলোড aws s3 cp myfile.pdf \ s3://my-bucket/uploads/myfile.pdf # Public URL পাবে এভাবে: # https://my-bucket.s3.amazonaws.com/...
🗄️ RDS
Relational Database Service

Managed SQL ডেটাবেস। MySQL, PostgreSQL, Oracle, SQL Server — নিজে install না করে AWS পরিচালনা করে দেয়।

  • Automated backups, patching, failover
  • Multi-AZ = দুটো AZ-তে replica, HA নিশ্চিত
  • Read Replica = Read traffic আলাদা করো
  • Aurora = AWS-এর নিজস্ব, ৫x দ্রুত MySQL
# RDS instance তৈরি aws rds create-db-instance \ --db-instance-identifier mydb \ --db-instance-class db.t3.micro \ --engine mysql \ --master-username admin \ --master-user-password password
⚡ Lambda
Serverless Functions

কোড লেখো, সার্ভার ভুলে যাও। Event trigger হলে function চলে, শেষ হলে বন্ধ। শুধু execution time-এর পয়সা দাও।

  • Node.js, Python, Java, Go, Ruby সাপোর্ট
  • Free tier: ১ মিলিয়ন request/মাস
  • API Gateway দিয়ে HTTP endpoint বানাও
  • S3 event, DynamoDB stream দিয়ে trigger
🌐 VPC
Virtual Private Cloud

তোমার নিজস্ব private network AWS-এ। Resources isolate করো, security নিশ্চিত করো। প্রতিটি AWS account-এ default VPC থাকে।

  • CIDR block দিয়ে IP range নির্ধারণ
  • Public/Private Subnet আলাদা করো
  • Security Group = Firewall rules
  • Peering = দুটো VPC connect করো
🔐 IAM
Identity & Access Management

কে কী করতে পারবে তা নিয়ন্ত্রণ করো। Users, Groups, Roles, Policies — AWS-এর security-র মূল ভিত্তি।

  • Least Privilege Principle — শুধু দরকারী permission
  • Role = EC2/Lambda নিজেই অন্য সার্ভিস access করতে পারে
  • MFA চালু রাখো root account-এ সবসময়
  • Policy = JSON-এ permission define করা
// IAM Policy উদাহরণ { "Effect": "Allow", "Action": ["s3:GetObject"], "Resource": "arn:aws:s3:::my-bucket/*" }
🚢 EKS
Elastic Kubernetes Service

AWS-এ Managed Kubernetes। Container orchestration — হাজারো container চালাও, scale করো, manage করো অটোমেটিক।

  • Control Plane AWS manage করে, তুমি শুধু Worker Node
  • Fargate দিয়ে serverless pod চালাও
  • kubectl দিয়ে deploy করো যেকোনো সময়
  • Helm chart দিয়ে application package করো
📦 ECR
Elastic Container Registry

Docker image store করার জায়গা। DockerHub-এর মতো, কিন্তু AWS-এর private registry। EKS/ECS-এর সাথে perfectly integrate।

  • docker push করো, ECR-এ রাখো
  • Image scanning — vulnerability চেক করে
  • Lifecycle policy — পুরনো image auto delete
  • Cross-region replication সম্ভব
🌍 CloudFront
CDN — Content Delivery Network

বিশ্বের ৪০০+ Edge Location থেকে content serve করো। User-এর কাছের সার্ভার থেকে দেয়, তাই দ্রুত লোড হয়।

  • S3 + CloudFront = blazing fast static site
  • HTTPS সার্টিফিকেট বিনামূল্যে (ACM)
  • DDoS protection built-in (AWS Shield)
  • Cache-Control header দিয়ে TTL set করো
📡 Route 53
DNS Service

Domain name management। myapp.com → EC2 IP, এই mapping করে। ৯৯.৯৯% SLA গ্যারান্টি দেওয়া একমাত্র AWS সার্ভিস।

  • A Record, CNAME, MX, TXT — সব ধরনের DNS
  • Health Check — সার্ভার down হলে traffic সরাও
  • Weighted routing — A/B testing-এ কাজে আসে
  • Latency-based routing — কাছের region-এ পাঠাও
📊 CloudWatch
Monitoring & Observability

AWS infrastructure monitor করো। Metrics, Logs, Alarms, Dashboards — সব এক জায়গায়। কিছু wrong হলে alert পাও।

  • CPU, Memory, Network — সব metric দেখো
  • Log Insights — log query করো SQL-এর মতো
  • Alarm — CPU >80% হলে email/SMS পাঠাও
  • EventBridge দিয়ে automated response
💡
AWS Exam টিপস: AWS Solutions Architect Associate পরীক্ষার জন্য — EC2, S3, VPC, IAM, RDS এই ৫টি সার্ভিস সবচেয়ে বেশি প্রশ্ন আসে। প্রতিটির Use Case মুখস্থ রাখো।
02 — Google Cloud Platform

GCP — Google-এর ক্লাউড

Google নিজে যেই infrastructure ব্যবহার করে Search, Gmail, YouTube চালাতে — সেটাই তোমাকে দেয় GCP হিসেবে। ML/AI-তে সবচেয়ে এগিয়ে।

☸️ GKE
Google Kubernetes Engine

Kubernetes Google-ই বানিয়েছে, তাই GKE সবচেয়ে mature। Autopilot mode-এ node management-ও Google করে দেয়।

  • Autopilot = node, scaling সব Google manage করে
  • Standard = তুমি নিজে node control করো
  • Workload Identity — pod-কে GCP API access দাও
  • Multi-cluster মানেজমেন্ট Anthos দিয়ে
# GKE cluster তৈরি gcloud container clusters create my-cluster \ --zone us-central1-a \ --num-nodes 3 # kubectl configure করো gcloud container clusters get-credentials my-cluster
🏃 Cloud Run
Serverless Containers

Docker container চালাও, সার্ভার ম্যানেজ করো না। Request এলে চলে, না থাকলে zero scale হয়। Lambda-র চেয়ে বেশি flexible।

  • যেকোনো ভাষা, যেকোনো framework — container হলেই হয়
  • ০ থেকে ১০০০ instance অটো scale
  • HTTPS endpoint automatically পাবে
  • Traffic split — Blue/Green deployment সহজে
# Cloud Run-এ deploy করো gcloud run deploy my-app \ --image gcr.io/my-project/my-app \ --platform managed \ --allow-unauthenticated
📊 BigQuery
Serverless Data Warehouse

পেটাবাইট ডেটা SQL দিয়ে query করো সেকেন্ডে। Data Analytics-এর জন্য সেরা। কোনো cluster manage করতে হয় না।

  • Standard SQL দিয়ে কোয়েরি করো
  • ML built-in — SQL দিয়েই model train করো
  • Streaming insert — real-time data আসুক
  • Connected Sheets — Excel-এর মতো দেখো
🗂️ Cloud Storage
Object Storage (S3-এর বিকল্প)

AWS S3-এর মতো Google-এর object storage। Bucket তৈরি করো, object রাখো। BigQuery-র সাথে নেটিভ integration।

  • Standard, Nearline, Coldline, Archive — ৪ class
  • ব্যবহার কম = পয়সা কম (Coldline সস্তা)
  • Uniform bucket-level access — IAM policy
  • Transfer Service — অন্য cloud থেকে migrate করো
🗃️ Cloud SQL
Managed Relational Database

AWS RDS-এর সমতুল্য। MySQL, PostgreSQL, SQL Server — Google manage করে। Automatic backup, failover সব আছে।

  • High Availability — regional failover
  • Read Replica — পৃথক read endpoint
  • Point-in-time recovery
  • Cloud SQL Auth Proxy — secure connection
🎯
কখন GCP বেছে নেবে? তোমার কাজে Machine Learning, Big Data Analytics, বা Kubernetes-heavy workload থাকলে GCP সেরা পছন্দ। Google-এর global fiber network-এর কারণে latency খুব কম।
03 — Microsoft Azure

Azure — Enterprise-এর পছন্দ

Microsoft Azure বড় বড় কর্পোরেট প্রতিষ্ঠানে সবচেয়ে বেশি ব্যবহৃত। Active Directory, Office 365, Windows Server-এর সাথে সেরা integration।

☸️ AKS
Azure Kubernetes Service

Microsoft-এর managed Kubernetes। Azure Active Directory-এর সাথে দারুণ integration। Enterprise-এ সবচেয়ে বেশি চলে।

  • Azure AD integration — company login দিয়ে cluster access
  • KEDA — event-driven autoscaling
  • Azure Monitor + Container Insights
  • Virtual Nodes — Serverless pod via ACI
# AKS cluster তৈরি az aks create \ --resource-group MyRG \ --name myAKSCluster \ --node-count 3 \ --enable-addons monitoring # kubeconfig update করো az aks get-credentials --name myAKSCluster
⚙️ Azure DevOps
CI/CD Pipeline Platform

Code থেকে Deployment পর্যন্ত সব এক জায়গায়। Repos, Pipelines, Boards, Artifacts, Test Plans — সম্পূর্ণ DevOps toolchain।

  • Azure Pipelines — YAML-based CI/CD
  • Azure Repos — Git repository hosting
  • Azure Boards — Scrum/Kanban project management
  • GitHub Actions-এর সাথেও integrate হয়
# azure-pipelines.yml trigger: - main pool: vmImage: 'ubuntu-latest' steps: - script: docker build -t myapp . - task: AzureWebAppContainer@1
🗃️ Blob Storage
Object Storage (S3-এর বিকল্প)

Azure-এর object storage। Unstructured data রাখো — images, videos, backups, logs। Hot/Cool/Archive তিনটি tier।

  • Hot = বারবার access, বেশি দাম
  • Cool = মাঝেমাঝে access, কম দাম
  • Archive = খুব কম access, সবচেয়ে সস্তা
  • Azure CDN দিয়ে globally distribute করো
🏢
Azure কেন Enterprise-এ জনপ্রিয়? কারণ বেশিরভাগ কর্পোরেট Windows + Office 365 + Active Directory চালায়। Azure এসব-এর সাথে seamlessly কাজ করে। Hybrid cloud (on-premise + cloud) সবচেয়ে ভালো Azure-এ।
04 — Serverless Architecture

Serverless — সার্ভার ভুলে যাও

Serverless মানে সার্ভার নেই এমন না — মানে তুমি সার্ভার নিয়ে ভাবো না। Cloud provider সব manage করে, তুমি শুধু কোড লেখো।

⚡ AWS Lambda
Function as a Service (FaaS)

Event-driven function। HTTP request, S3 upload, DynamoDB change, schedule — যেকোনো event-এ trigger হয়।

  • Cold Start: প্রথম request একটু slow
  • Timeout: সর্বোচ্চ ১৫ মিনিট run করতে পারে
  • Memory: ১২৮ MB থেকে ১০ GB পর্যন্ত
  • Layer দিয়ে shared code/library রাখো
# Python Lambda Function def handler(event, context): name = event['name'] return { 'statusCode': 200, 'body': f'হ্যালো, {name}!' }
⚡ Cloud Functions (GCP)
Google-এর FaaS

Lambda-র GCP সমতুল্য। HTTP trigger বা background event। Firebase-এর সাথে দারুণভাবে কাজ করে।

  • Gen 2 = Cloud Run-এর উপর build
  • Firebase Functions = Mobile app backend
  • Pub/Sub trigger = message queue থেকে
  • Concurrency: একটি instance একাধিক request handle করতে পারে

✅ ব্যবহার করো যখন:
  • Unpredictable traffic pattern
  • Event-driven processing
  • Cron jobs / scheduled tasks

  • Long-running processes (১৫ মিনিটের বেশি)
  • WebSocket / real-time connection
  • Heavy compute (ML training)

💰
Pay-per-execution মানে কী? EC2 চালু রাখলে ২৪/৭ পয়সা লাগে। Lambda শুধু function চললে পয়সা লাগে। ১ মিলিয়ন request পর্যন্ত AWS Free tier-এ ফ্রি! Small project-এ Lambda অনেক সস্তা।
05 — AWS Networking Deep Dive

VPC Networking — নেটওয়ার্কের ভেতরে

AWS Networking বোঝা মানে তোমার infrastructure কীভাবে internet-এর সাথে কথা বলে সেটা বোঝা। এটা ছাড়া production system বানানো অসম্ভব।

🏘️
VPC — Virtual Private Cloud
তোমার নিজস্ব private data center AWS-এ। একটি IP address range (CIDR) দিয়ে তৈরি। Region-wide — একটি region-এর সব AZ-তে ছড়িয়ে থাকে।
  • CIDR: 10.0.0.0/16 মানে 65,536 টি IP address
  • Default VPC প্রতিটি region-এ আগে থেকেই থাকে
10.0.0.0/16
🔲
Subnet — নেটওয়ার্ক ভাগ
VPC-কে ছোট ছোট ভাগে ভাগ করা। Public Subnet = internet থেকে accessible। Private Subnet = শুধু internal access। প্রতিটি Subnet একটি AZ-তে থাকে।
  • Public Subnet-এ: Load Balancer, Bastion Host, NAT Gateway
  • Private Subnet-এ: App server, Database, EKS nodes
10.0.1.0/24
🛡️
Security Group — Firewall
Instance-level firewall। Inbound ও Outbound rule define করো। Stateful — inbound allow করলে response automatically allow হয়।
# Security Group Rule উদাহরণ Inbound: Port 80 (HTTP) → Anywhere (0.0.0.0/0) Inbound: Port 443 (HTTPS) → Anywhere Inbound: Port 22 (SSH) → My IP only Outbound: All traffic → Anywhere (default)
sg-xxxxxxxx
🔀
NAT Gateway — Private থেকে Internet
Private subnet-এর server যখন internet-এ যেতে চায় (software update, API call) তখন NAT Gateway মাধ্যমে যায়। Internet থেকে কিন্তু private server-এ সরাসরি আসতে পারে না।
  • Public subnet-এ রাখতে হয়
  • Elastic IP লাগে (static public IP)
  • Managed service — highly available
nat-xxxxxxxx
🌐
Internet Gateway — VPC-র দরজা
VPC এবং Internet-এর মধ্যে সংযোগ। IGW ছাড়া কোনো resource internet-এ যেতে বা আসতে পারবে না। VPC-তে attach করতে হয়।
  • Horizontally scaled, redundant, highly available
  • প্রতি VPC-তে একটি মাত্র IGW
igw-xxxxxxxx
🗺️
Route Table — Traffic কোথায় যাবে
Traffic routing নির্ধারণ করে। প্রতিটি Subnet একটি Route Table-এর সাথে associated। Public subnet-এ IGW route থাকে, Private subnet-এ NAT route থাকে।
# Public Subnet Route Table Destination: 0.0.0.0/0 → Target: igw-xxxxx Destination: 10.0.0.0/16 → Target: local # Private Subnet Route Table Destination: 0.0.0.0/0 → Target: nat-xxxxx
rtb-xxxxxxxx
🔑
Golden Rule: Database কখনো Public Subnet-এ রাখবে না। সবসময় Private Subnet-এ। Web server Public-এ, App server Private-এ, Database আরো Private-এ। এটাকে বলে "Defense in Depth"।
