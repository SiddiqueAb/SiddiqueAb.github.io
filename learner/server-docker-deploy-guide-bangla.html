02
প্রজেক্ট ফোল্ডার স্ট্রাকচার
Docker ফাইলগুলো কোথায় কীভাবে রাখবেন
📁
কেন এই স্ট্রাকচার? Docker config ফাইলগুলো (Dockerfile, docker-compose.yml) প্রজেক্টের root-এ রাখলে সবকিছু একসাথে git-এ থাকে এবং যেকোনো সার্ভারে সহজে deploy হয়। docker/ folder-এ service-specific config রাখলে পরিষ্কার থাকে।
my-laravel-app/ ← আপনার Laravel প্রজেক্ট root ├── Dockerfile ← PHP-FPM Docker image তৈরির নির্দেশনা ├── docker-compose.yml ← সব service একসাথে পরিচালনা ├── .env ← environment variables (git-এ push করবেন না!) ├── .env.example ← template (এটা git-এ push করুন) ├── .dockerignore ← Docker build থেকে কী বাদ দেবে │ ├── docker/ ← Docker config ফাইলসমূহ │ ├── nginx/ │ │ └── default.conf ← Nginx web server configuration │ └── php/ │ └── php.ini ← PHP custom settings │ ├── app/ ← Laravel Models, Controllers ├── resources/js/ ← Vue 3 + Inertia components │ ├── Pages/ ← Inertia page components │ └── Components/ ← Reusable Vue components ├── public/ ← Web root (Nginx এটা serve করে) │ └── build/ ← Vite compiled assets (auto-generated) └── storage/ ← Logs, uploads, cache
03
Dockerfile তৈরি করুন
PHP 8.3-FPM + Node.js multi-stage build
🐳
Dockerfile কী এবং কেন? Dockerfile একটি রেসিপি — এটা Docker-কে বলে: "এই PHP version নাও, এই PHP extensions install করো, Composer দিয়ে packages নাও, কোড copy করো।" এর ফলে যেকোনো সার্ভারে হুবহু একই environment তৈরি হয় — "It works on my machine" সমস্যা চিরতরে শেষ।
⚡
Multi-Stage Build কেন ব্যবহার করছি? Stage 1: Node.js দিয়ে Vue/Vite build করি (JS → CSS, compiled JS তৈরি হয়)। Stage 2: PHP-FPM-এ শুধু compiled assets নিয়ে আসি। Final image-এ Node.js থাকে না — image size 700MB থেকে ~200MB হয়!

প্রজেক্টের root-এ Dockerfile নামে ফাইল তৈরি করুন:

Dockerfile
# ════════════════════════════════════════════════════
# STAGE 1: Node.js দিয়ে Frontend Build
# Vue 3 + Vite → public/build/ folder তৈরি করবে
# ════════════════════════════════════════════════════
FROM node:20-alpine AS frontend

WORKDIR /app

# package.json আগে copy করি — Docker layer cache ব্যবহার করে
# কোড না বদলালে npm install বারবার run করতে হবে না (build দ্রুত)
COPY package*.json ./
RUN npm ci --frozen-lockfile

# বাকি সব ফাইল copy করি
COPY . .

# Vite build চালাই — resources/js/* → public/build/ হয়
RUN npm run build


# ════════════════════════════════════════════════════
# STAGE 2: PHP 8.3-FPM Application
# Alpine Linux = হালকা, fast, production-ready
# ════════════════════════════════════════════════════
FROM php:8.3-fpm-alpine

# System-level dependencies install করি
# কেন? PHP extensions compile করতে এগুলো দরকার
RUN apk add --no-cache \
    git curl unzip libpng-dev libjpeg-turbo-dev \
    libwebp-dev freetype-dev libzip-dev \
    oniguruma-dev icu-dev linux-headers shadow

# PHP Extensions install করি
# pdo_mysql: MySQL connection
# mbstring: Bengali/Unicode string support
# gd: Image processing
# zip: ZIP file handling
# intl: Internationalization
# opcache: PHP code caching (performance)
RUN docker-php-ext-configure gd \
    --with-freetype --with-jpeg --with-webp \
  && docker-php-ext-install -j$(nproc) \
    pdo_mysql mbstring exif pcntl bcmath \
    gd zip intl opcache

# Redis PHP extension (session/cache এর জন্য)
RUN pecl install redis && docker-php-ext-enable redis

# Composer (PHP package manager) install
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

WORKDIR /var/www/html

# composer.json আগে copy → packages install → তারপর কোড copy
# এতে কোড বদলালে packages পুনরায় install হয় না (fast build)
COPY composer*.json ./
RUN composer install \
    --no-dev \
    --optimize-autoloader \
    --no-scripts \
    --no-interaction

# সম্পূর্ণ Laravel project copy করি
COPY . .

# Stage 1 থেকে compiled Vue/JS assets নিয়ে আসি
COPY --from=frontend /app/public/build ./public/build

# Production optimization: config, route, view cache করি
# এতে প্রতিটি request-এ file read কমে — app দ্রুত হয়
RUN php artisan config:cache \
 && php artisan route:cache \
 && php artisan view:cache

# storage ও bootstrap/cache এ write permission দিই
# www-data = Nginx/PHP-FPM এর user
RUN chown -R www-data:www-data /var/www/html \
 && chmod -R 775 storage bootstrap/cache

# PHP-FPM 9000 port-এ চলে
EXPOSE 9000
CMD ["php-fpm"]

04
docker-compose.yml তৈরি করুন
PHP + Nginx + MySQL + Redis একসাথে পরিচালনা
🔗
Docker Compose কেন? আমাদের app-এ 4টি parts: PHP app, Nginx web server, MySQL database, Redis cache। Docker Compose এদের সবাইকে একটি কমান্ডে (docker compose up -d) চালু করে। এরা নিজেদের নাম দিয়ে ডাকতে পারে (IP মনে রাখতে হয় না)।
docker-compose.yml
version: '3.9'

services:

  # ── PHP Laravel Application ──────────────────────
  app:
    build:
      context: .                   # Dockerfile এই folder-এ আছে
      dockerfile: Dockerfile
    container_name: laravel_app
    restart: unless-stopped          # Server restart হলে auto চালু হবে
    volumes:
      # storage host-এ রাখি, container delete হলেও files থাকবে
      - ./storage:/var/www/html/storage
    environment:
      - APP_ENV=production
      - APP_KEY=${APP_KEY}          # .env থেকে নেবে
      - DB_HOST=mysql               # service নাম = hostname
      - DB_DATABASE=${DB_DATABASE}
      - DB_USERNAME=${DB_USERNAME}
      - DB_PASSWORD=${DB_PASSWORD}
      - REDIS_HOST=redis
      - CACHE_DRIVER=redis
      - SESSION_DRIVER=redis
    depends_on:
      mysql:
        condition: service_healthy  # MySQL ready হলে তারপর start
      redis:
        condition: service_started
    networks:
      - app-network

  # ── Nginx Web Server ──────────────────────────────
  nginx:
    image: nginx:1.25-alpine
    container_name: laravel_nginx
    restart: unless-stopped
    ports:
      - "80:80"                      # সার্ভারের port 80 → Nginx 80
      - "443:443"                    # SSL এর জন্য (পরে setup করুন)
    volumes:
      # Nginx config mount করি
      - ./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
      # Public folder static files serve করতে
      - ./public:/var/www/html/public:ro
      # Uploaded files serve করতে
      - ./storage/app/public:/var/www/html/storage/app/public:ro
    depends_on:
      - app
    networks:
      - app-network

  # ── MySQL 8 Database ──────────────────────────────
  mysql:
    image: mysql:8.0
    container_name: laravel_mysql
    restart: unless-stopped
    environment:
      MYSQL_DATABASE: ${DB_DATABASE}
      MYSQL_USER: ${DB_USERNAME}
      MYSQL_PASSWORD: ${DB_PASSWORD}
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
    volumes:
      # Named volume — container delete হলেও data থাকবে!
      - mysql_data:/var/lib/mysql
    healthcheck:                      # MySQL ready কিনা check করে
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - app-network

  # ── Redis Cache & Session ─────────────────────────
  redis:
    image: redis:7-alpine
    container_name: laravel_redis
    restart: unless-stopped
    # appendonly yes = crash হলেও data recover হবে
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    networks:
      - app-network

# Persistent volumes — data সার্ভারে থাকে
volumes:
  mysql_data:
    driver: local
  redis_data:
    driver: local

# Internal network — services নিজেদের মধ্যে কথা বলে
# বাইরে থেকে এই network দেখা যায় না (secure)
networks:
  app-network:
    driver: bridge
05
Nginx কনফিগারেশন তৈরি করুন
Web server কীভাবে request handle করবে
⚡
Nginx কীভাবে কাজ করে? Browser request করে → Nginx receive করে → Static file (.jpg, .css, .js) হলে নিজেই দিয়ে দেয় (দ্রুত) → PHP (.php) হলে PHP-FPM (app container) এ পাঠায় → PHP process করে → Nginx browser-এ পাঠায়। Inertia.js-এর জন্য সব route index.php-এ যাবে।

ফাইল তৈরি করুন: docker/nginx/default.conf

docker/nginx/default.conf
server {
    listen 80;
    server_name _;  # _ মানে যেকোনো domain/IP accept করবে

    # Laravel-এর public folder = web document root
    # শুধু এই folder external access পায়
    root /var/www/html/public;
    index index.php index.html;

    # File upload maximum size (100MB)
    client_max_body_size 100M;

    # Gzip compression — response ছোট হয়, page দ্রুত load হয়
    gzip on;
    gzip_vary on;
    gzip_min_length 1000;
    gzip_types
        text/plain text/css text/xml text/javascript
        application/json application/javascript application/xml+rss
        application/atom+xml image/svg+xml;

    # Static assets (CSS, JS, images) — browser 1 বছর cache করবে
    # Vite-generated files content hash-সহ থাকে, cache safe
    location ~* \.(css|js|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot|map)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        try_files $uri =404;
    }

    # Laravel routing — এটাই সবচেয়ে গুরুত্বপূর্ণ line
    # try_files: file আছে? → দাও। folder আছে? → দাও। নাহলে index.php-এ route করো
    # Inertia.js এই ভাবেই Vue pages serve করে
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    # PHP files → PHP-FPM (app container) এ পাঠাও
    # fastcgi_pass app:9000 → 'app' service এর 9000 port
    location ~ \.php$ {
        fastcgi_pass app:9000;
        fastcgi_index index.php;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
        fastcgi_read_timeout 300;  # Long-running requests এর জন্য
        fastcgi_buffer_size 128k;
        fastcgi_buffers 4 256k;
    }

    # Hidden files (.htaccess, .git ইত্যাদি) block করি
    location ~ /\.(?!well-known).* {
        deny all;
        return 404;
    }

    # Security headers — basic protection
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # Nginx error/access log
    error_log /var/log/nginx/error.log;
    access_log /var/log/nginx/access.log;
}
06
PHP Configuration (php.ini)
Production-এর জন্য PHP settings optimize করুন

ফাইল তৈরি করুন: docker/php/php.ini

docker/php/php.ini
; ── Production PHP Settings ──

; Error display বন্ধ (logs-এ যাবে, browser-এ না)
display_errors = Off
log_errors = On
error_log = /var/log/php_errors.log

; Memory limit — Laravel + Vue SSR এর জন্য 256MB ভালো
memory_limit = 256M

; File upload settings
upload_max_filesize = 50M
post_max_size = 55M

; Request timeout
max_execution_time = 300
max_input_time = 300

; OPcache — PHP bytecode cache করে, app 3-5x দ্রুত হয়
opcache.enable = 1
opcache.memory_consumption = 256
opcache.max_accelerated_files = 20000
opcache.revalidate_freq = 0        ; Production: 0 (cache always)
opcache.validate_timestamps = 0    ; Production: 0 (faster)

; Date timezone
date.timezone = Asia/Dhaka

এখন Dockerfile-এ php.ini যোগ করতে এই line add করুন (COPY . . এর পর):

Dockerfile-এ যোগ করুন
# custom php.ini copy করি
COPY docker/php/php.ini /usr/local/etc/php/conf.d/custom.ini

সার্ভারে SSH দিয়ে Connect করুন
Windows Git Bash থেকে Hetzner VPS-এ ঢুকুন
🔑
নিরাপত্তা সতর্কতা Root password প্রথম login এর পরেই বদলে ফেলুন। SSH key setup করুন password-এর বদলে। Production server-এ root login বন্ধ করে দেওয়া উচিত।
Windows: Git Bash বা PowerShell খুলুন
# SSH দিয়ে সার্ভারে connect করুন
# -p 22 = SSH port (default 22)
ssh -p 22 root@178.104.78.234

# প্রথমবার এই message আসবে:
# The authenticity of host '178.104.78.234' can't be established.
# ECDSA key fingerprint is SHA256:xxxxx
# Are you sure you want to continue connecting? → "yes" টাইপ করুন

# Password দিন (টাইপ করলে দেখা যাবে না — এটা normal!)
# Successfully connected হলে দেখবেন: root@ubuntu-server:~#

সার্ভারে ঢুকে সিস্টেম আপডেট ও Firewall সেটআপ করুন:

Server Terminal (Ubuntu 22.04)
# সিস্টেম update করুন — security patch পেতে জরুরি
apt update && apt upgrade -y

# দরকারি tools install করুন
apt install -y curl git wget unzip nano htop ufw fail2ban

# ── Firewall Setup ──
# শুধু দরকারি ports খুলি — বাকি সব বন্ধ থাকবে
ufw default deny incoming    # সব incoming block
ufw default allow outgoing   # সব outgoing allow
ufw allow 22/tcp              # SSH (এটা না দিলে locked out!)
ufw allow 80/tcp              # HTTP
ufw allow 443/tcp             # HTTPS (SSL)
ufw --force enable            # Firewall চালু করুন
ufw status verbose            # Status দেখুন

# Swap memory তৈরি করুন (VPS-এ RAM কম হলে দরকার)
fallocate -l 2G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile
# Boot-এ auto enable করতে:
echo '/swapfile none swap sw 0 0' >> /etc/fstab
08
সার্ভারে Docker Engine ইনস্টল করুন
Official Docker repository থেকে latest version
🐳
Server-এ Docker Desktop লাগবে না Windows-এ Docker Desktop হলো GUI সহ। Ubuntu server-এ শুধু Docker Engine (CLI) + Docker Compose plugin লাগে। এটা অনেক হালকা এবং production-এর জন্য উপযুক্ত।
Server Terminal — Docker Official Install
# পুরনো Docker version থাকলে remove করুন
apt remove -y docker docker-engine docker.io containerd runc 2>/dev/null || true

# ── Step 1: Docker-এর GPG key যোগ করি ──
# কেন? Download হওয়া packages authentic কিনা verify করতে
curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
  | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# ── Step 2: Docker official repository যোগ করি ──
echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" \
  | tee /etc/apt/sources.list.d/docker.list > /dev/null

# ── Step 3: Docker install করি ──
apt update
apt install -y \
  docker-ce \            # Docker Engine
  docker-ce-cli \        # CLI tools
  containerd.io \        # Container runtime
  docker-buildx-plugin \ # Advanced build
  docker-compose-plugin  # Docker Compose v2

# ── Step 4: Verify ──
docker --version
# Expected: Docker version 26.x.x, build xxxxxxx

docker compose version
# Expected: Docker Compose version v2.x.x

# ── Step 5: Docker service auto-start enable ──
systemctl enable docker
systemctl start docker

# ── Test: hello-world container চালান ──
docker run --rm hello-world
# "Hello from Docker!" দেখালে সফল!

Git clone বা SCP — দুটো পদ্ধতি

Server Terminal
cd /var/www/myapp

# Public GitHub repo:
git clone https://github.com/yourusername/your-laravel-app.git .

# Private repo (Personal Access Token ব্যবহার করুন):
git clone https://YOUR_TOKEN@github.com/yourusername/your-laravel-app.git .
# Token তৈরি: GitHub → Settings → Developer settings → Personal access tokens

# Specific branch clone করতে:
git clone -b develop https://github.com/yourusername/repo.git .
পদ্ধতি B SCP দিয়ে Windows থেকে সরাসরি Copy করুন
কখন ব্যবহার করবেন? Git repo না থাকলে এই পদ্ধতি ব্যবহার করুন। Windows-এর Git Bash থেকে চালান।
Windows Git Bash (আপনার PC-তে)
# Windows থেকে সার্ভারে project copy করুন
# -r = recursive (subfolder সহ)
scp -P 22 -r /c/Users/YourName/projects/my-laravel-app/* \
    root@178.104.78.234:/var/www/myapp/

# Windows path → Git Bash path:
# C:\Users\YourName\projects → /c/Users/YourName/projects
# C:\xampp\htdocs\myapp → /c/xampp/htdocs/myapp

# node_modules ও vendor বাদ দিয়ে copy করতে rsync ব্যবহার করুন:
rsync -avz --progress \
  --exclude='node_modules' \
  --exclude='vendor' \
  --exclude='.env' \
  -e "ssh -p 22" \
  /c/Users/YourName/projects/my-laravel-app/ \
  root@178.104.78.234:/var/www/myapp/
