▸ 01 / FUNDAMENTALS

API কী এবং REST কীভাবে কাজ করে?

একজন senior engineer হিসেবে শুধু "API বানাতে পারা" যথেষ্ট না — বুঝতে হবে কেন এভাবে বানাচ্ছি।

📡 API কী?

API (Application Programming Interface) হলো দুটো সিস্টেমের মধ্যে কথা বলার চুক্তি। রেস্টুরেন্টের waiter-এর মতো — তুমি order করো (request), সে রান্নাঘর থেকে খাবার এনে দেয় (response)। তুমি রান্নাঘরের ভেতরে যাও না।

🏗️ REST কী?

REST (Representational State Transfer) হলো একটা architectural style — নিয়মের সেট। HTTP ব্যবহার করে, stateless, URL দিয়ে resource চেনা যায়, JSON/XML দিয়ে data আদান-প্রদান। Roy Fielding ২০০০ সালে define করেন।

1️⃣ Stateless

প্রতিটা request সম্পূর্ণ স্বতন্ত্র। Server আগের request মনে রাখবে না। Authentication token প্রতিবার পাঠাতে হবে।

2️⃣ Uniform Interface

একই নিয়মে সব resource access। GET /users, GET /products — একই pattern follow করে।

3️⃣ Client-Server

Frontend ও Backend আলাদা। Backend শুধু data দেয়, Frontend সেটা দেখায়। আলাদা scale করা যায়।

4️⃣ Cacheable

Response cache করা যাবে। GET /products বারবার না চেয়ে cache থেকে দাও। Server load কমে।

5️⃣ Layered System

Client জানে না সে directly server-এ কথা বলছে নাকি proxy/load balancer-এ। Layer add করা যায়।

6️⃣ Code on Demand (optional)

Server চাইলে executable code (JavaScript) পাঠাতে পারে। Optional constraint।

▸ 02 / DESIGN PRINCIPLES

API Design — Senior Engineer-এর নিয়ম

ভালো API design একবার শিখলে কোড লেখার চেয়ে বেশি কাজে আসে।

📝 URL Naming Convention — সঠিক vs ভুল
❌ ভুল (Beginner)✅ সঠিক (Senior)কারণ
/getUsers/usersURL noun হবে, verb না। HTTP method-ই verb।
/createNewProduct/productsPOST /products মানেই create।
/user_list/userslowercase, hyphen-case, plural noun।
/Users/GetById/5/users/5ID সরাসরি path-এ।
/deleteUser?id=5/users/5DELETE /users/5 — method বলে দেয়।
/user/5/getOrders/users/5/ordersNested resource দিয়ে relationship বোঝাও।
/searchProducts?q=laptop/products?search=laptopFilter query parameter হবে।
🏗️ Resource Hierarchy Design

সম্পর্ক বোঝানোর নিয়ম:

# একটা resource
GET    /users          # সব users
GET    /users/5        # user #5
POST   /users          # নতুন user
PUT    /users/5        # পুরো update
PATCH  /users/5        # আংশিক
DELETE /users/5        # মুছো

# Nested resources
GET    /users/5/orders # user 5-এর orders
GET    /orders/3/items # order 3-এর items

# ⚠️ Nesting ৩ level-এর বেশি না!
# ❌ /users/5/orders/3/items/7/reviews
# ✅ /order-items/7/reviews
🔍 Query Parameters — কখন কী?
# Filtering — কী দেখাবো?
GET /products?category=mobile&brand=samsung

# Sorting — কীভাবে সাজাবো?
GET /products?sort=price&order=asc

# Pagination — কত করে দেখাবো?
GET /products?page=2&per_page=20

# Searching — কী খুঁজছি?
GET /products?search=iphone+15

# Field selection — কোন field দরকার?
GET /users?fields=id,name,email

# সব একসাথে
GET /products?category=mobile
  &sort=price&order=asc
  &page=1&per_page=10
  &search=samsung
🎯

Senior Engineer Rule: URL একটা resource-এর address। যেমন ঘরের ঠিকানা দেয়া হয় — কী করতে হবে সেটা বলার দরকার নেই ঠিকানায়। HTTP method সেটা বলে। URL সবসময় lowercase, hyphen (-) দিয়ে শব্দ আলাদা, plural noun ব্যবহার করো।

// app/Http/Resources/ProductResource.php
class ProductResource extends JsonResource {
    public function toArray($request): array {
        return [
            'id'          => $this->id,
            'name'        => $this->name,
            'slug'        => $this->slug,
            'price'       => [
                'amount'   => $this->price,
                'currency' => 'BDT',
                'formatted'=> '৳' . number_format($this->price, 2),
            ],
            'stock'       => $this->stock,
            'in_stock'    => $this->stock > 0,
            'category'    => new CategoryResource($this->whenLoaded('category')),
            'created_at'  => $this->created_at->toISOString(),
        ];
    }
}

// app/Http/Controllers/Api/V1/ProductController.php
class ProductController extends BaseApiController {

    public function index(ProductIndexRequest $request): JsonResponse {
        $products = Product::query()
            ->with(['category'])                             // Eager load
            ->when($request->filled('search'), fn($q) =>
                $q->where('name', 'like', "%{$request->search}%"))
            ->when($request->filled('category'), fn($q) =>
                $q->whereHas('category', fn($c) =>
                    $c->where('slug', $request->category)))
            ->orderBy($request->get('sort', 'created_at'),
                     $request->get('order', 'desc'))
            ->paginate($request->get('per_page', 15));

        return $this->successResponse(
            ProductResource::collection($products),
            'Products retrieved successfully'
        );
    }

    public function store(StoreProductRequest $request): JsonResponse {
        $product = Product::create($request->validated());
        return $this->successResponse(
            new ProductResource($product),
            'Product created', 201
        );
    }
}

// app/Http/Controllers/Api/BaseApiController.php
abstract class BaseApiController extends Controller {

    protected function successResponse($data, $msg = 'Success', $code = 200): JsonResponse {
        return response()->json([
            'success' => true,
            'message' => $msg,
            'data'    => $data,
        ], $code);
    }

    protected function errorResponse($msg, $code = 400, $errors = []): JsonResponse {
        return response()->json([
            'success'    => false,
            'message'    => $msg,
            'errors'     => $errors,
            'trace_id'   => 'req_' . substr(uniqid(), -8),
        ], $code);
    }
}
▸ 05 / AUTHENTICATION

API Authentication — JWT vs Sanctum vs OAuth

কোনটা কখন ব্যবহার করবে এবং কীভাবে সঠিকভাবে implement করবে।

কখন: Microservice, multiple service

Third-party app-কে access দিতে হলে। Google Login, Facebook Login-এর মতো।

কখন: Public API, 3rd party access

// AuthController.php — Senior Level Implementation
class AuthController extends BaseApiController {

    public function login(LoginRequest $request): JsonResponse {
        // Rate limiting — brute force রোধে
        $key = 'login:' . $request->ip();
        if (RateLimiter::tooManyAttempts($key, 5)) {
            $seconds = RateLimiter::availableIn($key);
            return $this->errorResponse(
                "Too many attempts. Retry in {$seconds}s", 429
            );
        }

        if (! Auth::attempt($request->only('email', 'password'))) {
            RateLimiter::hit($key, 300); // 5 minutes
            return $this->errorResponse('Invalid credentials', 401);
        }

        RateLimiter::clear($key); // সফল হলে reset

        $user  = Auth::user();
        $token = $user->createToken(
            'api-token',
            ['*'],              // abilities/scopes
            now()->addDays(30)  // expires in 30 days
        );

        return $this->successResponse([
            'user'         => new UserResource($user),
            'access_token' => $token->plainTextToken(),
            'token_type'   => 'Bearer',
            'expires_at'   => now()->addDays(30)->toISOString(),
        ], 'Login successful');
    }

    public function logout(Request $request): JsonResponse {
        $request->user()->currentAccessToken()->delete();
        return $this->successResponse([], 'Logged out successfully');
    }
}

// LoginRequest.php — Validation
class LoginRequest extends FormRequest {
    public function rules(): array {
        return [
            'email'    => ['required', 'email', 'max:255'],
            'password' => ['required', 'string', 'min:8'],
        ];
    }
}
⚠️

Security Warning: Token কখনো URL-এ পাঠাবে না (?token=xyz)। সবসময় Authorization: Bearer TOKEN header ব্যবহার করো। Token localStorage-এ না রেখে httpOnly cookie-তে রাখো XSS attack থেকে বাঁচতে।

Laravel · Global Exception Handler
// app/Exceptions/Handler.php
class Handler extends ExceptionHandler {

    public function register(): void {

        // Validation errors
        $this->renderable(function (ValidationException $e, $req) {
            if ($req->expectsJson()) {
                return response()->json([
                    'success'    => false,
                    'message'    => 'Validation failed',
                    'errors'     => $e->errors(),
                    'error_code' => 'VALIDATION_ERROR',
                ], 422);
            }
        });

        // 404 Not Found
        $this->renderable(function (ModelNotFoundException $e, $req) {
            if ($req->expectsJson()) {
                $model = class_basename($e->getModel());
                return response()->json([
                    'success'    => false,
                    'message'    => "{$model} not found",
                    'error_code' => 'NOT_FOUND',
                ], 404);
            }
        });

        // 401 Unauthenticated
        $this->renderable(function (AuthenticationException $e, $req) {
            if ($req->expectsJson()) {
                return response()->json([
                    'success'    => false,
                    'message'    => 'Unauthenticated. Please login.',
                    'error_code' => 'UNAUTHENTICATED',
                ], 401);
            }
        });

        // 403 Forbidden
        $this->renderable(function (AuthorizationException $e, $req) {
            if ($req->expectsJson()) {
                return response()->json([
                    'success'    => false,
                    'message'    => 'You do not have permission.',
                    'error_code' => 'FORBIDDEN',
                ], 403);
            }
        });

        // 500 — Production-এ details hide করো!
        $this->renderable(function (Throwable $e, $req) {
            if ($req->expectsJson()) {
                Log::error($e); // log করো কিন্তু expose করো না
                return response()->json([
                    'success'    => false,
                    'message'    => app()->isProduction()
                        ? 'Something went wrong'
                        : $e->getMessage(),
                    'error_code' => 'SERVER_ERROR',
                ], 500);
            }
        });
    }
}
▸ 08 / PAGINATION

Pagination — ১ মিলিয়ন record একসাথে দিও না

Without pagination, একটা API call পুরো server ডোবাতে পারে।

📄 Offset Pagination Simple

page=2&per_page=20 মানে প্রথম 20টা skip করে পরের 20টা দাও।

⚠️

সমস্যা: বড় dataset-এ OFFSET ধীর হয়। page=1000&per_page=20 মানে ২০,০০০ row skip করা — expensive!

⚡ Cursor Pagination Scalable

শেষ দেখা item-এর ID/cursor ব্যবহার করে পরের page আনে। Large dataset-এ fast।

✅

Best for: Infinite scroll, large dataset (Twitter/Instagram feed)। Index ব্যবহার করে — O(log n)!

Laravel · Both Pagination Types
// ১. Offset Pagination — সাধারণ ব্যবহারে

            'last_page'    => $products->lastPage(),
            'per_page'     => $products->perPage(),

// app/Providers/RouteServiceProvider.php
use Illuminate\Cache\RateLimiting\Limit;

protected function configureRateLimiting(): void {

    // Public API
    RateLimiter::for('api', function (Request $req) {
        if ($req->user()) {
            // Authenticated: user plan অনুযায়ী
            $limit = match($req->user()->plan) {
                'premium' => 1000,
                'basic'   => 300,
                default   => 120,
            };
            return Limit::perMinute($limit)
                        ->by($req->user()->id);
        }
        // Guest: IP-based
        return Limit::perMinute(60)->by($req->ip());
    });

    // Login endpoint — brute force রোধে কঠোর
    RateLimiter::for('login', function (Request $req) {
        return Limit::perMinutes(5, 5)  // 5 min এ 5 attempt
                    ->by($req->input('email') . $req->ip())
                    ->response(function () {
                        return response()->json([
                            'success'    => false,
                            'message'    => 'Too many login attempts. Try again in 5 minutes.',
                            'error_code' => 'RATE_LIMITED',
                        ], 429);
                    });
    });

    // Search endpoint — DB heavy, কম limit
    RateLimiter::for('search', function (Request $req) {
        return Limit::perMinute(30)->by($req->user()?->id ?? $req->ip());
    });
}

▸ 11 / SECURITY

API Security — হামলা থেকে বাঁচার পূর্ণ গাইড

Security একটা একটা layer — কোনো একটা না থাকলে পুরো system vulnerable।

🔒 HTTPS Only
সব API call HTTPS-এ হতে হবে। HTTP redirect করো। TLS 1.2+ use করো।
💉 SQL Injection
Raw query avoid করো। Laravel Eloquent/Query Builder parameterized query ব্যবহার করে।
🌐 CORS
শুধু allowed origin-এর request accept করো। Wildcard (*) production-এ ব্যবহার করো না।
📦 Mass Assignment
Model-এ $fillable define করো। $guarded = [] কখনো করো না — সব field update হবে।
📧 Input Validation
সব input validate করো। Never trust client data. FormRequest class ব্যবহার করো।
🔑 Secret Management
.env file-এ secret রাখো। .env কখনো git-এ push করো না। AWS Secrets Manager ব্যবহার করো।
🚫 Sensitive Data
Password, card number API response-এ দিও না। Log-এও না। Mask করো।
🔐 Authorization
Authentication ≠ Authorization। Login করলেই সব data দেওয়া যাবে না। Policy ব্যবহার করো।
📝 Audit Log
কে কখন কী করলো সব log করো। Security breach হলে trace করা যাবে।

// ২. Authorization — Policy ব্যবহার করো
class OrderPolicy {
    public function view(User $user, Order $order): bool {
        // শুধু নিজের order দেখতে পারবে, অথবা admin
        return $user->id === $order->user_id || $user->isAdmin();

    public function delete(User $user, Order $order): bool {
        return $user->isAdmin(); // শুধু admin delete করতে পারবে
    }
}

// Nginx config:

// ══════════════════════════════════════════════
// ২. FIELD SELECTION — অপ্রয়োজনীয় data পাঠাবে না
// ══════════════════════════════════════════════
public function index(Request $request): JsonResponse {
    $fields  = $request->get('fields'); // ?fields=id,name,price
    $allowed = ['id', 'name', 'price', 'stock', 'category'];

    $select  = $fields
        ? array_intersect(explode(',', $fields), $allowed)
        : ['*'];

    $products = Product::select($select)->paginate(20);
    // GET /products?fields=id,name,price → শুধু ৩টা field
    // 100 products × 50 bytes vs 100 products × 500 bytes = 10x কম!
    return $this->successResponse($products);
}

// ══════════════════════════════════════════════
// ৩. CONDITIONAL REQUESTS — Change না হলে পাঠাবে না
// ══════════════════════════════════════════════
public function show($id, Request $request): Response|JsonResponse {
    $product = Product::findOrFail($id);
    $etag    = md5($product->updated_at . $product->id);

    // Client পাঠিয়েছে: If-None-Match: "abc123"
    if ($request->header('If-None-Match') === $etag) {
        return response('', 304); // 304 Not Modified — no data!
        // Bandwidth = 0! শুধু header
    }

    return response()->json(new ProductResource($product))
        ->withHeaders(['ETag' => $etag]);
}

// ══════════════════════════════════════════════
// ৪. LAZY LOADING — Unused data load করো না
// ══════════════════════════════════════════════
public function products(Request $request): JsonResponse {
    $query = Product::query();

    // Client যা চায় তাই include করো
    if ($request->has('include')) {
        $allowed = ['category', 'reviews', 'images'];
        $include = array_intersect(
            explode(',', $request->include), $allowed
        );
        $query->with($include);
    }
    // GET /products → শুধু product
    // GET /products?include=category → product + category
    // GET /products?include=category,reviews → সব
    return $this->successResponse($query->paginate(20));
}
