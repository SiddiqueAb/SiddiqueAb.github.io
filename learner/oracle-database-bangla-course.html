BeginnerTheory

Oracle Database কী?

Oracle Database হলো বিশ্বের সবচেয়ে শক্তিশালী ও জনপ্রিয় Relational Database Management System (RDBMS)। এটি Oracle Corporation দ্বারা তৈরি এবং ১৯৭৭ সাল থেকে ব্যবহৃত হয়ে আসছে। ব্যাংক, হাসপাতাল, সরকারি প্রতিষ্ঠান, বড় কোম্পানি — সবাই Oracle ব্যবহার করে কারণ এটি অত্যন্ত নির্ভরযোগ্য, দ্রুত ও সুরক্ষিত।

সহজ উদাহরণ: একটি বড় ব্যাংকের কথা ভাবুন — প্রতিদিন কোটি কোটি লেনদেন হয়। একটিও ভুল হলে বিপদ। Oracle Database এই কাজটি নিখুঁতভাবে করে — কোনো ডেটা হারায় না, কোনো ভুল হয় না।

Oracle আর্কিটেকচার

Oracle-এর ভেতরের কাঠামো — Instance, Database, Memory, Process

IntermediateArchitecture

Oracle Database বোঝার জন্য এর আর্কিটেকচার জানা অত্যন্ত জরুরি। এটি দুটি প্রধান অংশে বিভক্ত: Oracle Instance (মেমোরি + প্রক্রিয়া) এবং Oracle Database (ডিস্কের ফাইল)।

SGA-এর উপাদানগুলো

উপাদানকাজগুরুত্ব
Shared PoolSQL কোড ও ডেটা ডিকশনারি ক্যাশ করেপুনরায় পার্সিং এড়ানো
Buffer Cacheডিস্ক থেকে পড়া ডেটা ব্লক মেমোরিতে রাখেI/O কমানো
Redo Log Bufferপরিবর্তনের লগ অস্থায়ীভাবে রাখেরিকভারি নিশ্চিত করা
Large Poolবড় মেমোরি অপারেশনের জন্যRMAN ব্যাকআপ

Background Processes

📝 DBWn (Database Writer)

Buffer Cache থেকে পরিবর্তিত ডেটা ডিস্কে লেখে।

✏️ LGWR (Log Writer)

Redo Log Buffer থেকে Redo Log ফাইলে লেখে।

🔄 CKPT (Checkpoint)

🔧 SMON (System Monitor)

👤 PMON (Process Monitor)

ব্যর্থ ব্যবহারকারী প্রক্রিয়া পরিষ্কার করে।

🔒 ARCH (Archiver)

পুরোনো Redo Log ফাইল আর্কাইভ করে।

04

DDL — ডেটা সংজ্ঞায়ন ভাষা

টেবিল, সীমাবদ্ধতা, সিকোয়েন্স তৈরি ও পরিবর্তন

BeginnerSQL

DDL (Data Definition Language) হলো সেই SQL কমান্ড যা ডেটাবেসের কাঠামো তৈরি ও পরিবর্তন করে। মূল DDL কমান্ড: CREATE, ALTER, DROP, TRUNCATE, RENAME।

ডেটা টাইপসমূহ

ডেটা টাইপবিবরণউদাহরণ
VARCHAR2(n)পরিবর্তনশীল দৈর্ঘ্যের টেক্সট (সর্বোচ্চ ৪০০০)নাম, ঠিকানা
CHAR(n)নির্দিষ্ট দৈর্ঘ্যের টেক্সটলিঙ্গ ('M','F')
NUMBER(p,s)সংখ্যা (p=মোট সংখ্যা, s=দশমিক)বেতন, বয়স
DATEতারিখ ও সময়জন্মতারিখ
TIMESTAMPসময়সহ সূক্ষ্ম তারিখলেনদেনের সময়
CLOBবড় টেক্সট (৪GB পর্যন্ত)নিবন্ধ, মন্তব্য
BLOBবাইনারি ডেটা (ছবি, ফাইল)ফটো, পিডিএফ

টেবিল তৈরি — CREATE TABLE

কর্মচারী ডেটাবেস তৈরি
SQL — DDL
-- বিভাগ টেবিল
CREATE TABLE department (
    dept_id     NUMBER(5)       PRIMARY KEY,
    dept_name   VARCHAR2(100)   NOT NULL,
    location    VARCHAR2(150),
    budget      NUMBER(12,2)    DEFAULT 0
);

-- কর্মচারী টেবিল
CREATE TABLE employee (
    emp_id      NUMBER(6)       PRIMARY KEY,
    emp_name    VARCHAR2(100)   NOT NULL,
    email       VARCHAR2(200)   UNIQUE,
    phone       VARCHAR2(20),
    hire_date   DATE            DEFAULT SYSDATE,
    salary      NUMBER(10,2)    CHECK (salary > 0),
    dept_id     NUMBER(5)       REFERENCES department(dept_id),
    gender      CHAR(1)         CHECK (gender IN ('M', 'F'))
);

-- পণ্য টেবিল
CREATE TABLE product (
    product_id   NUMBER(8)      PRIMARY KEY,
    product_name VARCHAR2(200)  NOT NULL,
    category     VARCHAR2(50),
    price        NUMBER(10,2)   NOT NULL,
    stock_qty    NUMBER(8)      DEFAULT 0,
    created_at   TIMESTAMP     DEFAULT SYSTIMESTAMP
);

সীমাবদ্ধতা (Constraints)

🔑 PRIMARY KEY

প্রতিটি সারিকে অনন্যভাবে চিহ্নিত করে। NOT NULL + UNIQUE একসাথে।

🔗 FOREIGN KEY

অন্য টেবিলের সাথে সম্পর্ক তৈরি করে। রেফারেন্সিয়াল ইন্টেগ্রিটি নিশ্চিত করে।

🚫 NOT NULL

কলামে NULL মান রাখা যাবে না। বাধ্যতামূলক ক্ষেত্রের জন্য।

✅ CHECK

মানের বৈধতা পরীক্ষা করে। যেমন: salary > 0।

🔄 UNIQUE

সব মান আলাদা হতে হবে। NULL মান একাধিক হতে পারে।

📌 DEFAULT

কোনো মান না দিলে ডিফল্ট মান ব্যবহার হবে।

ALTER TABLE — টেবিল পরিবর্তন

MERGE — INSERT + UPDATE একসাথে (Upsert)

MERGE উদাহরণ
SQL
-- target টেবিলে source টেবিল থেকে ডেটা মার্জ করা
MERGE INTO employee target
USING new_employee_data source
ON (target.emp_id = source.emp_id)
WHEN MATCHED THEN
    UPDATE SET
        target.salary   = source.salary,
        target.dept_id  = source.dept_id
WHEN NOT MATCHED THEN
    INSERT (emp_id, emp_name, salary, dept_id)
    VALUES (source.emp_id, source.emp_name, source.salary, source.dept_id);
-- মিল থাকলে UPDATE, না থাকলে INSERT — দুটো কাজ একসাথে!
⚠️
DELETE vs TRUNCATEDELETE রোলব্যাক করা যায় এবং WHERE ক্লজ ব্যবহার করা যায়। TRUNCATE অনেক দ্রুত কিন্তু রোলব্যাক করা যায় না এবং সব ডেটা মুছে ফেলে।

FROM employee;

JOIN অপারেশন

একাধিক টেবিল থেকে ডেটা একসাথে আনা

ImportantSQL

JOIN হলো একাধিক টেবিলকে একত্রিত করার পদ্ধতি। এটি SQL-এর সবচেয়ে শক্তিশালী বৈশিষ্ট্যগুলোর একটি। Oracle-এ বিভিন্ন ধরনের JOIN আছে।

সব ধরনের JOIN উদাহরণ
SQL
-- INNER JOIN: উভয় টেবিলে মিল আছে এমন ডেটা
SELECT e.emp_name, e.salary, d.dept_name
FROM   employee   e
INNER JOIN department d ON e.dept_id = d.dept_id;

-- LEFT JOIN: employee-এর সব ডেটা (dept না থাকলেও)
SELECT e.emp_name, NVL(d.dept_name, 'বিভাগ নেই') dept
FROM   employee   e
LEFT JOIN department d ON e.dept_id = d.dept_id;

-- RIGHT JOIN: department-এর সব ডেটা (কর্মচারী না থাকলেও)
SELECT NVL(e.emp_name,'কর্মচারী নেই'), d.dept_name
FROM   employee   e
RIGHT JOIN department d ON e.dept_id = d.dept_id;

-- FULL OUTER JOIN: উভয়ের সব ডেটা
SELECT e.emp_name, d.dept_name
FROM   employee   e
FULL OUTER JOIN department d ON e.dept_id = d.dept_id;

-- SELF JOIN: একই টেবিলের সাথে নিজের JOIN
-- কর্মচারী ও তার ম্যানেজারের নাম
SELECT
    e.emp_name    AS employee,
    m.emp_name    AS manager
FROM      employee e
LEFT JOIN employee m ON e.manager_id = m.emp_id;

-- তিনটি টেবিল JOIN করা
SELECT
    e.emp_name,
    d.dept_name,
    p.project_name
FROM       employee    e
JOIN       department  d ON e.dept_id    = d.dept_id
JOIN       emp_project p ON e.emp_id     = p.emp_id;

সাবকোয়েরি ও CTE

কোয়েরির ভেতরে কোয়েরি এবং WITH ক্লজ

IntermediateSQL

সাবকোয়েরি কী?

সাবকোয়েরি হলো একটি SQL কোয়েরির ভেতরে আরেকটি SQL কোয়েরি। এটি জটিল প্রশ্নের উত্তর দিতে ব্যবহার হয়।

সাবকোয়েরি উদাহরণ
SQL
-- গড়ের চেয়ে বেশি বেতন পায় এমন কর্মচারী
SELECT emp_name, salary
FROM   employee
WHERE  salary > (
    SELECT AVG(salary) FROM employee   -- সাবকোয়েরি
);

-- সর্বোচ্চ বেতনধারী কর্মচারী
SELECT emp_name, salary
FROM   employee
WHERE  salary = (SELECT MAX(salary) FROM employee);

-- IN সাবকোয়েরি: আইটি বিভাগের সব কর্মচারী
SELECT emp_name
FROM   employee
WHERE  dept_id IN (
    SELECT dept_id
    FROM   department
    WHERE  dept_name LIKE '%আইটি%'
);

-- EXISTS সাবকোয়েরি
SELECT dept_name
FROM   department d
WHERE  EXISTS (
    SELECT 1 FROM employee e
    WHERE  e.dept_id = d.dept_id
      AND  e.salary > 50000
);

-- Inline View (FROM-এ সাবকোয়েরি)
SELECT dept_name, avg_sal
FROM (
    SELECT d.dept_name, AVG(e.salary) avg_sal
    FROM   employee e JOIN department d ON e.dept_id=d.dept_id
    GROUP BY d.dept_name
) dept_stats
WHERE avg_sal > 50000;

CTE — WITH ক্লজ (পাঠযোগ্য কোড)

CTE (Common Table Expression)
SQL
-- CTE দিয়ে পাঠযোগ্য কোড
WITH
dept_stats AS (
    SELECT
        dept_id,
        AVG(salary)  AS avg_salary,
        COUNT(*)     AS emp_count
    FROM employee
    GROUP BY dept_id
),
high_pay_depts AS (
    SELECT dept_id
    FROM   dept_stats
    WHERE  avg_salary > 50000
)
SELECT
    e.emp_name,
    e.salary,
    d.dept_name
FROM      employee       e
JOIN      department     d  ON e.dept_id = d.dept_id
JOIN      high_pay_depts h  ON e.dept_id = h.dept_id
ORDER BY  e.salary DESC;

-- Recursive CTE (হায়ারার্কি ট্রিভার্স)
WITH org_chart (emp_id, emp_name, manager_id, level_num) AS (
    -- Anchor: সিইও (ম্যানেজার নেই)
    SELECT emp_id, emp_name, manager_id, 1
    FROM   employee
    WHERE  manager_id IS NULL
    UNION ALL
    -- Recursive: অধীনস্থ কর্মচারী
    SELECT e.emp_id, e.emp_name, e.manager_id, o.level_num+1
    FROM   employee e
    JOIN   org_chart o ON e.manager_id = o.emp_id
)
SELECT LPAD(' ', level_num*2) || emp_name AS hierarchy
FROM   org_chart
ORDER BY level_num;

Analytic Functions (Window Functions)

Analytic / Window Functions
SQL
-- বিভাগ অনুযায়ী বেতনের র‍্যাংকিং
SELECT
    emp_name,
    dept_id,
    salary,
    RANK()        OVER (PARTITION BY dept_id ORDER BY salary DESC) AS dept_rank,
    DENSE_RANK()  OVER (PARTITION BY dept_id ORDER BY salary DESC) AS dense_rank,
    ROW_NUMBER()  OVER (PARTITION BY dept_id ORDER BY salary DESC) AS row_num,
    SUM(salary)   OVER (PARTITION BY dept_id) AS dept_total,
    AVG(salary)   OVER () AS company_avg
FROM employee;

VIEW — ভার্চুয়াল টেবিল

VIEW হলো একটি সংরক্ষিত SELECT কোয়েরি যা টেবিলের মতো ব্যবহার করা যায়। এটি ডেটা সুরক্ষা, জটিলতা লুকানো এবং পুনর্ব্যবহারযোগ্যতার জন্য ব্যবহার হয়।

VIEW তৈরি ও ব্যবহার
SQL
-- সহজ VIEW তৈরি
CREATE OR REPLACE VIEW v_employee_dept AS
SELECT
    e.emp_id, e.emp_name, e.salary,
    d.dept_name, d.location
FROM  employee   e
JOIN  department d ON e.dept_id = d.dept_id;

-- VIEW ব্যবহার (টেবিলের মতোই)
SELECT * FROM v_employee_dept WHERE salary > 50000;

-- নিরাপত্তার জন্য VIEW (পাসওয়ার্ড লুকানো)
CREATE OR REPLACE VIEW v_public_employee AS
SELECT emp_id, emp_name, dept_id
FROM   employee
WHERE  active_status = 'Y'
WITH READ ONLY;  -- এই VIEW থেকে পরিবর্তন করা যাবে না

-- Materialized View (ক্যাশড, দ্রুত)
CREATE MATERIALIZED VIEW mv_dept_summary
REFRESH COMPLETE ON DEMAND
AS
SELECT
    dept_id,
    COUNT(*) emp_count,
    SUM(salary) total_salary,
    AVG(salary) avg_salary
FROM employee
GROUP BY dept_id;

-- Materialized View রিফ্রেশ করা
EXEC DBMS_MVIEW.REFRESH('MV_DEPT_SUMMARY');

INDEX — দ্রুত অনুসন্ধান

Index হলো ডেটাবেসের "বইয়ের সূচি"। এটি ছাড়া Oracle প্রতিটি সারি দেখে (Full Table Scan), Index থাকলে সরাসরি লাফিয়ে যায়।

INDEX তৈরি ও পরিচালনা
SQL
-- সাধারণ B-Tree Index
CREATE INDEX idx_emp_name ON employee(emp_name);

-- Composite Index (একাধিক কলাম)
CREATE INDEX idx_emp_dept_sal ON employee(dept_id, salary);

-- Unique Index
CREATE UNIQUE INDEX idx_emp_email ON employee(email);

-- Function-Based Index
CREATE INDEX idx_emp_upper_name ON employee(UPPER(emp_name));
-- এখন UPPER(emp_name) = 'KARIM' দ্রুত কাজ করবে

-- Bitmap Index (কম unique মানের জন্য)
CREATE BITMAP INDEX idx_gender ON employee(gender);

-- Index দেখা
SELECT index_name, index_type, status
FROM   user_indexes
WHERE  table_name = 'EMPLOYEE';

-- Index মুছে ফেলা
DROP INDEX idx_emp_name;
✅
Index কখন করবেন?WHERE, JOIN, ORDER BY-তে বারবার ব্যবহৃত কলামে। বড় টেবিলে (১০,০০০+ সারি)। Primary Key ও Foreign Key-তে।
⚠️
Index কখন এড়াবেন?ছোট টেবিলে। INSERT/UPDATE/DELETE বেশি হলে (Index maintain করতে সময় লাগে)। কম unique মান থাকলে (M/F)।

TCL — Transaction Control
SQL
-- ব্যাংক ট্রান্সফারের উদাহরণ
BEGIN
    -- হিসাব ১ থেকে টাকা কাটা
    UPDATE accounts
    SET    balance = balance - 10000
    WHERE  account_no = 'ACC001';

    -- হিসাব ২-এ টাকা যোগ
    UPDATE accounts
    SET    balance = balance + 10000
    WHERE  account_no = 'ACC002';

    -- সব ঠিক থাকলে স্থায়ী করুন
    COMMIT;
EXCEPTION
    WHEN OTHERS THEN
        -- কোনো সমস্যা হলে সব বাতিল
        ROLLBACK;
        RAISE;
END;

-- SAVEPOINT ব্যবহার
SAVEPOINT before_update;
UPDATE employee SET salary = salary * 2;  -- ভুল হয়ে গেছে!
ROLLBACK TO before_update;                    -- Savepoint-এ ফিরে যাও

-- লক দেখা
SELECT username, lockwait, status FROM v$session WHERE username IS NOT NULL;
ℹ️
Oracle-এর Auto COMMIT নেইMySQL-এ Auto COMMIT চালু থাকে, কিন্তু Oracle-এ নেই। তাই Oracle-এ INSERT/UPDATE/DELETE করলে অবশ্যই COMMIT দিতে হবে। DDL কমান্ড (CREATE, DROP) স্বয়ংক্রিয়ভাবে COMMIT করে।

DECLARE

    v_name      VARCHAR2(100);
    v_salary    NUMBER(10,2);
    v_count     NUMBER  := 0;        -- প্রাথমিক মান

BEGIN

    INTO   v_name, v_salary
    FROM   employee
    WHERE  emp_id = 1001;

    -- আউটপুট
    DBMS_OUTPUT.PUT_LINE('নাম: ' || v_name);
    DBMS_OUTPUT.PUT_LINE('বেতন: ' || v_salary);

    WHEN OTHERS THEN

শর্ত ও লুপ

IF-ELSIF এবং LOOP
PL/SQL
DECLARE
    v_salary NUMBER := 55000;
    v_grade  VARCHAR2(20);
    v_counter NUMBER := 1;
BEGIN
    -- IF-ELSIF-ELSE
    IF    v_salary >= 70000 THEN  v_grade := 'A';
    ELSIF v_salary >= 50000 THEN  v_grade := 'B';
    ELSIF v_salary >= 30000 THEN  v_grade := 'C';
    ELSE                            v_grade := 'D';
    END IF;
    DBMS_OUTPUT.PUT_LINE('গ্রেড: ' || v_grade);

    -- FOR LOOP
    FOR i IN 1..5 LOOP
        DBMS_OUTPUT.PUT_LINE('সংখ্যা: ' || i);
    END LOOP;

    -- WHILE LOOP
    WHILE v_counter <= 3 LOOP
        DBMS_OUTPUT.PUT_LINE('কাউন্টার: ' || v_counter);
        v_counter := v_counter + 1;
    END LOOP;

    -- FOR LOOP টেবিলের উপর (Implicit Cursor)
    FOR emp_rec IN (SELECT * FROM employee WHERE dept_id = 10)
    LOOP
        DBMS_OUTPUT.PUT_LINE(emp_rec.emp_name || ': ' || emp_rec.salary);
    END LOOP;
END;

Stored Procedure

Procedure তৈরি ও কল করা
PL/SQL
-- Procedure তৈরি
CREATE OR REPLACE PROCEDURE raise_salary (
    p_emp_id    IN  NUMBER,
    p_percent   IN  NUMBER,
    p_new_sal   OUT NUMBER
) AS
    v_old_salary NUMBER;
BEGIN
    SELECT salary INTO v_old_salary
    FROM   employee
    WHERE  emp_id = p_emp_id;

    p_new_sal := v_old_salary * (1 + p_percent/100);

    UPDATE employee
    SET    salary = p_new_sal
    WHERE  emp_id = p_emp_id;

    COMMIT;
    DBMS_OUTPUT.PUT_LINE('পুরোনো বেতন: ' || v_old_salary);
    DBMS_OUTPUT.PUT_LINE('নতুন বেতন: ' || p_new_sal);

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        RAISE_APPLICATION_ERROR(-20001, 'কর্মচারী পাওয়া যায়নি: ' || p_emp_id);
END raise_salary;
/

-- Procedure কল করা
DECLARE
    v_new NUMBER;
BEGIN
    raise_salary(1001, 15, v_new);  -- ১৫% বেতন বাড়ানো
    DBMS_OUTPUT.PUT_LINE('ফলাফল: ' || v_new);
END;

Function

Function তৈরি ও ব্যবহার

-- Function তৈরি (মান ফেরত দেয়)
CREATE OR REPLACE FUNCTION get_annual_salary (
    p_emp_id IN NUMBER
) RETURN NUMBER IS
    v_salary     NUMBER;
    v_bonus      NUMBER;
    v_annual     NUMBER;
BEGIN
    SELECT salary, NVL(bonus, 0)
    INTO   v_salary, v_bonus
    FROM   employee
    WHERE  emp_id = p_emp_id;

    v_annual := (v_salary * 12) + v_bonus;
    RETURN v_annual;

EXCEPTION
    WHEN NO_DATA_FOUND THEN
        RETURN NULL;
END;
/

-- Function SQL-এ ব্যবহার
SELECT
    emp_name,
    salary,
    get_annual_salary(emp_id) AS annual_income
FROM employee;

    PROCEDURE add_employee(p_name VARCHAR2, p_salary NUMBER, p_dept NUMBER);

END emp_pkg;
/

CREATE OR REPLACE PACKAGE BODY emp_pkg AS
    PROCEDURE add_employee(p_name VARCHAR2, p_salary NUMBER, p_dept NUMBER) IS
    BEGIN
        INSERT INTO employee (emp_id, emp_name, salary, dept_id)
        VALUES (emp_seq.NEXTVAL, p_name, p_salary, p_dept);
        COMMIT;
    END;

    FUNCTION get_count(p_dept NUMBER) RETURN NUMBER IS
        v_cnt NUMBER;
    BEGIN
        SELECT COUNT(*) INTO v_cnt FROM employee WHERE dept_id = p_dept;
        RETURN v_cnt;
    END;
END emp_pkg;
/

Trigger — স্বয়ংক্রিয় ক্রিয়া

Trigger হলো একটি PL/SQL ব্লক যা কোনো ঘটনা ঘটলে (INSERT, UPDATE, DELETE) স্বয়ংক্রিয়ভাবে চালু হয়।

Trigger তৈরি
PL/SQL
-- Audit Trigger: পরিবর্তনের লগ রাখা
CREATE TABLE salary_audit (
    audit_id    NUMBER GENERATED ALWAYS AS IDENTITY,
    emp_id      NUMBER,
    old_salary  NUMBER,
    new_salary  NUMBER,
    changed_by  VARCHAR2(100),
    changed_at  TIMESTAMP DEFAULT SYSTIMESTAMP
);

CREATE OR REPLACE TRIGGER trg_salary_audit
AFTER UPDATE OF salary ON employee
FOR EACH ROW
BEGIN
    INSERT INTO salary_audit (emp_id, old_salary, new_salary, changed_by)
    VALUES (:OLD.emp_id, :OLD.salary, :NEW.salary, USER);
END;
/

-- BEFORE INSERT Trigger: ডেটা বৈধতা
CREATE OR REPLACE TRIGGER trg_emp_validate
BEFORE INSERT OR UPDATE ON employee
FOR EACH ROW
BEGIN
    -- ইমেইল ছোট হাতে রাখা
    :NEW.email := LOWER(:NEW.email);

    -- নেতিবাচক বেতন ঠেকানো
    IF :NEW.salary < 0 THEN
        RAISE_APPLICATION_ERROR(-20002, 'বেতন ঋণাত্মক হতে পারে না!');
    END IF;

    -- hire_date ডিফল্ট
    IF :NEW.hire_date IS NULL THEN
        :NEW.hire_date := SYSDATE;
    END IF;
END;
/

-- INSTEAD OF Trigger (VIEW-এর উপর)
CREATE OR REPLACE TRIGGER trg_instead_of_view
INSTEAD OF INSERT ON v_employee_dept
FOR EACH ROW
BEGIN
    INSERT INTO employee (emp_name, dept_id)
    VALUES (:NEW.emp_name, :NEW.dept_id);
END;
/

Explicit Cursor

Cursor ব্যবহার
PL/SQL
DECLARE
    -- Cursor ঘোষণা
    CURSOR c_emp (p_dept NUMBER) IS
        SELECT emp_id, emp_name, salary
        FROM   employee
        WHERE  dept_id = p_dept
        ORDER BY salary DESC;

    v_emp_id    NUMBER;
    v_emp_name  VARCHAR2(100);
    v_salary    NUMBER;
    v_total     NUMBER := 0;

BEGIN
    OPEN c_emp(10);     -- Cursor খোলা

    LOOP
        FETCH c_emp INTO v_emp_id, v_emp_name, v_salary;  -- সারি পড়া
        EXIT WHEN c_emp%NOTFOUND;  -- আর সারি নেই?

        v_total := v_total + v_salary;
        DBMS_OUTPUT.PUT_LINE(v_emp_name || ': ' || v_salary);
    END LOOP;

    CLOSE c_emp;    -- Cursor বন্ধ
    DBMS_OUTPUT.PUT_LINE('মোট বেতন: ' || v_total);

EXCEPTION
    WHEN OTHERS THEN
        IF c_emp%ISOPEN THEN CLOSE c_emp; END IF;
        RAISE;
END;
/

-- নির্দিষ্ট টেবিলস্পেস ব্যাকআপ

RESTORE DATABASE;
RECOVER DATABASE;
ALTER DATABASE OPEN RESETLOGS;

-- Point-in-Time Recovery (নির্দিষ্ট সময়ে ফেরা)
RUN {

    RECOVER DATABASE;
    ALTER DATABASE OPEN RESETLOGS;
}

  • HAVING-এ WHERE-এর মতো ফিল্টার
সাধারণ অপ্টিমাইজেশন উদাহরণ
SQL
-- ❌ খারাপ: Index ব্যবহার হবে না
SELECT * FROM employee WHERE UPPER(emp_name) = 'KARIM';
-- ✅ ভালো: Function-Based Index বা এভাবে
SELECT * FROM employee WHERE emp_name = 'Karim';

-- ❌ খারাপ: Implicit conversion
WHERE emp_id = '1001'  -- VARCHAR vs NUMBER তুলনা
-- ✅ ভালো:
WHERE emp_id = 1001

-- ❌ খারাপ: NOT IN with NULL
WHERE dept_id NOT IN (SELECT dept_id FROM department WHERE budget > 0)
-- ✅ ভালো: NOT EXISTS ব্যবহার করুন
WHERE NOT EXISTS (SELECT 1 FROM department d
                  WHERE d.dept_id = e.dept_id AND d.budget > 0)

    estimate_percent => DBMS_STATS.AUTO_SAMPLE_SIZE
