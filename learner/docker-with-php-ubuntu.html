🐘 ডকার ও পিএইচপি: এক স্থানীয় স্থাপন অ্যাডভেঞ্চার

📘 একদা এক দেশে — উবুন্টু রাজ্যে, এক ডেভেলপার গিটহাব থেকে একটি পিএইচপি প্রকল্প নিয়ে এলো। কিন্তু তার উবুন্টু মেশিনে সরাসরি পিএইচপি আর অ্যাপাচি ইনস্টল না করেই কীভাবে সেটা চালানো যায়? তখনই ডকার (Docker) এসে বাঁচালো সবাইকে!

⓵ বিশৃঙ্খল লোকাল পরিবেশ
👩‍💻 আমাদের গল্পের নায়িকা তৃষা। তিনি এইমাত্র গিটহাব থেকে “ব্লগওয়েল” নামে একটি চমৎকার পিএইচপি প্রজেক্ট ক্লোন করলেন। প্রকল্পটির দরকার PHP 8.2, MySQL এবং কিছু এক্সটেনশন। কিন্তু তৃষার উবুন্টুতে আরেকটি পুরোনো প্রকল্পের জন্য PHP 7.4 বসানো আছে — 😖 ডিপেন্ডেন্সি জঞ্জাল। “আমার ল্যাপটপের সব গোলমাল হয়ে যাবে!” তৃষা চিন্তিত। তখন ডকার নামক এক বন্ধু হাজির, বিশাল তিমির মত।
গল্পের মন্তব্য: ডকার কন্টেইনার হচ্ছে আলাদা আলাদা টিফিন বক্সের মতো — প্রতিটি প্রোজেক্ট তার নিজস্ব নির্দিষ্ট জিনিসপত্র (PHP, Apache, extension) পায়, কারো সাথে কারো ঝগড়া নেই।
⓶ ডকারের জাদুকরি বাক্সগুলো

ইমেজ (Image)

একটা রেসিপি / ব্লুপ্রিন্ট। যেমন: php:8.2-apache মানে তাতে PHP আর Apache আগে থেকে তৈরি আছে।

কন্টেইনার (Container)

ইমেজ থেকে তৈরি করা একটা চলমান টিফিন বক্স। এটা থামানো/আবার চালু করা যায়।

Dockerfile

নিজস্ব ইমেজ বানানোর রেসিপি লিখে রাখা ফাইল (যাতে বলে দিই কোন ফাইল কপি করতে হবে, কোন এক্সটেনশন লাগবে)।

docker-compose

যখন একাধিক কন্টেইনার একসাথে চালানো দরকার (PHP + MySQL + phpMyAdmin), তখন তাদের অর্কেস্ট্রা করার জন্য।

⓷ তৃষা গিটহাব থেকে প্রোজেক্ট নামালো
🧪 প্রোজেক্টের নাম "তিমিব্লগ" (WhaleBlog) – একটা সাদাসিধে পিএইচপি ব্লগ। তৃষা উবুন্টু টার্মিনাল খুলে লিখলেন:
cd ~/projects
git clone https://github.com/bangla/whaleblog.git
cd whaleblog
ফোল্ডারের ভেতরে index.php, একটি Dockerfile এবং docker-compose.yml (আসল নায়ক) আছে। চলুন উঁকি দিই।
⓸ Dockerfile — পিএইচপি বক্স বানানোর রেসিপি
তৃষা Dockerfile ওপেন করে দেখল। এটা নির্দেশনা সমষ্টি, যেন একটি রান্নার বই।
# অফিসিয়াল PHP আর অ্যাপাচি ইমেজ থেকে শুরু করছি
FROM php:8.2-apache

# mysqli এক্সটেনশন ইনস্টল (ডাটাবেজের জন্য)
RUN docker-php-ext-install mysqli && docker-php-ext-enable mysqli

# বর্তমান ফোল্ডারের সব PHP ফাইল কন্টেইনারের ওয়েব ডিরেক্টরিতে কপি
COPY . /var/www/html/

# ওয়ার্কিং ডিরেক্টরি সেট
WORKDIR /var/www/html

# পোর্ট ৮০ খুলে দিচ্ছি (Apache শুনবে)
EXPOSE 80
তৃষা ভাবে: "ঠিক যেমন রান্না শুরু করি বেস হিসেবে — তারপর তরকারি (extension), তারপর আমার ফাইলগুলো বসাই।"
⓹ docker-compose: PHP আর MySQL একসঙ্গে
তিমিব্লগ চালাতে ডাটাবেজও দরকার। Docker Compose দিয়ে আমরা দুটো কন্টেইনার এক কমান্ডেই চালাতে পারি। docker-compose.yml ফাইলের ভেতর:
version: '3.8'
services:
  web:
    build: . # Dockerfile থেকে বিল্ড করো
    ports:
      - "8080:80" # হোস্টের 8080 → কন্টেইনারের 80
    volumes:
      - .:/var/www/html # ফাইল লাইভ সিঙ্ক (কোড বদলালে সাথে সাথে)
  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: rootpass
      MYSQL_DATABASE: whaleblog
    ports:
      - "3306:3306"
এই ফাইল দুই কন্টেইনারের কথা বলে: web (PHP) আর db (MySQL)। তারা সার্ভিসের নাম ধরে একে অপরকে চিনতে পারে।
⓺ উবুন্টুতে স্থানীয়ভাবে প্রজেক্ট চালু করা
তৃষা টার্মিনালে পবিত্র মন্ত্রটি টাইপ করে (whaleblog ফোল্ডারের ভেতর):
docker-compose up -d
ডকার কম্পোজ ফাইল পড়ে → PHP ইমেজ বানায় (যদি আগে না থাকে) → MySQL ইমেজ ডাউনলোড করে → দুই কন্টেইনার চালু করে। কয়েক সেকেন্ড পর:
✔ whaleblog-web-1 চালু
✔ whaleblog-db-1 চালু
তৃষা ব্রাউজার খুলে http://localhost:8080 এ গেল — তিমিব্লগ-এর হোমপেজ দেখা যাচ্ছে! 🚀 কাজ করছে!
⓻ উবুন্টু + গিটহাব + লোকাল = ❤️
শুরু থেকে অ্যাপ চালানো পর্যন্ত পুরো যাত্রাটা আবার দেখি:
১. গিটহাব ক্লোন ২. উবুন্টু টার্মিনাল ৩. Dockerfile + কম্পোজ ৪. docker-compose up ৫. localhost:8080
ডকার উবুন্টুতে খুব মসৃণে চলে। আর পিএইচপি-মাইএসকিউএল সরাসরি ইনস্টলের ঝামেলা নেই — সব কন্টেইনারের ভেতর। তৃষার উবুন্টু একদম পরিষ্কার থাকে, আর গিটহাব থেকে আনা প্রোজেক্ট একদম ঠিক মতো চলে।
⓼ তৃষার দৈনন্দিন ডকার মন্ত্র

কন্টেইনার চালু

docker-compose up -d

কন্টেইনার বন্ধ

docker-compose down

চলমান কন্টেইনার দেখা

docker ps

PHP লগ দেখা

docker logs whaleblog-web-1

পুনঃনির্মাণ (rebuild)

docker-compose build

অব্যবহৃত জিনিস মুছুন

docker system prune
⓽ শিক্ষা: ব্যথাহীন, বহনযোগ্য পিএইচপি স্থাপন
🌟 তৃষা এখন যেকোনো পিএইচপি প্রকল্প স্থানীয়ভাবে চালাতে পারে মাত্র দুটি কমান্ডে: git clone তারপর docker-compose up। একই পদ্ধতি অন্য যেকোনো মেশিনে (সহকর্মীর ম্যাক, প্রোডাকশন সার্ভার) কাজ করে। "আমার মেশিনে তো কাজ করে!" — এই যুক্তি এখন অতীত।
ডকারের কল্যাণে তৃষা ডিপেন্ডেন্সির জটিলতাকে আলাদা আলাদা নিয়ন্ত্রিত বাক্সে বন্দী করল — যেমন প্রতিটি পদ আলাদা তাজা কড়াইতে রান্না করা। গিটহাবের প্রোজেক্ট উবুন্টুতে খুশি মনে চলে, আর লোকাল ডিপ্লয়মেন্ট মানে শুধু docker-compose up টাইপ করা।

🐳 একটি গল্পেই সব বিষয়

📌 উবুন্টু টার্মিনাল থেকে শুরু → গিটহাব ক্লোন → Dockerfile + compose → docker-compose up → অ্যাপ চালু localhost:8080-এ। sudo apt install php/mysql ছাড়াই। এই হলো ডকারের গল্প।
