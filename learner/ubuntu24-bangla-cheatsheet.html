§01

Terminal ও মূল কমান্ড

Basic
💡

Terminal খোলার শর্টকাট: Ctrl + Alt + T চাপুন। অথবা Applications মেনু থেকে Terminal খুঁজুন। Ubuntu 24-এ ডিফল্ট shell হলো Bash।

সিস্টেম তথ্য

$uname -a
সিস্টেমের সম্পূর্ণ তথ্য — kernel version, hostname, architecture সব একসাথে
$lsb_release -a
Ubuntu version দেখুন — কোন Ubuntu ইনস্টল আছে জানতে
$hostname
কম্পিউটারের নাম দেখুন। hostname -I দিলে IP address দেখাবে
$whoami
বর্তমানে কোন user হিসেবে কাজ করছেন তা দেখায়
$uptime
সিস্টেম কতক্ষণ চলছে, load average সহ
$date
বর্তমান তারিখ ও সময়। date +"%Y-%m-%d" দিলে শুধু তারিখ
$cal
Terminal-এ ক্যালেন্ডার দেখান। cal 2025 পুরো বছর দেখায়
$history
আগে দেওয়া সব কমান্ডের তালিকা। history 20 শেষ ২০টি দেখায়
$clear
Terminal পরিষ্কার করুন। শর্টকাট: Ctrl + L
$exit
Terminal বন্ধ করুন। শর্টকাট: Ctrl + D

Directory Navigation

Basic
Directory ঘুরে বেড়ানো bash 
pwd                    # বর্তমান কোথায় আছেন দেখুন (Print Working Directory)
ls                     # ফাইল ও ফোল্ডার তালিকা
ls -la                 # লুকানো ফাইলসহ বিস্তারিত তালিকা
ls -lh                 # ফাইল size মানুষের পড়ার মতো (KB, MB)
ls -lt                 # সময় অনুযায়ী sort করা তালিকা
ls --color             # রঙিন তালিকা (ডিফল্টেই থাকে)

cd /home/user/Documents # নির্দিষ্ট পথে যান
cd ~                    # Home directory-তে যান (cd লিখলেও হয়)
cd ..                   # এক ধাপ উপরে যান
cd ../..               # দুই ধাপ উপরে যান
cd -                    # আগের directory-তে ফিরে যান
cd Documents/Projects  # Relative path দিয়ে যান

Ubuntu-র গুরুত্বপূর্ণ ফোল্ডার

Pathবাংলা নামকী আছে
/Rootসব কিছুর শুরু। সর্বোচ্চ ফোল্ডার
/home/userহোমআপনার ব্যক্তিগত ফাইল, ডকুমেন্ট সব এখানে
/etcকনফিগসিস্টেমের সব সেটিং ফাইল এখানে
/var/logলগসব এরর ও activity লগ ফাইল
/tmpঅস্থায়ীReboot-এ মুছে যায়
/usr/binপ্রোগ্রামইনস্টল করা software-এর বাইনারি
/bootবুটLinux kernel ও bootloader এখানে
/devডিভাইসHardware device ফাইল (disk, USB)
/procপ্রসেসচলমান process তথ্য (virtual)
/mnt, /mediaমাউন্টUSB, external drive এখানে মাউন্ট হয়
/optঅপশনালতৃতীয় পক্ষের বড় software
/rootRoot হোমroot ব্যবহারকারীর home directory

§04

ফাইল ও ফোল্ডার অপারেশন

Basic

তৈরি করুন

Create — ফাইল ও ফোল্ডার তৈরিbash
touch file.txt              # খালি ফাইল তৈরি / timestamp update
touch a.txt b.txt c.txt    # একসাথে তিনটি ফাইল তৈরি
mkdir projects             # নতুন ফোল্ডার তৈরি
mkdir -p a/b/c/d           # nested ফোল্ডার একসাথে তৈরি (-p = parents)
mkdir -m 755 mydir         # নির্দিষ্ট permission দিয়ে ফোল্ডার তৈরি
echo "হ্যালো বাংলাদেশ" > file.txt   # ফাইলে লেখা (overwrite)
echo "আরো লেখা" >> file.txt        # ফাইলে যোগ করুন (append)

দেখুন ও পড়ুন

Read — ফাইল দেখাbash
cat file.txt               # পুরো ফাইল একবারে দেখুন
cat -n file.txt            # line number সহ দেখুন
less file.txt              # বড় ফাইল scroll করে দেখুন (q = বের হন)
more file.txt              # পেজে পেজে দেখুন (space = পরের পেজ)
head file.txt              # প্রথম ১০ লাইন দেখুন
head -n 20 file.txt        # প্রথম ২০ লাইন দেখুন
tail file.txt              # শেষ ১০ লাইন দেখুন
tail -n 50 file.txt        # শেষ ৫০ লাইন দেখুন
tail -f /var/log/syslog    # Real-time log দেখুন (Ctrl+C বন্ধ করুন)
wc -l file.txt             # কত লাইন আছে গণনা করুন
wc -w file.txt             # কত শব্দ আছে গণনা করুন

কপি, সরান ও মুছুন

Copy · Move · Deletebash
cp file.txt /backup/       # ফাইল কপি করুন
cp -r folder/ /backup/    # পুরো ফোল্ডার কপি (-r = recursive)
cp -v *.txt /backup/       # কপি করার সময় দেখান কী হচ্ছে (-v = verbose)
cp -u file.txt /backup/   # শুধু নতুন/পরিবর্তিত হলে কপি করুন

mv file.txt /documents/   # ফাইল সরান (cut+paste)
mv old.txt new.txt         # ফাইলের নাম বদলান
mv dir1/ dir2/             # ফোল্ডার সরান

rm file.txt                # ফাইল মুছুন (Recycle Bin-এ যায় না!)
rm -i file.txt             # মুছার আগে confirm চাইবে
rm -r myfolder/           # পুরো ফোল্ডার মুছুন
rm -rf myfolder/          # জোর করে মুছুন (সাবধান!)
rmdir empty_folder/       # খালি ফোল্ডার মুছুন
⚠️

সাবধান! rm -rf ব্যবহারে সতর্ক থাকুন। মুছলে ফিরে পাওয়া কঠিন। rm -rf / কখনো দেবেন না — এটি পুরো সিস্টেম মুছে দেবে!

খোঁজা — Find, Grep, Locate

Intermediate
find — ফাইল খুঁজুনbash
find /home -name "*.txt"          # .txt ফাইল খুঁজুন
find . -name "config.php"          # বর্তমান ফোল্ডারে খুঁজুন
find / -type f -name "*.log"      # শুধু ফাইল খুঁজুন
find / -type d -name "backup"     # শুধু ফোল্ডার খুঁজুন
find /home -size +100M             # ১০০MB-এর বড় ফাইল
find . -mtime -7                    # শেষ ৭ দিনে পরিবর্তিত
find . -perm 777                    # নির্দিষ্ট permission দিয়ে
find . -name "*.py" -delete        # খুঁজে মুছুন (সাবধান!)
find . -name "*.txt" -exec ls -lh {} \;  # খুঁজে কমান্ড চালান
grep — ফাইলের ভেতরে খুঁজুনbash
grep "error" logfile.txt          # "error" শব্দ খুঁজুন
grep -i "ERROR" logfile.txt       # Case-insensitive খোঁজ
grep -n "error" logfile.txt       # Line number সহ
grep -r "TODO" ./src/             # ফোল্ডারের সব ফাইলে খুঁজুন
grep -v "debug" log.txt           # যে লাইনে নেই সেগুলো দেখান
grep -c "error" log.txt           # কতবার আছে গণনা করুন
grep -l "config" *.php            # কোন ফাইলে আছে শুধু নাম দেখান
grep -A 3 "error" log.txt         # Match-এর পরের ৩ লাইন দেখান
grep -E "error|warning" log.txt  # Regex দিয়ে খোঁজ
ps aux | grep "nginx"            # Pipe দিয়ে ফিল্টার করুন
$locate filename
দ্রুত খোঁজ — database থেকে। প্রথমে sudo updatedb চালান
$whereis python3
প্রোগ্রামের binary, source, man page কোথায় আছে দেখান
§06

File Permission ও Owner

Intermediate
🔐

Permission বোঝার চাবিকাঠি: -rwxrwxrwx — প্রথম অক্ষর ফাইল type, তারপর ৩টি করে তিন গ্রুপ: Owner | Group | Others এবং প্রতিটি গ্রুপে r (read=4), w (write=2), x (execute=1)

7
rwx
সব অধিকার
6
rw-
পড়া + লেখা
5
r-x
পড়া + চালানো
4
r--
শুধু পড়া
2
-w-
শুধু লেখা
0
---
কোনো অধিকার নেই
chmod · chown · chgrpbash
# chmod — Permission পরিবর্তন
chmod 755 script.sh          # owner:rwx, group:r-x, others:r-x
chmod 644 file.txt           # owner:rw-, group:r--, others:r--
chmod 777 public.html        # সবাইকে সব অধিকার (বিপজ্জনক!)
chmod 600 private.key        # শুধু owner পড়তে ও লিখতে পারবে
chmod +x script.sh          # Execute permission যোগ করুন
chmod -w file.txt           # Write permission সরান
chmod -R 755 myfolder/      # ফোল্ডারসহ সব কিছুতে apply (-R)
chmod u=rwx,g=rx,o=r f.sh  # Symbolic পদ্ধতি

# chown — মালিকানা পরিবর্তন
chown raju file.txt         # file-এর owner = raju
chown raju:developers f.txt # owner=raju, group=developers
chown -R www-data:www-data /var/www/  # ওয়েব ফোল্ডার

# ls দিয়ে Permission দেখুন
ls -l file.txt
-rw-r--r-- 1 raju developers 2048 Jan 15 file.txt
#  ↑ type+perm  ↑links ↑owner ↑group  ↑size ↑date  ↑name
$sudo chmod 400 ~/.ssh/id_rsa
SSH key-এর সঠিক permission — শুধু owner পড়তে পারবে
$umask 022
নতুন ফাইলের default permission সেট করুন

Intermediate
Links তৈরিbash

APT Package Manager

Basic
📦

APT (Advanced Package Tool) হলো Ubuntu-র software ইনস্টল করার প্রধান পদ্ধতি। sudo ছাড়া বেশিরভাগ apt কমান্ড কাজ করবে না।

APT সম্পূর্ণ গাইডbash
# ===== সিস্টেম আপডেট =====
sudo apt update                 # Package list refresh করুন (সবসময় আগে করুন)
sudo apt upgrade                # সব আপডেট ইনস্টল করুন
sudo apt full-upgrade           # Dependencies সহ সম্পূর্ণ upgrade
sudo apt dist-upgrade           # Distribution upgrade
sudo apt update && sudo apt upgrade -y  # দুটি একসাথে

# ===== ইনস্টল করুন =====
sudo apt install nginx          # একটি software ইনস্টল করুন
sudo apt install nginx php mysql-server  # একসাথে অনেক ইনস্টল
sudo apt install -y curl       # -y = confirm ছাড়াই install
sudo apt install --reinstall nginx  # পুনরায় ইনস্টল করুন

# ===== সরান =====
sudo apt remove nginx           # software সরান (config রাখুন)
sudo apt purge nginx            # config সহ সম্পূর্ণ সরান
sudo apt autoremove             # অপ্রয়োজনীয় package মুছুন
sudo apt autoclean              # পুরানো .deb cache মুছুন
sudo apt clean                  # সব .deb cache মুছুন

# ===== খোঁজুন =====
apt search nginx               # Package খুঁজুন
apt show nginx                 # Package বিস্তারিত তথ্য
apt list --installed           # ইনস্টল করা সব package
apt list --upgradable          # আপডেট দেওয়া যাবে এমন package
dpkg -l | grep nginx          # নির্দিষ্ট package ইনস্টল আছে কিনা
dpkg -L nginx                  # Package কোথায় কোথায় ফাইল রেখেছে

# ===== .deb ফাইল ইনস্টল =====
sudo dpkg -i package.deb       # .deb ফাইল ইনস্টল
sudo dpkg -r package-name     # .deb দিয়ে ইনস্টল করা সরান
sudo apt install -f            # Broken dependency ঠিক করুন

# ===== Snap Package =====
snap install vscode --classic  # Snap package ইনস্টল
snap list                       # ইনস্টল করা snap দেখুন
snap remove vscode             # Snap সরান
snap refresh                    # সব snap আপডেট করুন
§09

Process Management

Intermediate

Process হলো চলমান প্রোগ্রাম। প্রতিটির একটি unique PID (Process ID) থাকে।

R — Running (চলছে) S — Sleeping (ঘুমাচ্ছে) T — Stopped (থেমে আছে) Z — Zombie (মৃত কিন্তু আছে) D — Uninterruptible (I/O অপেক্ষা)
Process দেখা ও নিয়ন্ত্রণbash
# ===== Process দেখুন =====
ps                             # বর্তমান shell-এর process
ps aux                         # সব user-এর সব process
ps aux | grep nginx           # নির্দিষ্ট process খুঁজুন
ps -ef                         # Full-format process list
pstree                         # Process tree দেখুন

# ===== Real-time monitoring =====

elif [ $AGE -ge 13 ]; then
    echo "কিশোর"
else
    echo "শিশু"
fi

if [ "$NAME" == "রাহুল" ]; then
    echo "স্বাগতম রাহুল!"

if [ -f "/etc/nginx/nginx.conf" ]; then
    echo "Nginx config পাওয়া গেছে"

    echo "web directory আছে"

fi

for i in 1 2 3 4 5; do
    echo "গণনা: $i"
done

for file in /home/*.txt; do
    echo "ফাইল: $file"
done

# ===== While Loop =====
COUNT=1
while [ $COUNT -le 5 ]; do
    echo "$COUNT"
    COUNT=$(( COUNT + 1 ))
done

    echo "হ্যালো, $1!"       # $1 = প্রথম argument

}

ls /nonexistent

⏰

Cron দিয়ে নির্দিষ্ট সময়ে স্বয়ংক্রিয়ভাবে কমান্ড চালানো যায়। যেমন: প্রতিদিন রাতে backup নেওয়া।

Crontab Format ও উদাহরণbash
# Crontab format:  মিনিট  ঘণ্টা  দিন  মাস  সপ্তাহবার  কমান্ড
#                   0-59  0-23  1-31 1-12    0-7
#                   * = যেকোনো, */5 = প্রতি ৫, 1,3 = ১ বা ৩

# ===== সম্পাদনা =====
crontab -e                    # নিজের crontab edit করুন
crontab -l                    # বর্তমান crontab দেখুন
crontab -r                    # সব cron job মুছুন
sudo crontab -u raju -e      # অন্য user-এর crontab

# ===== উদাহরণ =====
*  *  *  *  * /script.sh   # প্রতি মিনিটে
0  * *  *  * /script.sh    # প্রতি ঘণ্টায় (XX:00 তে)
0  0  *  *  * /backup.sh  # প্রতিদিন মধ্যরাতে
0  9  *  *  1 /report.sh  # প্রতি সোমবার সকাল ৯টায়
0  0  1  *  * /monthly.sh # প্রতি মাসের ১ তারিখে
*/5 *  *  *  * /check.sh # প্রতি ৫ মিনিটে
30 23 * * 1-5 /backup.sh  # সোম-শুক্র রাত ১১:৩০

# ===== Special Keywords =====
@reboot /start.sh            # Boot-এ একবার
@daily  /daily.sh            # প্রতিদিন (= 0 0 * * *)
@weekly /weekly.sh           # প্রতি সপ্তাহে
@monthly /monthly.sh         # প্রতি মাসে
@hourly /hourly.sh           # প্রতি ঘণ্টায়

# ===== Log সহ =====
0 2 * * * /backup.sh >> /var/log/backup.log 2>&1

Archive ও Compression

Basic
tar · gzip · zip · unzipbash
# ===== TAR (Linux-এর প্রধান archive tool) =====
# c=create, x=extract, v=verbose, f=file, z=gzip, j=bzip2
tar -czf backup.tar.gz myfolder/   # ফোল্ডার compress করুন
tar -czf backup.tar.gz *.txt       # সব .txt compress করুন
tar -xzf backup.tar.gz              # বর্তমান ফোল্ডারে extract
tar -xzf backup.tar.gz -C /tmp/  # নির্দিষ্ট জায়গায় extract
tar -tzf backup.tar.gz              # ভেতরে কী আছে দেখুন (extract ছাড়া)
tar -cjf backup.tar.bz2 folder/   # bzip2 (বেশি compress)
tar -cJf backup.tar.xz folder/    # xz (সবচেয়ে বেশি compress)

# ===== GZIP =====
gzip file.txt                      # → file.txt.gz (মূল মুছে)
gzip -k file.txt                   # → মূল ফাইল রাখে (-k = keep)
gunzip file.txt.gz                 # Extract করুন
zcat file.txt.gz                   # Extract ছাড়াই পড়ুন

# ===== ZIP (Windows compatible) =====
zip archive.zip file1.txt file2.txt   # ZIP তৈরি
zip -r archive.zip myfolder/          # ফোল্ডার ZIP করুন
zip -e secure.zip file.txt            # Password দিয়ে ZIP
unzip archive.zip                      # Extract করুন
unzip -d /destination/ archive.zip    # নির্দিষ্ট জায়গায়
unzip -l archive.zip                  # ভেতরে কী আছে দেখুন
