01

Cron Job কী এবং কেন দরকার?

Cron Job হলো Linux/Unix অপারেটিং সিস্টেমের একটি Time-based Job Scheduler। এটি দিয়ে তুমি নির্দিষ্ট সময়ে, নির্দিষ্ট ব্যবধানে কোনো কমান্ড বা স্ক্রিপ্ট স্বয়ংক্রিয়ভাবে চালাতে পারো — কোনো মানুষের হাত ছাড়াই।

প্রতিদিন সকালে Newsletter বা Report ইমেইল করা

🗄️

রাত ২টায় স্বয়ংক্রিয়ভাবে DB backup নেওয়া

🧹

প্রতি ঘণ্টায় পুরনো cache ও temp ফাইল মুছে ফেলা

📊

সপ্তাহান্তে Sales report তৈরি করা

🔄

তৃতীয়পক্ষের API থেকে ডেটা sync করা

⚠️

Health Check

প্রতি ৫ মিনিটে server ও service চেক করা

💡
"Cron" শব্দটি এসেছে Greek শব্দ Chronos (χρόνος) থেকে, যার অর্থ "সময়"। Cron ডেমন সার্ভিসটি সার্ভার চলার সময় সবসময় background-এ active থাকে।
02

কীভাবে কাজ করে?

Cron Job-এর পুরো সিস্টেমটা মূলত তিনটি অংশ নিয়ে তৈরি:

🔧 Cron Daemon

সার্ভারের background-এ সবসময় চলতে থাকা service। নাম crond। এটি প্রতি মিনিটে crontab চেক করে।

📄 Crontab File

একটি text ফাইল যেখানে তোমার সব Cron Job-এর সময়সূচি লেখা থাকে। প্রতি ব্যবহারকারীর আলাদা crontab থাকে।

⚡ Script / Command

যে script বা command নির্দিষ্ট সময়ে চালাতে চাও। PHP, Python, Bash, Node.js যেকোনো কিছু হতে পারে।

🔄 Cron Job চালু হওয়ার Flow
1

Cron Daemon চালু থাকে

সার্ভার start হলেই crond service background-এ চালু হয়। এটি প্রতি মিনিটে একবার করে জেগে ওঠে।

2

Crontab পড়ে

Daemon সব ব্যবহারকারীর crontab file পড়ে এবং বর্তমান সময়ের সাথে মেলায়।

3

সময় মিললে Execute করে

যদি কোনো Job-এর সময় এখনকার সময়ের সাথে মেলে, Daemon সেই command বা script execute করে।

4

Output পাঠায় বা Log করে

Job চলার output ইমেইলে পাঠানো যায় বা ফাইলে log করা যায়।

⚠️
মনে রাখো: Cron সবচেয়ে ছোট interval হলো ১ মিনিট। ১ মিনিটের চেয়ে কম সময়ে কিছু চালাতে চাইলে অন্য পদ্ধতি লাগবে।

PHP দিয়ে Cron Job

PHP script-কে Cron Job-এ চালানোর দুটো উপায় আছে — CLI (command line) দিয়ে এবং URL hit করে। CLI পদ্ধতি বেশি ভালো ও নিরাপদ।

📝 PHP Script তৈরি করা

প্রথমে একটা PHP file বানাও:

PHP — /home/user/scripts/send_email.php
#!/usr/bin/env php <?php /** * Cron Job: Daily Email Sender * প্রতিদিন সকাল ৯টায় newsletter পাঠাবে */ // Script শুরুর সময় নোট করা $start_time = date('Y-m-d H:i:s'); echo "[{$start_time}] Cron শুরু হয়েছে\n"; // Database connection $host = 'localhost'; $db = 'mydb'; $user = 'root'; $pass = 'secret'; try { $pdo = new PDO("mysql:host={$host};dbname={$db}", $user, $pass); // সকল subscriber আনা $stmt = $pdo->query("SELECT email, name FROM subscribers WHERE active = 1"); $subscribers = $stmt->fetchAll(PDO::FETCH_ASSOC); $count = 0; foreach ($subscribers as $sub) { // Email পাঠানো $subject = "আজকের Newsletter"; $message = "হ্যালো {$sub['name']}, আজকের আপডেট..."; $headers = "From: no-reply@mysite.com\r\n"; if (mail($sub['email'], $subject, $message, $headers)) { $count++; } } echo "সফলভাবে {$count} টি email পাঠানো হয়েছে\n"; } catch (PDOException $e) { echo "Error: " . $e->getMessage() . "\n"; exit(1); // Error হলে exit code 1 দাও } echo "[". date('H:i:s') ."] Cron শেষ হয়েছে\n"; exit(0); // সফল হলে exit code 0
⚙️ Crontab-এ যোগ করা
Terminal — crontab edit করো
# crontab editor খোলো crontab -e # এই লাইনটি যোগ করো — প্রতিদিন সকাল ৯টায় চলবে 0 9 * * * /usr/bin/php /home/user/scripts/send_email.php # PHP-এর path নিশ্চিত করতে (আগে terminal-এ চালাও) which php # output: /usr/bin/php ← এটাই cron-এ ব্যবহার করো # Output log ফাইলে সেভ করতে চাইলে 0 9 * * * /usr/bin/php /home/user/scripts/send_email.php >> /home/user/logs/cron.log 2>&1
💡
2>&1 মানে — error output (stderr) কেও log ফাইলে লেখো। শুধু >> দিলে শুধু normal output লেখা হয়।

🗄️ Database Backup Cron (PHP)

PHP — /home/user/scripts/backup_db.php
<?php // Database Backup Script — রাত ২টায় চলবে $db_name = 'mydb'; $db_user = 'root'; $db_pass = 'secret'; $backup_dir = '/home/user/backups'; $filename = $backup_dir . '/backup_' . date('Y-m-d_H-i-s') . '.sql.gz'; // Backup directory না থাকলে তৈরি করো if (!is_dir($backup_dir)) { mkdir($backup_dir, 0755, true); } // mysqldump command চালাও $command = "mysqldump -u{$db_user} -p{$db_pass} {$db_name} | gzip > {$filename} 2>&1"; exec($command, $output, $return_code); if ($return_code === 0) { echo "✅ Backup সফল: {$filename}\n"; // ৭ দিনের পুরনো backup মুছে দাও exec("find {$backup_dir} -name '*.sql.gz' -mtime +7 -delete"); echo "🗑️ পুরনো backup মুছে ফেলা হয়েছে\n"; } else { echo "❌ Backup ব্যর্থ! Error: " . implode("\n", $output) . "\n"; } # Crontab line: প্রতিদিন রাত ২টায় # 0 2 * * * /usr/bin/php /home/user/scripts/backup_db.php >> /home/user/logs/backup.log 2>&1

🚀 Laravel Schedule (আধুনিক পদ্ধতি)

PHP — app/Console/Kernel.php (Laravel)
protected function schedule(Schedule $schedule) { // প্রতিদিন সকাল ৯টায় email পাঠাবে $schedule->command('email:send-newsletter') ->dailyAt('09:00'); // প্রতি ঘণ্টায় cache clear $schedule->command('cache:clear') ->hourly(); // প্রতি সোমবার report generate $schedule->job(new GenerateWeeklyReport()) ->weekly() ->mondays() ->at('08:00'); // Closure দিয়েও কাজ করা যায় $schedule->call(function () { DB::table('sessions') ->where('last_activity', '<', now()->subHour()) ->delete(); })->hourly(); }

Python দিয়ে Cron Job

Python script-কে cron-এ চালানো খুবই সহজ। তবে virtual environment ব্যবহার করলে path সঠিকভাবে দিতে হবে।

Python — /home/user/scripts/scraper.py
#!/usr/bin/env python3 # -*- coding: utf-8 -*- """ Cron Script: Data Scraper প্রতি ৬ ঘণ্টায় ডেটা সংগ্রহ করে DB-তে সেভ করবে """ import requests import sqlite3 import logging from datetime import datetime # Logging setup logging.basicConfig( filename='/home/user/logs/scraper.log', level=logging.INFO, format='%(asctime)s - %(levelname)s - %(message)s' ) logger = logging.getLogger(__name__) def fetch_data(): """API থেকে ডেটা আনে""" try: response = requests.get( 'https://api.example.com/data', timeout=30 ) response.raise_for_status() return response.json() except requests.exceptions.RequestException as e: logger.error(f"API error: {e}") return None def save_to_db(data): """ডেটা SQLite-তে সেভ করে""" conn = sqlite3.connect('/home/user/data/mydata.db') cursor = conn.cursor() cursor.execute('''CREATE TABLE IF NOT EXISTS records (id INTEGER PRIMARY KEY, value TEXT, created_at TEXT)''') for item in data: cursor.execute( 'INSERT INTO records (value, created_at) VALUES (?, ?)', (str(item), datetime.now().isoformat()) ) conn.commit() conn.close() return len(data) if __name__ == '__main__': logger.info('Cron শুরু হয়েছে') data = fetch_data() if data: count = save_to_db(data) logger.info(f'{count} টি record সেভ হয়েছে') else: logger.warning('ডেটা পাওয়া যায়নি') logger.info('Cron শেষ হয়েছে')

Virtual Environment ব্যবহার করলে venv-এর python দাও 0 */6 * * * /home/user/venv/bin/python /home/user/scripts/scraper.py # ৩. Script-কে executable করে শেবাং দিয়ে 0 */6 * * * /home/user/scripts/scraper.py >> /home/user/logs/cron.log 2>&1

Node.js দিয়ে Cron Job

Node.js-এ দুটো উপায়ে Cron করা যায়: ১) সিস্টেম crontab দিয়ে script চালানো, ২) node-cron বা cron package ব্যবহার করে।

পদ্ধতি ১ — System Crontab + Node Script

JavaScript — /home/user/scripts/send-report.js
#!/usr/bin/env node 'use strict'; const nodemailer = require('nodemailer'); const mysql = require('mysql2/promise'); async function sendDailyReport() { console.log(`[${new Date().toISOString()}] Report generation শুরু`); // DB connection const conn = await mysql.createConnection({ host: 'localhost', user: 'root', password: 'secret', database: 'mydb' }); // ডেটা আনা const [rows] = await conn.execute( 'SELECT COUNT(*) as total, SUM(amount) as revenue FROM orders WHERE DATE(created_at) = CURDATE()' ); await conn.end(); const { total, revenue } = rows[0]; // Email পাঠানো const transporter = nodemailer.createTransporter({ service: 'gmail', auth: { user: 'admin@mysite.com', pass: 'apppassword' } }); await transporter.sendMail({ from: 'admin@mysite.com', to: 'boss@mysite.com', subject: `Daily Report — ${new Date().toLocaleDateString('bn-BD')}`, html: `<h2>আজকের রিপোর্ট</h2> <p>মোট অর্ডার: <b>${total}</b></p> <p>মোট আয়: <b>৳${revenue}</b></p>` }); console.log('✅ Report email পাঠানো হয়েছে'); } sendDailyReport().catch(err => { console.error('❌ Error:', err); process.exit(1); });

const cron = require('node-cron'); // প্রতি ৫ মিনিটে চলবে cron.schedule('*/5 * * * *', () => { console.log('প্রতি ৫ মিনিটে কাজ হচ্ছে...'); checkServerHealth(); }); // প্রতিদিন রাত ১২টায় cron.schedule('0 0 * * *', () => { console.log('দৈনিক cleanup শুরু...'); cleanupOldFiles(); }, { timezone: 'Asia/Dhaka' // ✅ বাংলাদেশ সময় অনুযায়ী }); // প্রতি সোমবার সকাল ৮টায় cron.schedule('0 8 * * 1', async () => { await generateWeeklyReport(); await sendReportEmail(); }, { timezone: 'Asia/Dhaka' }); console.log('⏰ Cron Scheduler চালু হয়েছে');

URL Hit করে Cron চালানো

কখনো কখনো সরাসরি script চালানোর বদলে একটা URL request পাঠিয়ে cron চালানো হয়। Shared hosting-এ এই পদ্ধতি জনপ্রিয়।

⚠️
সতর্কতা: URL hit করে cron চালালে অবশ্যই secret token বা IP whitelist দিয়ে protect করো, নাহলে যে কেউ URL call করলে cron চলে যাবে।
PHP — cron_endpoint.php (Protected URL)
<?php // URL: https://mysite.com/cron/cron_endpoint.php?token=YOUR_SECRET // ১. Secret token দিয়ে protect করা $secret = 'my_super_secret_cron_token_2024'; $given_token = $_GET['token'] ?? ''; if (!hash_equals($secret, $given_token)) { http_response_code(403); exit('Forbidden'); } // ২. শুধু নির্দিষ্ট IP থেকে allow করা (আরও ভালো) $allowed_ips = ['127.0.0.1', '10.0.0.1']; if (!in_array($_SERVER['REMOTE_ADDR'], $allowed_ips)) { http_response_code(403); exit(); } // Cron কাজ এখানে করো set_time_limit(300); // ৫ মিনিট পর্যন্ত চলতে দাও ignore_user_abort(true); // connection বন্ধ হলেও চলতে থাকবে echo "Cron চলছে...\n"; // ... তোমার কাজ ... echo "Done!\n";
Crontab — wget / curl দিয়ে URL hit করা
# curl দিয়ে URL hit করা (recommended) 0 9 * * * curl -s "https://mysite.com/cron/cron_endpoint.php?token=SECRET" > /dev/null 2>&1 # wget দিয়ে 0 9 * * * wget -q -O /dev/null "https://mysite.com/cron/cron_endpoint.php?token=SECRET" # Timeout দিয়ে (৬০ সেকেন্ড) 0 9 * * * curl -s --max-time 60 "https://mysite.com/cron/endpoint.php?token=SECRET"

Log ও Debug করা

Cron Job ঠিকমতো চলছে কিনা বোঝার জন্য logging অপরিহার্য।

Crontab — বিভিন্ন Log পদ্ধতি
# ১. সব output একটা log ফাইলে পাঠাও (append করবে) * * * * * /usr/bin/php /home/user/script.php >> /home/user/logs/cron.log 2>&1 # ২. আলাদা error log * * * * * /usr/bin/php /home/user/script.php >> /home/user/logs/output.log 2>> /home/user/logs/error.log # ৩. timestamp সহ log * * * * * /usr/bin/php /home/user/script.php >> /home/user/logs/cron_$(date +\%Y-\%m-\%d).log 2>&1 # ৪. Output সম্পূর্ণ বাদ দিতে (শান্ত থাকতে চাইলে) * * * * * /usr/bin/php /home/user/script.php > /dev/null 2>&1 # ৫. Cron-এর নিজস্ব log দেখতে (Linux) grep CRON /var/log/syslog grep CRON /var/log/cron
Terminal — Debug Commands
# বর্তমান crontab দেখো crontab -l # crontab edit করো crontab -e # অন্য user-এর crontab দেখো (root হিসেবে) crontab -l -u username # Cron daemon চলছে কিনা দেখো systemctl status cron # Ubuntu/Debian systemctl status crond # CentOS/RHEL # Cron লগ real-time দেখো tail -f /var/log/syslog | grep CRON # Script manually test করো (cron চালানোর আগে) /usr/bin/php /home/user/scripts/send_email.php /usr/bin/python3 /home/user/scripts/scraper.py # Log ফাইল দেখো tail -50 /home/user/logs/cron.log cat /home/user/logs/cron.log | grep "Error"
🔴
সাধারণ ভুল: Cron-এর environment সীমিত। PATH, HOME ইত্যাদি variable সেট করা থাকে না। তাই সবসময় full path ব্যবহার করো: /usr/bin/php না লিখে শুধু php লিখলে কাজ করবে না!
Crontab — Environment Variables সেট করা
# Crontab-এর শুরুতে environment variables দাও SHELL=/bin/bash PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin MAILTO="" # তারপর cron job লেখো 0 9 * * * /usr/bin/php /home/user/scripts/send_email.php >> /home/user/logs/cron.log 2>&1

Pro Tips ও Best Practices

🕐 Timezone সেট করো

Cron UTC সময়ে চলে। বাংলাদেশ সময় UTC+6। তাই ঢাকা সময় সকাল ৯টা = UTC ৩টা।

# Crontab-এ timezone সেট করো CRON_TZ=Asia/Dhaka 0 9 * * * /usr/bin/php /script.php
🔒 Lock File ব্যবহার করো

একই সময়ে দুটো instance চলা আটকাতে lock file ব্যবহার করো।

<?php $lock = '/tmp/myscript.lock'; if (file_exists($lock)) exit("চলছে ইতিমধ্যে\n"); file_put_contents($lock, getmypid()); // কাজ করো... unlink($lock); // শেষে মুছে দাও
📨 Email Alert সেট করো

Error হলে ইমেইলে জানতে চাইলে MAILTO সেট করো।

# সব output ইমেইলে পাঠাবে MAILTO=admin@mysite.com # শুধু error পাঠাতে (script-এ stderr ব্যবহার করো) MAILTO="" # প্রথমে বন্ধ করো
⚡ Cron Load Test করো

Deploy করার আগে script manually চালিয়ে দেখো।

# Cron-এর মতো environment-এ test করো env -i HOME=/home/user \ PATH=/usr/local/bin:/usr/bin:/bin \ /usr/bin/php /home/user/script.php
📋 সংক্ষিপ্ত চেকলিস্ট
✅Script-এ সবসময় full absolute path ব্যবহার করো
✅PHP/Python binary-র full path দাও (which php)
✅Output সবসময় log file-এ save করো (>> log.txt 2>&1)
✅Script-এ error handling ও exit code ব্যবহার করো
✅Timezone সম্পর্কে সচেতন থাকো (UTC vs local)
✅URL-based cron হলে token দিয়ে protect করো
✅Deploy করার আগে script manually test করো
✅Long-running job-এ lock file ব্যবহার করো
✅Cron script web root-এর বাইরে রাখো
✅শুধু প্রয়োজনীয় পারমিশন দাও (chmod 755)
