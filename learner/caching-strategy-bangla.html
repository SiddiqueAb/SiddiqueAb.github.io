# What is Caching & Why

ক্যাশিং হলো ডেটা অস্থায়ীভাবে দ্রুততর স্টোরেজে রাখার প্রক্রিয়া যাতে পরবর্তীবার একই ডেটা চাইলে মূল সোর্স থেকে না এনে সরাসরি ক্যাশ থেকে দেওয়া যায়।

মনে করো তুমি প্রতিদিন একই প্রশ্নের উত্তর দিতে হয়। বুদ্ধিমানের কাজ হলো উত্তরটি একটি নোটবুকে লিখে রাখা — এটাই ক্যাশ।

⚡ মূল সুবিধা

ডেটাবেস কোয়েরি কমে, সার্ভার লোড কমে, রেসপন্স টাইম মিলিসেকেন্ডে নামে, ব্যান্ডউইডথ সাশ্রয় হয় এবং ব্যবহারকারীর অভিজ্ঞতা অনেক ভালো হয়।

# HTTP Caching Deep Dive

HTTP ক্যাশিং হলো ওয়েবের সবচেয়ে মৌলিক ক্যাশিং মেকানিজম। এটি HTTP হেডার-এর মাধ্যমে ব্রাউজার, প্রক্সি ও CDN-কে বলে দেয় রেসপন্স কতক্ষণ ক্যাশ করতে হবে।

গুরুত্বপূর্ণ HTTP Cache Headers

Cache-Control

সবচেয়ে গুরুত্বপূর্ণ হেডার। ব্রাউজার ও প্রক্সি উভয়কে নির্দেশনা দেয়। HTTP/1.1 এ যোগ হয়েছে।

HTTP Headers
# পাবলিক রিসোর্স — ১ বছর ক্যাশ
Cache-Control: public, max-age=31536000, immutable

# প্রাইভেট ডেটা — শুধু ব্রাউজার ক্যাশ করবে
Cache-Control: private, max-age=3600

# রিভ্যালিডেট করো, কিন্তু stale ব্যবহার করো
Cache-Control: stale-while-revalidate=60

# একদমই ক্যাশ করবে না
Cache-Control: no-store

# ক্যাশ করো কিন্তু আগে ভ্যালিডেট করো
Cache-Control: no-cache

# সর্বোচ্চ বয়স + শেয়ার্ড ক্যাশ বয়স
Cache-Control: public, max-age=600, s-maxage=86400
ETag ও Last-Modified

Conditional Request-এর জন্য ব্যবহার হয়। সার্ভার জানায় রিসোর্স পরিবর্তন হয়েছে কিনা।

🧠 Cache Busting — ক্যাশ ভাঙার কৌশল

ফাইল পরিবর্তন হলে পুরনো ক্যাশ ভাঙতে হয়। সবচেয়ে ভালো উপায়: ফাইলনামে content hash যোগ করো। যেমন: main.a3f9b2c1.js — এতে max-age অনেক বড় দেওয়া যায়, নতুন ফাইল হলে নাম বদলে যাবে।

Nginx — HTTP Cache Config
# /etc/nginx/sites-available/mysite
server {
    listen 80;

    # স্ট্যাটিক অ্যাসেট — ১ বছর, immutable
    location ~* \.(js|css|png|jpg|svg|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, max-age=31536000, immutable";
        add_header Vary "Accept-Encoding";
    }

    # HTML — no-cache (ভ্যালিডেট করবে)
    location ~* \.html$ {
        add_header Cache-Control "no-cache, must-revalidate";
        etag on;
    }

    # API — সাধারণত ক্যাশ করা উচিত না
    location /api/ {
        add_header Cache-Control "no-store";
        proxy_pass http://backend;
    }
}

JavaScript — localStorage ক্যাশিং প্যাটার্ন
// সিম্পল localStorage ক্যাশ ইমপ্লিমেন্টেশন
const CacheManager = {
  // ডেটা সেট করো TTL সহ
  set(key, data, ttlSeconds = 300) {
    const item = {
      data,
      expires: Date.now() + (ttlSeconds * 1000),
      createdAt: new Date().toISOString()
    };
    localStorage.setItem(key, JSON.stringify(item));
  },

  // ডেটা পাও — expire হলে null
  get(key) {
    const raw = localStorage.getItem(key);
    if (!raw) return null;

    const item = JSON.parse(raw);
    if (Date.now() > item.expires) {
      localStorage.removeItem(key); // পুরনো মুছে দাও
      return null;
    }
    return item.data;
  },

  // Cache-or-Fetch প্যাটার্ন
  async getOrFetch(key, fetchFn, ttl = 300) {
    const cached = this.get(key);
    if (cached) {
      console.log(`✅ Cache HIT: ${key}`);
      return cached;
    }
    console.log(`❌ Cache MISS: ${key}`);
    const freshData = await fetchFn();
    this.set(key, freshData, ttl);
    return freshData;
  }
};

// ব্যবহার
const products = await CacheManager.getOrFetch(
  'products_list',
  () => fetch('/api/products').then(r => r.json()),
  600 // ১০ মিনিট
);

Redis — সম্পূর্ণ গাইড
# Redis In-Memory Database

Redis (Remote Dictionary Server) হলো সবচেয়ে জনপ্রিয় ইন-মেমরি ডেটা স্টোর। এটি শুধু ক্যাশ নয় — একটি সম্পূর্ণ ডেটা স্ট্রাকচার সার্ভার।

Redis কোথায় ডেটা রাখে?

RAM (প্রাথমিক স্টোরেজ)
সমস্ত ডেটা RAM-এ থাকে। তাই অবিশ্বাস্য দ্রুত — microsecond latency।
RDB Snapshot (ডিস্ক)
নির্দিষ্ট সময় পর পর সম্পূর্ণ ডেটাসেটের snapshot ডিস্কে সেভ। restart-এ লোড হয়।
AOF Log (ডিস্ক)
প্রতিটি write অপারেশন log ফাইলে লেখা হয়। Crash recovery-তে সাহায্য করে।
Replica (অন্য সার্ভার)
Master-Replica replication। High availability ও read scaling-এর জন্য।
💾 Redis Data Types
  • String — সাধারণ key-value, counter
  • Hash — Object/Map (user profile)
  • List — Ordered list, queue, stack
  • Set — Unique values, tags
  • Sorted Set — Leaderboard, ranking
  • Stream — Event log, message queue
  • Bitmap — Feature flags, analytics
  • HyperLogLog — Unique count (approx)

Node.js-এ Redis ব্যবহার (ioredis)

Node.js — Redis Cache Implementation
const Redis = require('ioredis');
const redis = new Redis({
  host: 'localhost',
  port: 6379,
  retryStrategy: (times) => Math.min(times * 50, 2000)
});

// ১. সিম্পল String ক্যাশ
async function getUserById(userId) {
  const cacheKey = `user:${userId}`;

  // প্রথমে ক্যাশ চেক
  const cached = await redis.get(cacheKey);
  if (cached) return JSON.parse(cached);

  // ক্যাশ miss — DB থেকে আনো
  const user = await db.query('SELECT * FROM users WHERE id = ?', [userId]);

  // ক্যাশে রাখো — ৩৬০০ সেকেন্ড TTL
  await redis.setex(cacheKey, 3600, JSON.stringify(user));
  return user;
}

// ২. Hash দিয়ে Object ক্যাশ
async function cacheUserProfile(userId, profile) {
  await redis.hset(`profile:${userId}`, {
    name: profile.name,
    email: profile.email,
    score: profile.score
  });
  await redis.expire(`profile:${userId}`, 7200);
}

// ৩. Sorted Set — Leaderboard
async function updateLeaderboard(username, score) {
  await redis.zadd('leaderboard', score, username);
}

async function getTopPlayers(n = 10) {
  // সর্বোচ্চ স্কোর থেকে ঘুরে নাও
  return redis.zrevrange('leaderboard', 0, n - 1, 'WITHSCORES');
}

// ৪. Cache Invalidation — ব্যবহারকারী আপডেট হলে
async function updateUser(userId, data) {
  await db.query('UPDATE users SET ...', data);
  // ক্যাশ মুছে দাও
  await redis.del(`user:${userId}`);
  await redis.del(`profile:${userId}`);
}

Redis Pub/Sub Pattern
// Publisher — ডেটা পরিবর্তন হলে broadcast
async function invalidateCacheAcrossServers(pattern) {
  await redisPublisher.publish('cache:invalidate', pattern);
}

// Subscriber — প্রতিটি সার্ভার শুনছে
redisSubscriber.subscribe('cache:invalidate');
redisSubscriber.on('message', async (channel, pattern) => {
  const keys = await redis.keys(pattern);
  if (keys.length > 0) {
    await redis.del(...keys);
    console.log(`Cleared ${keys.length} cache entries`);
  }
});

Memcached — ডিস্ট্রিবিউটেড ক্যাশ
# Memcached Complete Guide

Memcached হলো একটি সিম্পল, দ্রুত, ডিস্ট্রিবিউটেড মেমরি ক্যাশিং সিস্টেম। এটি শুধু String key-value স্টোর। Redis-এর মতো rich data types নেই, কিন্তু horizontal scaling অনেক সহজ।

Memcached কোথায় ডেটা রাখে?

শুধু RAM-এ। কোনো persistence নেই। Server restart করলে সব মুছে যায়। একাধিক সার্ভার জুড়ে consistent hashing দিয়ে key বিতরণ হয়।

✅ Memcached সুবিধা
  • অতি সরল আর্কিটেকচার
  • Multi-threaded
  • Horizontal scaling সহজ
  • কম ওভারহেড
❌ সীমাবদ্ধতা
  • শুধু String type
  • Persistence নেই
  • Pub/Sub নেই
  • Replication নেই
Node.js — Memcached Implementation
const Memcached = require('memcached');

  '192.168.1.3:11211'

  retry: 10000,
  remove: true,
  failOverServers: ['192.168.1.4:11211']

// Set — ডেটা সেভ করো
const cacheSet = (key, value, lifetime = 600) =>
  new Promise((resolve, reject) =>
    memcached.set(key, JSON.stringify(value), lifetime,
      err => err ? reject(err) : resolve(true)));

// Get — ডেটা পাও
const cacheGet = (key) =>
  new Promise((resolve, reject) =>
    memcached.get(key, (err, data) =>
      err ? reject(err) : resolve(data ? JSON.parse(data) : null)));

// Express middleware হিসেবে ব্যবহার
const cacheMiddleware = (duration) => async (req, res, next) => {
  const key = `page:${req.url}`;
  const cached = await cacheGet(key);

  if (cached) {
    res.setHeader('X-Cache', 'HIT');
    return res.json(cached);
  }
  res.sendResponse = res.json;
  res.json = async (body) => {
    await cacheSet(key, body, duration);
    res.setHeader('X-Cache', 'MISS');
    res.sendResponse(body);
  };
  next();
};

// Route-এ ব্যবহার
app.get('/api/products', cacheMiddleware(300), getProducts);

addEventListener('fetch', event => {
  event.respondWith(handleRequest(event.request));
});

async function handleRequest(request) {
  const url = new URL(request.url);
  const cache = caches.default;

  // API রেসপন্স ক্যাশ করো
  if (url.pathname.startsWith('/api/public')) {
    let cachedResponse = await cache.match(request);
    if (cachedResponse) return cachedResponse;

    const response = await fetch(request);
    const newResponse = new Response(response.body, {
      status: response.status,
      headers: {
        ...response.headers,
        'Cache-Control': 'public, max-age=300',
        'CF-Cache-Status': 'HIT'
      }
    });
    event.waitUntil(cache.put(request, newResponse.clone()));
    return newResponse;
  }

  return fetch(request);
}

# Service Worker & PWA Cache

Service Worker হলো ব্রাউজারে একটি background JavaScript thread যা নেটওয়ার্ক রিকোয়েস্ট intercept করতে পারে এবং Cache API ব্যবহার করে অফলাইনেও কাজ করতে পারে।

sw.js — Service Worker
const CACHE_NAME = 'my-app-v1.0.0';
const STATIC_ASSETS = [
  '/', '/index.html',
  '/css/app.css', '/js/app.js',
  '/offline.html'
];

// ইনস্টল — স্ট্যাটিক ফাইল আগে থেকে ক্যাশ করো
self.addEventListener('install', async event => {
  event.waitUntil(
    caches.open(CACHE_NAME).then(cache =>
      cache.addAll(STATIC_ASSETS))
  );
  self.skipWaiting(); // তাৎক্ষণিক activate
});

// Activate — পুরনো cache মুছে দাও
self.addEventListener('activate', event => {
  event.waitUntil(
    caches.keys().then(keys =>
      Promise.all(
        keys.filter(k => k !== CACHE_NAME)
            .map(k => caches.delete(k))
      ))
  );
  self.clients.claim();
});

// Fetch — নেটওয়ার্ক রিকোয়েস্ট intercept
self.addEventListener('fetch', event => {
  const { request } = event;

  // API রিকোয়েস্ট: Network First
  if (request.url.includes('/api/')) {
    event.respondWith(
      fetch(request)
        .then(async res => {
          const cache = await caches.open(CACHE_NAME);
          cache.put(request, res.clone());
          return res;
        })
        .catch(() => caches.match(request)) // offline fallback
    );
    return;
  }

  // স্ট্যাটিক: Cache First
  event.respondWith(
    caches.match(request)
      .then(cached => cached || fetch(request))
      .catch(() => caches.match('/offline.html'))
  );
});

# Caching Patterns & Strategies
1. Cache-Aside (Lazy Loading)
সবচেয়ে প্রচলিত প্যাটার্ন। অ্যাপ নিজেই ক্যাশ ম্যানেজ করে। Cache Miss হলে DB থেকে এনে ক্যাশে রাখে।

প্রবাহ: Cache Check → Miss হলে DB → DB ডেটা Cache-এ রাখো → Return

✅ সহজ ✅ resilient ⚠️ Thundering herd
2. Write-Through
ডেটা লেখার সময় একসাথে ক্যাশ ও DB দুটোতেই লেখা হয়। ক্যাশ সবসময় up-to-date থাকে।

প্রবাহ: Write → Cache-এ লেখো → DB-তে লেখো → Return

✅ সামঞ্জস্যপূর্ণ ⚠️ Write latency বাড়ে
3. Write-Behind (Write-Back)
আগে ক্যাশে লেখো, পরে অ্যাসিঙ্ক্রোনাসলি DB-তে লেখো। Write অনেক দ্রুত হয় কিন্তু ডেটা লস ঝুঁকি আছে।

প্রবাহ: Write → Cache → (async) → DB পরে

✅ দ্রুত write ⚠️ ডেটা লস ঝুঁকি
4. Read-Through
Cache নিজেই DB-এর সামনে বসে থাকে। অ্যাপ শুধু ক্যাশ থেকে পড়ে, ক্যাশ নিজে DB-কে কল করে।

প্রবাহ: App → Cache → Cache Miss হলে → DB → Cache update → App

✅ সহজ App Logic ⚠️ প্রথম read ধীর
5. Refresh-Ahead
Cache expire হওয়ার আগেই প্রি-এম্পটিভলি নতুন ডেটা লোড করো। কখনো Cache Miss হয় না।

প্রবাহ: TTL ৮০% শেষ → Background refresh → ব্যবহারকারী সবসময় fresh ডেটা পায়

✅ Low latency ⚠️ অপ্রয়োজনীয় refresh
6. Stale-While-Revalidate
পুরনো (stale) ডেটা এখনই দাও, পিছনে নতুন ডেটা আনো। HTTP Cache-Control এ বিদ্যমান।

প্রবাহ: Request → Stale দাও তাৎক্ষণিক → Background-এ নতুন আনো → পরের request-এ নতুন

✅ UX চমৎকার ⚠️ Briefly stale

Node.js — Tag-Based Cache Invalidation
// Tag-based invalidation Redis দিয়ে
class TaggedCache {
  constructor(redis) { this.redis = redis; }

  // ক্যাশ সেট করো tags সহ
  async set(key, value, tags = [], ttl = 3600) {
    const pipeline = this.redis.pipeline();
    pipeline.setex(key, ttl, JSON.stringify(value));

    // প্রতিটি tag-এ key যোগ করো
    for (const tag of tags) {
      pipeline.sadd(`tag:${tag}`, key);
      pipeline.expire(`tag:${tag}`, ttl + 60);
    }
    await pipeline.exec();
  }

  // একটি tag-এর সব cache মুছে দাও
  async invalidateTag(tag) {
    const keys = await this.redis.smembers(`tag:${tag}`);
    if (keys.length === 0) return;

    const pipeline = this.redis.pipeline();
    keys.forEach(key => pipeline.del(key));
    pipeline.del(`tag:${tag}`);
    await pipeline.exec();
    console.log(`🗑️ ${keys.length} cache entries cleared for tag: ${tag}`);
  }
}

// ব্যবহার
const cache = new TaggedCache(redis);

// Product list ক্যাশ — tag: 'products', 'category:electronics'
await cache.set('products:list', productData, ['products', 'category:electronics']);

// Product আপডেট হলে — সব সম্পর্কিত ক্যাশ মুছো
await cache.invalidateTag('products'); // সব product ক্যাশ মুছবে

# Database Query Caching
Node.js — Smart Query Cache Layer
// Database Query Cache Wrapper
class QueryCache {
  constructor(db, redis) {
    this.db = db;
    this.redis = redis;
  }

  // Query hash তৈরি (unique key)
  hashQuery(sql, params) {
    const crypto = require('crypto');
    return 'qcache:' + crypto
      .createHash('md5')
      .update(sql + JSON.stringify(params))
      .digest('hex');
  }

  // Cached query execute
  async query(sql, params = [], ttl = 300) {
    const key = this.hashQuery(sql, params);

    // ক্যাশ চেক
    const cached = await this.redis.get(key);
    if (cached) {
      return { data: JSON.parse(cached), fromCache: true };
    }

    // DB থেকে আনো
    const rows = await this.db.execute(sql, params);

    // ক্যাশে রাখো
    await this.redis.setex(key, ttl, JSON.stringify(rows));

    return { data: rows, fromCache: false };
  }
}

// Memoization — Function Level Cache
function memoize(fn, ttl = 60000) {
  const cache = new Map();
  return async function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) {
      const { value, expires } = cache.get(key);
      if (Date.now() < expires) return value;
    }
    const result = await fn.apply(this, args);
    cache.set(key, { value: result, expires: Date.now() + ttl });
    return result;
  };
}

// ব্যবহার
const cachedGetUser = memoize(getUserById, 30000);
const user = await cachedGetUser(42);

Best Practices — প্রফেশনাল কৌশল
# Senior Engineer Best Practices
✅ করো (DO)
  • ক্যাশ key-এ version বা hash যোগ করো
  • TTL সবসময় set করো — orphan cache এড়াতে
  • Graceful degradation: ক্যাশ unavailable হলে DB থেকে আনো
  • Cache stampede/thundering herd প্রতিরোধ করো (lock দিয়ে)
  • মেমরি সীমা ও eviction policy সেট করো
  • ক্যাশ hit/miss metrics monitor করো
  • সেনসিটিভ ডেটা ক্যাশ করার আগে encrypt করো
  • ক্যাশ warming করো deployment-এর পরে
❌ করো না (DON'T)
  • ব্যক্তিগত/সেনসিটিভ ডেটা CDN-এ ক্যাশ করো না
  • TTL ছাড়া ক্যাশ করো না
  • সব কিছু ক্যাশ করো না — overhead বাড়ে
  • ক্যাশ invalidation ignore করো না
  • ক্যাশকে primary data source হিসেবে ধরো না
  • ক্যাশ failure-এ app crash করো না
  • অত্যন্ত ছোট TTL দিও না — লাভ হবে না
  • একটি বিশাল ক্যাশ key তৈরি করো না

Redis — Mutex Lock দিয়ে Stampede প্রতিরোধ
// একই সময়ে হাজার request আসলে DB-তে চাপ পড়া ঠেকাও
async function getWithLock(key, fetchFn, ttl = 300) {
  // ক্যাশ চেক
  let data = await redis.get(key);
  if (data) return JSON.parse(data);

  const lockKey = `lock:${key}`;
  const lockValue = Date.now().toString();

  // Mutex lock নাও (NX = not exists)
  const locked = await redis.set(lockKey, lockValue, 'EX', 10, 'NX');

  if (locked) {
    try {
      // আমরাই DB fetch করি
      const freshData = await fetchFn();
      await redis.setex(key, ttl, JSON.stringify(freshData));
      return freshData;
    } finally {
      await redis.del(lockKey); // lock ছাড়ো
    }
  } else {
    // অন্যরা অপেক্ষা করে পুনরায় চেক করবে
    await new Promise(r => setTimeout(r, 100));
    return getWithLock(key, fetchFn, ttl); // retry
  }
}

🔴 কোথায় ডেটা থাকে

  • Memcached — শুধু RAM
  • Browser — RAM + ডিস্ক

  • Simple string, scale → Memcached
  • Static files → CDN + HTTP
