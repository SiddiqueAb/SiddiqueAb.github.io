ফোল্ডার স্ট্রাকচার

📁 কেন সঠিক স্ট্রাকচার দরকার?

বড় প্রজেক্টে যখন ৫-১০ জন ডেভেলপার কাজ করেন, তখন সঠিক ফোল্ডার স্ট্রাকচার না থাকলে কোড খুঁজে পাওয়া, বোঝা এবং মেইনটেইন করা কঠিন হয়ে পড়ে। Laravel-এর ডিফল্ট স্ট্রাকচার ছোট প্রজেক্টের জন্য ভালো, কিন্তু মিড/লার্জ প্রজেক্টে আমাদের আরও কাস্টম লেয়ার যোগ করতে হয়।

03

Service Layer — Business Logic

⚙️ Service Layer কী এবং কেন?

Service Layer হলো আপনার application-এর "মগজ"। সব business logic — যেমন payment process, order calculation, email পাঠানো, notification — এখানে থাকবে। Controller শুধু request নেবে এবং response দেবে, কিন্তু কোনো business logic জানবে না।

  • Controller পাতলা (thin) থাকে, শুধু request/response handle করে
  • একই service অনেক জায়গা থেকে ব্যবহার করা যায় (reusable)
  • Unit testing করা সহজ হয়
  • Code পরিবর্তন করলে এক জায়গাতেই পরিবর্তন করতে হয়
❌ ভুল — Controller-এ Business Logic
public function register(Request $request)
{
    // সব কিছু controller-এ লেখা ❌
    $user = User::create([...]);
    Mail::to($user)->send(...);
    Cache::forget('users');
    Log::info('User created');
    // ৫০+ লাইনের controller!
}
✅ সঠিক — Service Layer ব্যবহার
public function register(
    StoreUserRequest $request,
    UserService $userService
) {
    $user = $userService
        ->register($request->validated());

    return new UserResource($user);
    // পরিষ্কার ৫ লাইন! ✅
}

📝 UserService সম্পূর্ণ উদাহরণ

PHPnamespace App\Services;

use App\Models\User;
use App\Repositories\Interfaces\UserRepositoryInterface;
use App\Events\UserRegistered;
use App\Exceptions\UserCreationException;
use Illuminate\Support\Facades\Hash;
use Illuminate\Support\Facades\DB;

class UserService
{
    /**
     * Constructor injection — dependency injection ব্যবহার করুন
     */
    public function __construct(
        private readonly UserRepositoryInterface $userRepository,
    ) {}

    /**
     * নতুন user register করা
     * @throws UserCreationException
     */
    public function register(array $data): User
    {
        // Transaction ব্যবহার — একটি ব্যর্থ হলে সব rollback
        return DB::transaction(function () use ($data) {
            $user = $this->userRepository->create([
                'name'     => $data['name'],
                'email'    => $data['email'],
                'password' => Hash::make($data['password']),
            ]);

            // Event fire — listener আলাদাভাবে কাজ করবে
            event(new UserRegistered($user));

            return $user;
        });
    }

    /**
     * User profile update — শুধু allowed fields পরিবর্তন
     */
    public function updateProfile(User $user, array $data): User
    {
        $allowedFields = ['name', 'phone', 'bio', 'avatar'];

        $updateData = array_intersect_key(
            $data,
            array_flip($allowedFields)
        );

        return $this->userRepository->update($user->id, $updateData);
    }
}
04

Repository Pattern

🗄️ Repository Pattern কী?

Repository হলো Database access-এর মধ্যস্থতাকারী। Service কখনো সরাসরি Eloquent model ব্যবহার করবে না — সবসময় Repository-র মাধ্যমে data নেবে। এতে পরে database change করলেও (MySQL → MongoDB) শুধু Repository পরিবর্তন করলেই হবে।

📋 ধাপ ১: Interface তৈরি করুন

PHP// app/Repositories/Interfaces/UserRepositoryInterface.php
namespace App\Repositories\Interfaces;

use App\Models\User;
use Illuminate\Pagination\LengthAwarePaginator;

interface UserRepositoryInterface
{
    public function findById(int $id): ?User;
    public function findByEmail(string $email): ?User;
    public function create(array $data): User;
    public function update(int $id, array $data): User;
    public function delete(int $id): bool;
    public function paginate(int $perPage = 15): LengthAwarePaginator;
}

🔧 ধাপ ২: Repository Implement করুন

PHP// app/Repositories/UserRepository.php
namespace App\Repositories;

use App\Models\User;
use App\Repositories\Interfaces\UserRepositoryInterface;

class UserRepository implements UserRepositoryInterface
{
    public function __construct(
        private readonly User $model
    ) {}

    public function findById(int $id): ?User
    {
        return $this->model->find($id);
    }

    public function create(array $data): User
    {
        return $this->model->create($data);
    }

    public function paginate(int $perPage = 15): LengthAwarePaginator
    {
        return $this->model
            ->latest()
            ->paginate($perPage);
    }

    // ... অন্যান্য methods
}

🔗 ধাপ ৩: ServiceProvider-এ Bind করুন

PHP// app/Providers/RepositoryServiceProvider.php
namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use App\Repositories\Interfaces\UserRepositoryInterface;
use App\Repositories\UserRepository;

class RepositoryServiceProvider extends ServiceProvider
{
    public function register(): void
    {
        // Interface চাইলে, এই concrete class দাও
        $this->app->bind(
            UserRepositoryInterface::class,
            UserRepository::class
        );
    }
}

Controller Guidelines

✅ Controller কী করবে

  • Request validate করা (Form Request দিয়ে)
  • Service call করা
  • Response return করা
  • HTTP status code সেট করা

🚫 Controller কী করবে না

  • Database query লেখা নয়
  • Email/Notification পাঠানো নয়
  • Business logic রাখা নয়
  • ৩০+ লাইনের method রাখা নয়

📝 সঠিক Controller উদাহরণ

PHPnamespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use App\Http\Requests\User\StoreUserRequest;
use App\Http\Requests\User\UpdateUserRequest;
use App\Http\Resources\UserResource;
use App\Services\UserService;
use App\Models\User;

class UserController extends Controller
{
    public function __construct(
        private readonly UserService $userService
    ) {}

    /**
     * সব users দেখাও (paginated)
     */
    public function index(): JsonResponse
    {
        $users = $this->userService->getAllPaginated();

        return UserResource::collection($users)
            ->response();
    }

    /**
     * নতুন user তৈরি করো
     */
    public function store(StoreUserRequest $request): JsonResponse
    {
        $user = $this->userService
            ->register($request->validated());

        return (new UserResource($user))
            ->response()
            ->setStatusCode(201);
    }

    /**
     * একজন user দেখাও
     */
    public function show(User $user): JsonResponse
    {
        // Route Model Binding ব্যবহার করুন — $user auto-loaded
        return (new UserResource($user))->response();
    }
}

Model Best Practices

🏗️ সঠিকভাবে Model লেখা

PHPnamespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\SoftDeletes;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use App\Enums\UserStatus;

class User extends Model
{
    use HasFactory, SoftDeletes;

    // ১. সবসময় $fillable বা $guarded লিখুন
    protected $fillable = [
        'name', 'email', 'password',
        'phone', 'status',
    ];

    // ২. hidden fields — API response-এ দেখাবে না
    protected $hidden = ['password', 'remember_token'];

    // ৩. Casts — type নিশ্চিত করুন
    protected function casts(): array
    {
        return [
            'email_verified_at' => 'datetime',
            'password'          => 'hashed',
            'status'            => UserStatus::class, // Enum cast
            'settings'          => 'array',
        ];
    }

    // ৪. Relationships — সব relationship method লিখুন
    public function orders(): HasMany
    {
        return $this->hasMany(Order::class);
    }

    public function profile(): HasOne
    {
        return $this->hasOne(UserProfile::class);
    }

    // ৫. Scope — reusable query filters
    public function scopeActive($query): $query
    {
        return $query->where('status', UserStatus::Active);
    }
}

⚡ Enum ব্যবহার করুন (Laravel 10+)

PHP// app/Enums/UserStatus.php
namespace App\Enums;

enum UserStatus: string
{
    case Active   = 'active';
    case Inactive = 'inactive';
    case Banned   = 'banned';

    public function label(): string
    {
        return match($this) {
            UserStatus::Active   => 'সক্রিয়',
            UserStatus::Inactive => 'নিষ্ক্রিয়',
            UserStatus::Banned   => 'নিষিদ্ধ',
        };
    }
}

Form Request Validation

🛡️ Validation Controller-এ নয়, Request-এ লিখুন

PHP// app/Http/Requests/User/StoreUserRequest.php
namespace App\Http\Requests\User;

use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Validation\Rules\Password;

class StoreUserRequest extends FormRequest
{
    /**
     * এই request কে করতে পারবে?
     */
    public function authorize(): bool
    {
        return true; // বা policy check: auth()->user()->can('create', User::class)
    }

    /**
     * Validation rules
     */
    public function rules(): array
    {
        return [
            'name'     => ['required', 'string', 'max:100'],
            'email'    => ['required', 'email', 'unique:users,email'],
            'password' => ['required', 'confirmed',
                            Password::min(8)->mixedCase()->numbers()],
            'phone'    => ['nullable', 'string', 'regex:/^01[3-9]\d{8}$/'],
        ];
    }

    /**
     * Custom error messages — ইউজারের বোঝার জন্য
     */
    public function messages(): array
    {
        return [
            'email.unique'   => 'এই ইমেইল ইতিমধ্যে ব্যবহার হচ্ছে।',
            'phone.regex'    => 'বাংলাদেশের সঠিক ফোন নম্বর দিন।',
            'password.min'   => 'পাসওয়ার্ড কমপক্ষে ৮ অক্ষরের হতে হবে।',
        ];
    }

    /**
     * Data পরিষ্কার করুন validation-এর আগে
     */
    protected function prepareForValidation(): void
    {
        $this->merge([
            'email' => strtolower($this->email ?? ''),
        ]);
    }
}

API Resource

📦 API Resource কেন ব্যবহার করবেন?

Model সরাসরি return করলে database structure expose হয়ে যায়। API Resource দিয়ে আপনি ঠিক করতে পারেন ক্লায়েন্ট কোন data দেখবে এবং কোন format-এ দেখবে।

🎨 UserResource উদাহরণ

PHP// app/Http/Resources/UserResource.php
namespace App\Http\Resources;

use Illuminate\Http\Request;
use Illuminate\Http\Resources\Json\JsonResource;

class UserResource extends JsonResource
{
    public function toArray(Request $request): array
    {
        return [
            'id'         => $this->id,
            'name'       => $this->name,
            'email'      => $this->email,
            'phone'      => $this->phone,
            'status'     => $this->status,
            'status_label' => $this->status?->label(),

            // Conditional — শুধু যখন load করা আছে
            'orders'     => OrderResource::collection(
                                $this->whenLoaded('orders')
                             ),
            'profile'    => new ProfileResource(
                                $this->whenLoaded('profile')
                             ),

            // শুধু admin-রা দেখবে
            'ip_address' => $this->when(
                                $request->user()?->isAdmin(),
                                $this->last_ip
                             ),

            'created_at' => $this->created_at?->format('d M, Y'),
        ];
    }
}
✅
whenLoaded() ব্যবহার করুন — এটি N+1 query সমস্যা এড়াতে সাহায্য করে। Relationship load না হলে response-এ আসবে না।

Events & Listeners

🔔 Events ব্যবহার করুন Side Effects-এর জন্য

User register হওয়ার পর email পাঠানো, notification দেওয়া, log রাখা — এগুলো Service-এ না লিখে Event fire করুন এবং Listener-এ handle করুন। এতে Service পরিষ্কার থাকে।

PHP// app/Events/UserRegistered.php
class UserRegistered
{
    public function __construct(
        public readonly User $user
    ) {}
}

// app/Listeners/SendWelcomeEmail.php
class SendWelcomeEmail
{
    public function handle(UserRegistered $event): void
    {
        Mail::to($event->user)
            ->queue(new WelcomeEmail($event->user));
    }
}

// app/Listeners/CreateUserWallet.php
class CreateUserWallet
{
    public function handle(UserRegistered $event): void
    {
        // একই event, আলাদা listener
        Wallet::create(['user_id' => $event->user->id]);
    }
}

// EventServiceProvider-এ register করুন:
protected $listen = [
    UserRegistered::class => [
        SendWelcomeEmail::class,
        CreateUserWallet::class,
        NotifyAdmins::class,
    ],
];

🔧 Job উদাহরণ

PHP// app/Jobs/ProcessOrderExport.php
namespace App\Jobs;

use Illuminate\Bus\Queueable;
use Illuminate\Contracts\Queue\ShouldQueue;
use Illuminate\Foundation\Bus\Dispatchable;

class ProcessOrderExport implements ShouldQueue
{
    use Dispatchable, Queueable;

    // কতবার retry করবে
    public $tries = 3;

    // কত সেকেন্ডে timeout
    public $timeout = 120;

    public function __construct(
        private readonly int $userId,
        private readonly string $dateRange,
    ) {}

    public function handle(OrderExportService $exportService): void
    {
        $exportService->generate($this->userId, $this->dateRange);
    }

    // ব্যর্থ হলে কী করবে?
    public function failed(\Throwable $exception): void
    {
        Log::error('Export failed', [
            'user_id' => $this->userId,
            'error'   => $exception->getMessage(),
        ]);
    }
}

// dispatch করার উপায়:
ProcessOrderExport::dispatch($userId, '2024-01');
ProcessOrderExport::dispatch($userId, '2024-01')->delay(now()->addMinutes(5));

Caching Strategy

⚡ সঠিক Caching Pattern

PHPuse Illuminate\Support\Facades\Cache;

class UserService
{
    private const CACHE_TTL = 3600; // ১ ঘণ্টা

    public function getUserProfile(int $userId): User
    {
        // remember: আছে দাও, নেই তো তৈরি করে দাও
        return Cache::remember(
            "user.profile.{$userId}",
            self::CACHE_TTL,
            fn() => $this->userRepository->findWithRelations($userId)
        );
    }

    public function updateProfile(User $user, array $data): User
    {
        $updated = $this->userRepository->update($user->id, $data);

        // Update করলে cache মুছে ফেলুন
        Cache::forget("user.profile.{$user->id}");

        return $updated;
    }

    // সম্পর্কিত সব cache মুছতে Tags ব্যবহার করুন (Redis)
    public function clearUserCache(int $userId): void
    {
        Cache::tags(["user.{$userId}"])->flush();
    }
}

SOLID Principles — Laravel-এ

S Single Responsibility

প্রতিটি class শুধু একটি কাজ করবে। UserService শুধু user-সম্পর্কিত কাজ করবে, email পাঠাবে না।

O Open/Closed

নতুন feature যোগ করতে পুরনো code পরিবর্তন না করে extend করুন। Interface দিয়ে নতুন implementation যোগ করুন।

L Liskov Substitution

Interface implement করা যেকোনো class, সেই interface-এর জায়গায় ব্যবহার করা যাবে। MySQLUserRepo এবং MongoUserRepo উভয়ই একই interface implement করে।

I Interface Segregation

বড় interface না বানিয়ে ছোট ছোট specific interface বানান। ReadableRepository এবং WritableRepository আলাদা রাখুন।

D Dependency Inversion — সবচেয়ে গুরুত্বপূর্ণ

High-level module (Service) নিচের module (Repository)-এর concrete class জানবে না — শুধু interface জানবে।

PHP// ❌ ভুল — concrete class-এ depend করা
class UserService
{
    public function __construct(
        private UserRepository $repo // concrete class!
    ) {}
}

// ✅ সঠিক — interface-এ depend করা
class UserService
{
    public function __construct(
        private UserRepositoryInterface $repo // interface!
    ) {}
    // এখন MySQL → MongoDB → Redis যেকোনো কিছুতে switch করা যাবে
}
