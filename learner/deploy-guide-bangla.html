Local vs Live — কোথায় কী করবেন?
এটা প্রথমে বুঝলে বাকি সব সহজ হয়ে যাবে
💡
সহজ ব্যাখ্যা Local = আপনার Windows ল্যাপটপ। এখানে কোড লিখবেন, test করবেন।
Live (Server) = Hetzner VPS (178.104.78.234)। এখানে আসল অ্যাপ চলবে, ক্লায়েন্ট ব্যবহার করবে।
কাজ / ফাইল 🖥️ Local (আপনার PC) 🌐 Live (Server)
Dockerfile এখানে তৈরি করবেন, git-এ push করবেন git pull করলে এসে যাবে, build হবে
docker-compose.yml এখানে তৈরি ও edit করবেন git pull করলে এসে যাবে
docker/nginx/default.conf এখানে তৈরি ও edit করবেন git pull করলে এসে যাবে
.env ফাইল local-এ আলাদা .env (local DB, APP_DEBUG=true) সার্ভারে manually তৈরি (real password, DEBUG=false)
.env.example তৈরি করুন, git-এ push করুন reference হিসেবে থাকবে, .env-এর template
vendor/ (Composer) local-এ composer install চালান Dockerfile-এর ভেতরে install হয় — আলাদা চালাতে হয় না
node_modules/ (npm) local-এ npm install চালান Dockerfile-এর ভেতরে install হয় — আলাদা চালাতে হয় না
public/build/ (Vite) local-এ npm run dev (hot reload) Dockerfile build-এ npm run build — compiled file তৈরি হয়
npm run dev ✅ Local development-এ চালান ❌ সার্ভারে চালাবেন না — npm run build চলে
npm run build local test-এ চালাতে পারেন Dockerfile-এ automatically চলে
php artisan serve ✅ Local dev-এ চালান (port 8000) ❌ সার্ভারে চালাবেন না — Nginx + PHP-FPM ব্যবহার হয়
php artisan migrate local DB-তে চালান docker compose exec app php artisan migrate --force
Docker Desktop Windows-এ install করুন (optional test) ❌ লাগবে না — Docker Engine (CLI only) use হয়
APP_DEBUG true (error দেখা যাবে) false — ক্লায়েন্ট error দেখতে পারবে না
DB_HOST 127.0.0.1 বা localhost mysql (Docker service নাম)
APP_URL http://localhost বা http://localhost:8000 http://178.104.78.234 (বা domain)
⚠️
সবচেয়ে গুরুত্বপূর্ণ পার্থক্য Local-এ npm run dev চালিয়ে কোড লিখবেন। Server-এ npm run dev কখনো চালাবেন না — npm run build চলে এবং সেটা Dockerfile-এই হয়ে যায়।

02
প্রজেক্ট ফোল্ডার স্ট্রাকচার
কোন ফাইল কোথায় থাকবে — visual tree view
📁
কেন এই স্ট্রাকচার? Docker config ফাইলগুলো (Dockerfile, docker-compose.yml) প্রজেক্টের root-এ রাখলে সবকিছু একসাথে git-এ থাকে। docker/ folder-এ Nginx, PHP config আলাদা রাখলে পরিষ্কার থাকে।
রং মানে: 🔴 নতুন তৈরি করতে হবে 🟢 Laravel install-এই থাকে 🔵 git-এ push করুন 🟡 সার্ভারে manually তৈরি 🟠 auto-generated
my-laravel-app/ ← আপনার Laravel প্রজেক্ট root (git repo)
├── Dockerfile নতুন তৈরি করুন PHP image-এর রেসিপি
├── docker-compose.yml নতুন তৈরি করুন সব service (PHP+Nginx+MySQL+Redis) একসাথে
├── .dockerignore নতুন তৈরি করুন image build-এ vendor, node_modules বাদ দেবে
├── .env সার্ভারে manually তৈরি DB password, APP_KEY — git-এ push করবেন না!
├── .env.example git-এ push করুন password ছাড়া template — সবার জন্য
├── docker/ নতুন তৈরি করুন Docker service config
│ ├── nginx/
│ │ └── default.conf নতুন তৈরি করুন Nginx web server config — routing, PHP-FPM
│ └── php/
│ └── php.ini নতুন তৈরি করুন PHP memory limit, upload size, OPcache settings
├── app/ Laravel-এ আছে Models, Controllers, Services, Middleware
├── database/ Laravel-এ আছে Migrations, Seeders, Factories
├── resources/ Laravel-এ আছে
│ ├── js/ Vue 3 + Inertia source files
│ │ ├── Pages/ Inertia page components (Home.vue, Dashboard.vue...)
│ │ ├── Components/ Reusable Vue components
│ │ └── app.js Vue + Inertia bootstrap entry point
│ └── views/ app.blade.php (Inertia root template)
├── public/ Laravel-এ আছে Web root — Nginx এটাই serve করে
│ └── build/ auto-generated Vite compiled JS/CSS — git-এ রাখবেন না (.gitignore)
├── routes/ Laravel-এ আছে web.php, api.php — Laravel routes
├── storage/ Laravel-এ আছে Logs, file uploads, session, cache
├── composer.json Laravel-এ আছে PHP package dependencies
├── package.json Laravel-এ আছে Node/Vue dependencies (Vite, Vue 3, Inertia)
└── vite.config.js Laravel-এ আছে Vite configuration — Vue plugin, input files
⚠️
.gitignore-এ এগুলো অবশ্যই থাকতে হবে /vendor, /node_modules, /public/build, .env — এগুলো git-এ push করবেন না। Laravel-এর default .gitignore-এ সাধারণত থাকে।
03
Dockerfile — প্রতিটি অংশ কেন লাগবে?
Laravel 12 + Inertia + Vue 3 এর জন্য বিস্তারিত ব্যাখ্যা
🐳
Dockerfile কী? Dockerfile হলো একটি রেসিপি ফাইল। এটা Docker-কে বলে: "এই Linux version নাও, এই software install করো, এই কোড copy করো।" এর ফলে যেকোনো সার্ভারে হুবহু একই environment তৈরি হয়।
⚡
Multi-Stage Build কেন? Stage 1: Node.js দিয়ে Vue/Vite build করি। Stage 2: PHP-FPM-এ শুধু compiled assets নিয়ে আসি। Final image-এ Node.js থাকে না — image size 700MB → ~180MB হয়!
🔵 Stage 1 — Node.js দিয়ে Frontend Build (Vue 3 + Vite compile করতে)
STAGE 1
FROM node:20-alpine AS frontend
Node.js base image — Vue compile করার environment
▶
Dockerfile — Line 1
FROM node:20-alpine AS frontend
কেন?
Vue 3 এবং Inertia.js হলো JavaScript framework। Browser বুঝতে পারে না raw .vue file — এটাকে compile করতে হয়। এই compile কাজ করে Vite, আর Vite চালাতে লাগে Node.js।

alpine = হালকা Linux (মাত্র 5MB), full Ubuntu না।
AS frontend = এই stage-এর নাম, Stage 2 থেকে ডাকতে হবে।
STAGE 1
COPY package*.json → RUN npm ci
Vue dependencies install — cache-friendly পদ্ধতি
▶
Dockerfile
WORKDIR /app
# package.json আগে copy করি
COPY package*.json ./
RUN npm ci --frozen-lockfile
কেন আগে শুধু package.json?
Docker layer cache ব্যবহার করে। আপনি Home.vue বদলালে package.json বদলায় না — তাই npm install বারবার run হবে না। Build অনেক দ্রুত হয়।

npm ci = npm install-এর faster version। package-lock.json থেকে exact version নেয় — সব সার্ভারে same version।
STAGE 1
COPY . . → RUN npm run build
Vue 3 + Inertia → public/build/ compiled assets তৈরি
▶
Dockerfile
COPY . .
# Vite build — Vue components compile করে
RUN npm run build
এটা কী করে?
Vite আপনার Vue files compile করে browser-friendly JS/CSS বানায়:

resources/js/Pages/Home.vue → public/build/assets/Home-abc123.js
resources/js/app.js → public/build/assets/app-xyz789.js

Browser raw .vue file বোঝে না, কিন্তু compiled .js বোঝে।

এটাই local-এ npm run dev আর server-এ npm run build-এর পার্থক্য।
🟢 Stage 2 — PHP 8.3-FPM (Laravel Application) — এটাই final production image
STAGE 2
FROM php:8.3-fpm-alpine
PHP base image — Laravel চালানোর environment
▶
Dockerfile
FROM php:8.3-fpm-alpine
কেন php-fpm?
PHP-FPM (FastCGI Process Manager) = Laravel application চালানোর engine।

কীভাবে কাজ করে: Browser → Nginx (request receive) → PHP-FPM (Laravel process) → Nginx → Browser।

এই Stage 2-ই হবে final image। Stage 1 (Node.js) final image-এ থাকবে না — শুধু compiled Vue files নিয়ে আসবে।
STAGE 2
RUN apk add — System Libraries
PHP extensions compile করার জন্য system-level C libraries
▶
Dockerfile
RUN apk add --no-cache \
    git curl unzip libpng-dev libjpeg-turbo-dev \
    libwebp-dev freetype-dev libzip-dev \
    oniguruma-dev icu-dev linux-headers shadow
কেন এই packages?
PHP extensions (যেমন GD image library) C language-এ লেখা source code compile করে install হয়। Compile করতে এই C libraries দরকার। এগুলো না থাকলে PHP extension install হবে না।
libpng-dev, libjpeg-turbo-dev
GD extension-এর জন্য — image resize, thumbnail তৈরি
libzip-dev
zip extension-এর জন্য — ZIP file handle করা
oniguruma-dev
mbstring-এর জন্য — Bengali/Unicode string support
icu-dev
intl extension-এর জন্য — date, number, currency format
freetype-dev
GD font rendering — image-এ text লেখা
git, curl, unzip
Composer দিয়ে packages download করতে
💡
Inertia+Vue প্রজেক্টে image feature না থাকলে? Image upload/resize না থাকলে libpng-dev libjpeg-turbo-dev libwebp-dev freetype-dev বাদ দিতে পারেন। GD extension-ও বাদ দিন। Image size ছোট হবে।
STAGE 2
docker-php-ext-install — PHP Extensions
Laravel-এর জন্য দরকারি PHP modules — কোনটা কেন লাগবে
▶
Dockerfile
RUN docker-php-ext-configure gd \
    --with-freetype --with-jpeg --with-webp \
  && docker-php-ext-install -j$(nproc) \
    pdo_mysql mbstring exif pcntl bcmath \
    gd zip intl opcache
pdo_mysql 🔴
MySQL database connect করতে — ছাড়া অ্যাপ চলবেই না
mbstring 🔴
Bengali/Unicode text — Laravel itself require করে
opcache 🔴
PHP bytecode cache — app 3-5x দ্রুত হয়। Production-এ অবশ্যই
bcmath 🔴
Precise calculation — Laravel-এর কিছু feature require করে
pcntl 🟡
Queue worker, async jobs — Laravel Horizon লাগলে দরকার
gd 🟡
Image resize/thumbnail — image feature না থাকলে বাদ দিন
zip 🟡
ZIP file — Composer দরকার, export feature থাকলে
intl 🟠
Date/number/currency format — optional, but useful
🔴 = না হলেই নয় 🟡 = থাকলে ভালো 🟠 = optional
STAGE 2
pecl install redis
Redis PHP extension — cache এবং session-এর জন্য
▶
Dockerfile
RUN pecl install redis && docker-php-ext-enable redis
কেন আলাদা?
Redis extension PHP core-এর অংশ না — third-party (PECL)। তাই আলাদা command লাগে।

.env-এ CACHE_DRIVER=redis এবং SESSION_DRIVER=redis দিলে এই extension দরকার। Redis ব্যবহার না করলে বাদ দিতে পারেন (তখন CACHE_DRIVER=file দিন)।
STAGE 2
Composer install + COPY কোড
PHP packages install করা এবং কোড image-এ রাখা
▶
Dockerfile
# Composer (PHP package manager) install
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

WORKDIR /var/www/html

# composer.json আগে copy — npm-এর মতো cache benefit
COPY composer*.json ./
RUN composer install --no-dev --optimize-autoloader --no-scripts

# সব Laravel কোড copy করি
COPY . .

# Stage 1 থেকে compiled Vue/JS assets নিয়ে আসি
COPY --from=frontend /app/public/build ./public/build
--no-dev কেন?
--no-dev = development packages (PHPUnit, Faker ইত্যাদি) install করবে না। Production-এ দরকার নেই, image ছোট থাকে।

--optimize-autoloader = class autoloading দ্রুত হয়।

COPY --from=frontend = এখানেই multi-stage magic! Stage 1-এ compile হওয়া Vue assets এখানে আসে।
STAGE 2
Permission + Cache + CMD
Laravel final setup এবং container start command
▶
Dockerfile
# Laravel production cache optimize
RUN php artisan config:cache \
 && php artisan route:cache \
 && php artisan view:cache

# storage folder write permission
# www-data = Nginx/PHP-FPM-এর user
RUN chown -R www-data:www-data /var/www/html \
 && chmod -R 775 storage bootstrap/cache

EXPOSE 9000
CMD ["php-fpm"]
cache কেন?
config:cache = সব config file একটা file-এ। প্রতিটি request-এ config পড়তে হয় না — অনেক দ্রুত।
route:cache = routes compile হয়। বড় অ্যাপে অনেক speed পাওয়া যায়।
view:cache = Blade templates pre-compile হয়।

www-data permission: Nginx/PHP-FPM এই user হিসেবে চলে। storage folder-এ লিখতে পারতে হবে — logs, uploads, sessions রাখতে।

প্রজেক্টের root-এ সম্পূর্ণ Dockerfile তৈরি করুন:

Dockerfile — সম্পূর্ণ ফাইল
# ════════════════════════════════════════════════
# STAGE 1: Node.js দিয়ে Frontend Build
# Vue 3 + Vite → public/build/ তৈরি করবে
# ════════════════════════════════════════════════
FROM node:20-alpine AS frontend

WORKDIR /app

# package.json আগে copy — cache optimization
COPY package*.json ./
RUN npm ci --frozen-lockfile

# সব ফাইল copy করে Vite build চালাই
COPY . .
RUN npm run build
# ফলে: resources/js/*.vue → public/build/assets/*.js


# ════════════════════════════════════════════════
# STAGE 2: PHP 8.3-FPM Application
# Alpine = হালকা Linux, production-ready
# ════════════════════════════════════════════════
FROM php:8.3-fpm-alpine

# PHP extensions compile করতে C libraries লাগে
RUN apk add --no-cache \
    git curl unzip libpng-dev libjpeg-turbo-dev \
    libwebp-dev freetype-dev libzip-dev \
    oniguruma-dev icu-dev linux-headers shadow

# PHP Extensions install
# pdo_mysql=DB, mbstring=Unicode, opcache=Speed
RUN docker-php-ext-configure gd \
    --with-freetype --with-jpeg --with-webp \
  && docker-php-ext-install -j$(nproc) \
    pdo_mysql mbstring exif pcntl bcmath \
    gd zip intl opcache

# Redis extension (session, cache এর জন্য)
RUN pecl install redis && docker-php-ext-enable redis

# Composer (PHP package manager)
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer

WORKDIR /var/www/html

# composer.json আগে copy → cache optimization
COPY composer*.json ./
RUN composer install --no-dev --optimize-autoloader --no-scripts

# সম্পূর্ণ Laravel project copy
COPY . .

# custom php.ini copy
COPY docker/php/php.ini /usr/local/etc/php/conf.d/custom.ini

# Stage 1 থেকে compiled Vue assets নিয়ে আসি
COPY --from=frontend /app/public/build ./public/build

# Production cache optimize
RUN php artisan config:cache \
 && php artisan route:cache \
 && php artisan view:cache

# storage folder-এ www-data permission দিই
RUN chown -R www-data:www-data /var/www/html \
 && chmod -R 775 storage bootstrap/cache

EXPOSE 9000
CMD ["php-fpm"]

04
docker-compose.yml তৈরি করুন
PHP + Nginx + MySQL + Redis একসাথে পরিচালনা
🔗
Docker Compose কেন? আমাদের app-এ 4টি parts। Docker Compose এদের সবাইকে একটি কমান্ডে (docker compose up -d) চালু করে। এরা নিজেদের service নাম দিয়ে ডাকতে পারে — DB_HOST=mysql মানে mysql container।
docker-compose.yml
version: '3.9'

services:

  # ── PHP Laravel Application ───────────────────
  app:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: laravel_app
    restart: unless-stopped   # server restart হলে auto চালু
    volumes:
      # storage host-এ mount — container delete হলেও files থাকে
      - ./storage:/var/www/html/storage
    environment:
      - APP_ENV=production
      - APP_KEY=${APP_KEY}         # .env থেকে নেবে
      - DB_HOST=mysql              # docker service নাম = hostname
      - DB_DATABASE=${DB_DATABASE}
      - DB_USERNAME=${DB_USERNAME}
      - DB_PASSWORD=${DB_PASSWORD}
      - REDIS_HOST=redis
      - CACHE_DRIVER=redis
      - SESSION_DRIVER=redis
    depends_on:
      mysql:
        condition: service_healthy  # MySQL ready হলে তারপর start
      redis:
        condition: service_started
    networks:
      - app-network

  # ── Nginx Web Server ──────────────────────────
  nginx:
    image: nginx:1.25-alpine
    container_name: laravel_nginx
    restart: unless-stopped
    ports:
      - "80:80"             # সার্ভারের port 80 → Nginx port 80
      - "443:443"           # HTTPS SSL এর জন্য
    volumes:
      - ./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf:ro
      - ./public:/var/www/html/public:ro
      - ./storage/app/public:/var/www/html/storage/app/public:ro
    depends_on:
      - app
    networks:
      - app-network

  # ── MySQL 8 Database ──────────────────────────
  mysql:
    image: mysql:8.0
    container_name: laravel_mysql
    restart: unless-stopped
    environment:
      MYSQL_DATABASE: ${DB_DATABASE}
      MYSQL_USER: ${DB_USERNAME}
      MYSQL_PASSWORD: ${DB_PASSWORD}
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql   # data persistent
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - app-network

  # ── Redis Cache & Session ─────────────────────
  redis:
    image: redis:7-alpine
    container_name: laravel_redis
    restart: unless-stopped
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    networks:
      - app-network

# Persistent volumes — data সার্ভারে থাকে
volumes:
  mysql_data:
  redis_data:

# Internal network — বাইরে থেকে দেখা যায় না
networks:
  app-network:
    driver: bridge

Nginx Config তৈরি করুন
docker/nginx/default.conf
⚡
Nginx কীভাবে কাজ করে? Browser → Nginx (80 port) → Static file? নিজেই দেয়। PHP file? PHP-FPM (app:9000)-এ পাঠায় → Laravel process করে → Browser-এ ফেরত। Inertia.js-এ সব request index.php-এ যায়, Laravel route করে।
docker/nginx/default.conf
server {
    listen 80;
    server_name _;   # _ = যেকোনো domain/IP accept করবে

    # Laravel-এর public folder = web root
    root  /var/www/html/public;
    index index.php index.html;

    client_max_body_size 100M;

    # Gzip compression — page দ্রুত load হয়
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/javascript;

    # Static assets — browser 1 বছর cache করবে
    location ~* \.(css|js|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        try_files $uri =404;
    }

    # Laravel routing — সবচেয়ে গুরুত্বপূর্ণ line
    # Inertia.js সব route index.php দিয়ে serve করে
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    # PHP files → PHP-FPM container-এ পাঠাও
    # app:9000 মানে 'app' service, port 9000
    location ~ \.php$ {
        fastcgi_pass   app:9000;
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include        fastcgi_params;
        fastcgi_read_timeout 300;
    }

    # Hidden files block (.htaccess, .git ইত্যাদি)
    location ~ /\.(?!well-known).* {
        deny all;
    }

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
}
